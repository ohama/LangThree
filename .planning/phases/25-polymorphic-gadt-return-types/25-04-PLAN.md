Fix the GADT check-mode branch folder in Bidir.fs so that when `expected` is an unbound
TVar (polymorphic mode), each branch gets an independent result type. This makes
`eval : 'a Expr -> 'a` work — IntLit branch returns int, BoolLit branch returns bool,
without the first branch's unification poisoning the second.

Purpose: Close COV-01 and TYP-03 gaps. The phase goal specifies OCaml-style polymorphic
GADT return as a must-have; without this fix the feature is incomplete.

Output: Bidir.fs with per-branch independent result in polymorphic mode, new GADT-05 F#
unit test, updated gadt-poly-eval.flt testing the two-branch cross-type pattern.

  <name>Task 1: Fix Bidir.fs GADT check-mode folder for per-branch independent result type</name>
  <files>src/LangThree/Bidir.fs</files>
  <action>
The root cause is in the GADT check-mode `folder` function (lines 541-618).

The folder signature is `(s, idx) -> (pat, guard, body) -> (s', idx+1)`.
`s` accumulates substitutions across branches. On line 605:

    let refinedExpected = apply sGuard (apply (compose combinedLocalS s) expected)

When `expected` is a fresh TVar from synth delegation, branch 1 (IntLit n -> n) runs
`bodyS = check body int`, which unifies freshTy=int. This bodyS is composed into s
on line 618. So when branch 2 (BoolLit b -> b) runs, `apply s expected` resolves to
`int` before localS can refine it to `bool` — causing E0301.

**The fix:** Detect whether `expected` is an unbound TVar at the point we compute
`refinedExpected`. If it is polymorphic (the TVar is not bound in `s`), create a
FRESH local expected for each branch and do NOT compose `bodyS` into the cross-branch
accumulator `s` for that binding. Concrete mode (expected is already a concrete type
or TVar bound in s) keeps the current behavior exactly.

Step-by-step implementation in `src/LangThree/Bidir.fs`:

1. Add a helper just before or at the start of the GADT check match handler
   (around line 538). Name it `isPolyExpected`:

   ```fsharp
   // Returns true if `expected` under substitution `s` is still an unbound TVar.
   // In this mode each branch must refine independently — we give each branch a
   // fresh local result TVar rather than sharing the same one.
   let isPolyExpected (s: Subst) (expected: Type) : bool =
       match apply s expected with
       | TVar _ -> true
       | _ -> false
   ```

   Place this as a local `let` inside the `| Match (scrutinee, clauses, span) when isGadtMatch ...`
   arm, defined before the `folder` function.

2. Inside the GADT branch of `folder` (the `| Some ctorInfo when ctorInfo.IsGadt ->` arm),
   replace lines 604-618 with the following logic:

   **If polymorphic mode** (`isPolyExpected s expected` is true):
   - Create a fresh local TVar: `let localExpected = freshVar()`
   - Apply localS to it (it starts unbound, localS pins it via the constructor): since
     localExpected is fresh and not constrained by localS yet, we actually want:
     `let refinedLocalExpected = apply combinedLocalS localExpected`
     But localExpected is fresh so this is just `localExpected`. Instead, do:
     `let refinedLocalExpected = apply (compose combinedLocalS s) localExpected`
     which is still `localExpected` (unbound). So the correct approach is:
     We want the local result to be `apply combinedLocalS expected` — use localS to
     refine a fresh copy of expected. Since expected is TVar n (unbound), and localS
     may not map n at all (it maps the scrutinee type param), we need:
     `let refinedLocalExpected = apply combinedLocalS (freshVar())`
     This fresh var is independent per branch and localS does not bind it either.

     Actually the cleanest approach: create a fresh per-branch result TVar that is
     ONLY constrained by `bodyS` for that branch. The branch body is checked against
     this fresh var. We do NOT compose `bodyS` into the accumulator `s` — we just
     let the branch succeed or fail locally.

     ```fsharp
     if isPolyExpected s expected then
         // Polymorphic mode: each branch gets an independent result TVar.
         // bodyS constraints are branch-local and not propagated to s.
         let branchResultTy = freshVar()
         let sGuardApplied = apply sGuard branchResultTy  // still TVar (unbound by sGuard)
         let _ = check ctorEnv recEnv clauseCtx branchEnv' body branchResultTy
         // Existential escape check still applies
         let resultTy = apply bodyS branchResultTy
         for ev in existentialFreshVars do
             if Set.contains ev (freeVars resultTy) then
                 raise (TypeException { Kind = ExistentialEscape ev; Span = span
                                        Term = None; ContextStack = clauseCtx; Trace = [] })
         // Do NOT compose bodyS into s — branch result stays independent
         (s, idx + 1)
     else
         // Concrete mode: keep existing behavior exactly
         let refinedExpected = apply sGuard (apply (compose combinedLocalS s) expected)
         let bodyS = check ctorEnv recEnv clauseCtx branchEnv' body refinedExpected
         let resultTy = apply bodyS refinedExpected
         for ev in existentialFreshVars do
             if Set.contains ev (freeVars resultTy) then
                 raise (TypeException { Kind = ExistentialEscape ev; Span = span
                                        Term = None; ContextStack = clauseCtx; Trace = [] })
         (compose bodyS s, idx + 1)
     ```

     Note: `bodyS` from the check call is the substitution returned by `check`; assign it
     properly. The polymorphic branch does `let bodyS = check ...` but discards it from `s`.

3. The `finalS` returned from `List.fold folder (s1, 0) clauses` is still `finalS` on
   line 655. In polymorphic mode finalS == s1 (the scrutinee synth subst). The synth
   caller at line 280 does `(s, apply s freshTy)` — `apply finalS freshTy` will remain
   TVar (unbound) since no branch pinned it. This is correct: the function `eval` will
   have a polymorphic type `'a Expr -> 'a` at the let-binding boundary where
   generalization occurs.
