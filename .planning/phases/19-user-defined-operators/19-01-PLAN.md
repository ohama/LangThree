<objective>
Implement user-defined infix operators using the OCaml INFIXOP0-4 approach.

Lexer classifies multi-character operator tokens by first character into precedence
buckets. Parser has one grammar rule per bucket. Operators desugar to function application.
</objective>

<tasks>

<task id="1" title="Add INFIXOP0-4 tokens to lexer">
**Lexer.fsl:**

1. Add token declarations for INFIXOP0-4 (each carries a string payload):
   - Already in Parser.fsy: declare `%token<string> INFIXOP0 INFIXOP1 INFIXOP2 INFIXOP3 INFIXOP4`

2. Add operator character set and classification function.
   Define helper `classifyOperator` that maps first character to token:
   - `=`, `<`, `>`, `|`, `&`, `$`, `!` → INFIXOP0 (comparison level)
   - `@`, `^` → INFIXOP1 (concatenation, right-assoc)
   - `+`, `-` → INFIXOP2 (additive)
   - `*`, `/`, `%` → INFIXOP3 (multiplicative)
   - `*` followed by `*` → INFIXOP4 (exponentiation, right-assoc)

3. Add catch-all operator rule AFTER all existing specific rules:
   ```
   | op_char op_char+   { classifyOperator (LexBuffer.LexemeString lexbuf) }
   ```

   Where op_char includes: `! $ % & * + - . / < = > ? @ ^ | ~`

   **CRITICAL:** This must come AFTER all existing operator rules (`|>`, `>>`, `<<`,
   `&&`, `||`, `<=`, `>=`, `<>`, `::`, `->`, `<-`, `..`, `+`, `-`, `*`, `/`, `=`,
   `<`, `>`, `|`, `.`). The fslex longest-match + first-match guarantees existing
   operators match first.

4. Also need single-char custom operators? No — single chars like `+` are already
   built-in. Custom operators must be 2+ chars (e.g., `++`, `<+>`, `|>|`).
   The regex `op_char op_char+` ensures minimum 2 characters.

Build and verify existing tests still pass.
</task>

<task id="2" title="Add INFIXOP parser rules and operator definition syntax">
**Parser.fsy:**

1. Declare INFIXOP tokens:
   ```
   %token<string> INFIXOP0 INFIXOP1 INFIXOP2 INFIXOP3 INFIXOP4
   ```

2. Declare precedence (insert at appropriate levels):
   ```
   %left INFIXOP0
   %right INFIXOP1
   %left INFIXOP2
   %left INFIXOP3
   %right INFIXOP4
   ```

3. Add expression rules for each INFIXOP level. Each desugars to function application:
   ```
   | Expr INFIXOP0 Expr  { App(App(Var($2, ...), $1, ...), $3, ...) }
   | Expr INFIXOP1 Expr  { App(App(Var($2, ...), $1, ...), $3, ...) }
   | Expr INFIXOP2 Expr  { App(App(Var($2, ...), $1, ...), $3, ...) }
   | Expr INFIXOP3 Expr  { App(App(Var($2, ...), $1, ...), $3, ...) }
   | Expr INFIXOP4 Expr  { App(App(Var($2, ...), $1, ...), $3, ...) }
   ```

   Place these at the correct precedence level in the grammar (INFIXOP2 near +/-,
   INFIXOP3 near *//, etc.).

4. Add operator definition syntax — `let (op) params = body`:
   Add an `OpName` nonterminal:
   ```
   OpName:
       | LPAREN INFIXOP0 RPAREN { $2 }
       | LPAREN INFIXOP1 RPAREN { $2 }
       | LPAREN INFIXOP2 RPAREN { $2 }
       | LPAREN INFIXOP3 RPAREN { $2 }
       | LPAREN INFIXOP4 RPAREN { $2 }
   ```

   Modify LetDecl rules to accept OpName as the binding name:
   ```
   | LET OpName IDENT IDENT EQUALS Expr NEWLINE
       { LetDecl($2, desugarParams [$3; $4] $6 ..., ...) }
   ```

   Similarly for LetRecDecl.

5. Add `(op)` as an Atom (operator used as function):
   ```
   | LPAREN INFIXOP0 RPAREN { Var($2, ...) }
   | LPAREN INFIXOP1 RPAREN { Var($2, ...) }
   // etc.
   ```

Build and fix any LALR(1) conflicts.
</task>
