1. In Exhaustive.fs, add a new function that filters a constructor set to only include constructors whose return type is compatible with the scrutinee type:

```fsharp
/// Filter constructors to only those possible for the given scrutinee type.
/// For GADT types, eliminates constructors whose return type head doesn't match.
/// For example, if scrutinee is `int expr`, eliminates `Bool : bool -> bool expr`
/// because `bool expr` and `int expr` have different type arguments.
///
/// Simple approach: only check the HEAD of type arguments for concrete mismatches.
/// TVar arguments are always compatible (they could be anything).
let filterPossibleConstructors
    (ctorEnv: Type.ConstructorEnv)
    (scrutineeType: Type.Type)
    (fullConstructorSet: ConstructorSet)
    : ConstructorSet =
    match scrutineeType with
    | Type.TData(typeName, scrutArgs) ->
        // Check if this type has any GADT constructors
        let hasGadt =
            ctorEnv
            |> Map.exists (fun _ info ->
                match info.ResultType with
                | Type.TData(n, _) when n = typeName -> info.IsGadt
                | _ -> false)
        if not hasGadt then
            fullConstructorSet  // Non-GADT type: all constructors possible
        else
            // Check if scrutinee args contain any type variables
            let scrutHasVars = scrutArgs |> List.exists (fun arg ->
                not (Set.isEmpty (Type.freeVars arg)))
            if scrutHasVars then
                fullConstructorSet  // Generic scrutinee: all constructors possible
            else
                // Concrete scrutinee: filter by type argument compatibility
                fullConstructorSet
                |> List.filter (fun ctor ->
                    match Map.tryFind ctor.Name ctorEnv with
                    | Some info when info.IsGadt ->
                        match info.ResultType with
                        | Type.TData(_, ctorArgs) ->
                            // Check each arg pair: concrete args must match
                            List.forall2 (fun scrutArg ctorArg ->
                                // If constructor arg has type vars, it's compatible
                                if not (Set.isEmpty (Type.freeVars ctorArg)) then true
                                // If scrutinee arg has type vars, it's compatible
                                elif not (Set.isEmpty (Type.freeVars scrutArg)) then true
                                // Both concrete: must be equal
                                else scrutArg = ctorArg
                            ) scrutArgs ctorArgs
                        | _ -> true  // Non-TData result: keep it
                    | _ -> true  // Non-GADT or unknown: keep it
                )
    | _ -> fullConstructorSet  // Non-TData scrutinee: keep all
```

2. In TypeCheck.fs, modify the exhaustiveness checking section of `typeCheckModule` to use the filtered constructor set for GADT types.

Currently, the code does:
```fsharp
let constructorSet = getConstructorsFromEnv ctorEnv scrTy
```

Change to also compute a filtered set when the type is a GADT:
```fsharp
let constructorSet = getConstructorsFromEnv ctorEnv scrTy
// For GADT types with concrete scrutinee, filter impossible constructors
let possibleConstructors = Exhaustive.filterPossibleConstructors ctorEnv scrTy constructorSet
```

Then use `possibleConstructors` for exhaustiveness/redundancy checking instead of `constructorSet`.

BUT WAIT: the scrTy used here comes from `inferTypeFromPatterns` which returns the generic type (from plan 02). For exhaustiveness, we want the GENERIC type (to list all constructors), then filter based on the actual scrutinee type from type checking.

The challenge: `typeCheckModule` currently doesn't have the inferred scrutinee type from Bidir.synth available in the exhaustiveness section -- it only has the patterns and the match span.

Two approaches:
a. Thread the scrutinee type through (complex, requires refactoring collectMatches)
b. Infer the scrutinee type from the constructors in patterns (use pattern constructor return types)

Approach (b) is simpler: look at the constructor patterns, get their return types, and determine the most specific scrutinee type. If all patterns use `Int` and `Add` (both `int expr`), the scrutinee is `int expr`. If patterns mix types or use generic constructors like `If : bool expr * 'a expr * 'a expr -> 'a expr`, then some args are type variables.

Actually, the simpler approach is: for exhaustiveness with GADTs, we still check against ALL constructors (generic type) but accept that some "missing" patterns are impossible. So instead of filtering the constructor set before checking, filter the RESULT: after checkExhaustive reports missing patterns, filter out those whose constructors are impossible.

Simpler implementation:

```fsharp
// After computing missing patterns from checkExhaustive:
match Exhaustive.checkExhaustive constructorSet casePats with
| Exhaustive.NonExhaustive missing ->
    // For GADT types, filter out impossible missing patterns
    let actualMissing =
        if hasGadtConstructors ctorEnv scrTy then
            missing |> List.filter (fun missingPat ->
                match missingPat with
                | Exhaustive.ConstructorPat(name, _) ->
                    // Check if this constructor is possible for the scrutinee type
                    // (leave it if possible, remove if impossible)
                    List.exists (fun c -> c.Name = name) possibleConstructors
                | _ -> true)
        else missing
    if not (List.isEmpty actualMissing) then
        // ... emit warning with actualMissing
```

Actually, let me simplify: since we don't have the concrete scrutinee type in the current exhaustiveness path, and getting it requires significant refactoring, let's take the simpler approach:

**Use filterPossibleConstructors to reduce the constructor set BEFORE exhaustiveness checking.** The scrTy from `inferTypeFromPatterns` currently returns the generic type. Change it: also try to infer a more specific type from the patterns. If all ConstructorPat constructors have the same concrete type args in their result types, use that as the scrutinee type.

```fsharp
/// Infer the most specific scrutinee type from constructor patterns
let inferScrutineeType (ctorEnv: ConstructorEnv) (patterns: Pattern list) : Type option =
    let ctorResultTypes =
        patterns
        |> List.choose (fun pat ->
            match pat with
            | Ast.ConstructorPat(name, _, _) ->
                match Map.tryFind name ctorEnv with
                | Some info -> Some info.ResultType
                | None -> None
            | _ -> None)
    match ctorResultTypes with
    | [] -> None
    | first :: _ -> Some first  // Use first constructor's specific return type
```

Wait, that's basically what the current `inferTypeFromPatterns` does. The difference is that now for GADTs, `info.ResultType` is the SPECIFIC return type (e.g., TData("expr", [TInt])), not the generic one. So the current code would actually return the first pattern's specific type.

So the flow becomes:
1. Get scrTy from `inferTypeFromPatterns` -- for GADT, this might be TData("expr", [TInt])
2. Get ALL constructors via `getConstructorsFromEnv` using the generic type name
3. Filter with `filterPossibleConstructors` using scrTy
4. Run exhaustiveness check with filtered set

The issue: `getConstructorsFromEnv` needs the type NAME to find all constructors. It currently matches on `TData(typeName, _)` -- the typeName is the same regardless of instantiation, so this works fine.

So update `inferTypeFromPatterns` to return the SPECIFIC result type of the first constructor pattern (which it already does now that GADT constructors have specific result types), and pass that to `filterPossibleConstructors`.

But also keep the generic lookup for `getConstructorsFromEnv`:
```fsharp
match inferTypeFromPatterns patterns with
| Some scrTy ->
    match scrTy with
    | TData(typeName, _) ->
        let genericTy = TData(typeName, ???)  // Need generic params
        let allConstructors = getConstructorsFromEnv ctorEnv (TData(typeName, []))
        // Actually getConstructorsFromEnv only uses the typeName, not the args
```

Looking at `getConstructorsFromEnv`: it matches on `TData(typeName, _)` and filters by result type name. It doesn't care about type args. So passing ANY TData with the right name works.

Final implementation plan:
1. `inferTypeFromPatterns` returns first constructor's ResultType (specific for GADT)
2. `getConstructorsFromEnv` extracts all constructors for that type name
3. `filterPossibleConstructors` filters based on the specific scrutinee type
4. Use filtered set for exhaustiveness

This approach requires NO refactoring of collectMatches or the type-checking pipeline. It uses the constructor patterns to infer the scrutinee type, which is sufficient for the common case.
