**Approach:** The existing `inferTypeFromPatterns` in TypeCheck.fs already returns the first constructor's ResultType from ctorEnv. After Plan 04-02, GADT constructors have SPECIFIC result types (e.g., `TData("expr", [TInt])` for `Int`), not the generic one. We leverage this: (1) `inferTypeFromPatterns` returns the specific scrutinee type, (2) `getConstructorsFromEnv` uses just the type name to get ALL constructors, (3) `filterPossibleConstructors` filters by type arg compatibility, (4) exhaustiveness runs on filtered set.

**1. In Exhaustive.fs**, add `filterPossibleConstructors`:

```fsharp
let filterPossibleConstructors
    (ctorEnv: Type.ConstructorEnv)
    (scrutineeType: Type.Type)
    (fullConstructorSet: ConstructorSet)
    : ConstructorSet =
    match scrutineeType with
    | Type.TData(typeName, scrutArgs) ->
        let hasGadt =
            ctorEnv
            |> Map.exists (fun _ info ->
                match info.ResultType with
                | Type.TData(n, _) when n = typeName -> info.IsGadt
                | _ -> false)
        if not hasGadt then
            fullConstructorSet
        else
            let scrutHasVars = scrutArgs |> List.exists (fun arg ->
                not (Set.isEmpty (Type.freeVars arg)))
            if scrutHasVars then
                fullConstructorSet  // Generic scrutinee: all constructors possible
            else
                fullConstructorSet
                |> List.filter (fun ctor ->
                    match Map.tryFind ctor.Name ctorEnv with
                    | Some info when info.IsGadt ->
                        match info.ResultType with
                        | Type.TData(_, ctorArgs) ->
                            List.forall2 (fun scrutArg ctorArg ->
                                if not (Set.isEmpty (Type.freeVars ctorArg)) then true
                                elif not (Set.isEmpty (Type.freeVars scrutArg)) then true
                                else scrutArg = ctorArg
                            ) scrutArgs ctorArgs
                        | _ -> true
                    | _ -> true)
    | _ -> fullConstructorSet
```

**2. In TypeCheck.fs**, modify the exhaustiveness checking section. After computing `scrTy` from `inferTypeFromPatterns` and `constructorSet` from `getConstructorsFromEnv`:

```fsharp
let constructorSet = getConstructorsFromEnv ctorEnv scrTy
let possibleConstructors = Exhaustive.filterPossibleConstructors ctorEnv scrTy constructorSet
```

Use `possibleConstructors` instead of `constructorSet` for both `checkExhaustive` and `checkRedundant` calls.

Note: `getConstructorsFromEnv` only uses the type NAME from `TData(typeName, _)`, so it correctly returns ALL constructors regardless of the specific type args in scrTy.
