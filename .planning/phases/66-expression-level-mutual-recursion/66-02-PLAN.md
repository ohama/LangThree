  <action>
**Bidir.fs** -- Rewrite the `LetRec` synth handler. Adapt from the TypeCheck.fs LetRecDecl handler (lines ~852-903):

1. Create fresh type variables for each function in the bindings list:
   ```fsharp
   let funcTypes = bindings |> List.map (fun (name, param, paramTyOpt, _, _) ->
       let paramTy = match paramTyOpt with Some te -> elaborateTypeExpr te | None -> freshVar()
       let retTy = freshVar()
       (name, param, TArrow(paramTy, retTy), paramTy, retTy))
   ```
2. Add ALL functions to env simultaneously with monomorphic schemes
3. Type-check each body in the extended env, unify body type with expected return type
4. Compose all substitutions
5. Generalize all function types and extend env for inExpr
6. Synth the inExpr in the final env
7. Return composed substitution and inExpr type

Handle return type annotations: if the parser rule had `COLON TypeExpr`, the body is wrapped in `Annot(body, retTypeExpr, span)` -- this is handled naturally by the existing synth dispatch, no special code needed here.

**Infer.fs** -- Same pattern as Bidir.fs but using `inferWithContext` instead of `synth`. Mirror the Bidir.fs implementation.

**Eval.fs** -- Rewrite the `LetRec` eval handler. Adapt from the LetRecDecl eval handler (lines ~1632):

1. Create a `sharedEnvRef = ref env`
2. Map over bindings to create BuiltinValue wrappers that capture `sharedEnvRef`
3. Build `mutualEnv` by folding all (name, funcValue) pairs into env
4. Set `sharedEnvRef := mutualEnv`
5. Eval `inExpr` in `mutualEnv` with `tailPos`

This is the pattern from the research section "Pattern 4: Evaluator."

IMPORTANT: For `InLetRecBody` diagnostic context in Bidir.fs/Infer.fs, use the per-binding name and span from the binding tuple.
