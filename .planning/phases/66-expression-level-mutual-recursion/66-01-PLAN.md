  <action>
Update all 4 expression-level `let rec` rule variants (lines 176-206) to include `LetRecContinuation` between the body and `IN`.

Read the existing 4 rules first. They follow these patterns:
1. `LET REC IDENT MixedParamList EQUALS Expr IN SeqExpr` (no return type, no indent)
2. `LET REC IDENT MixedParamList EQUALS INDENT Expr DEDENT IN SeqExpr` (no return type, with indent)
3. `LET REC IDENT MixedParamList COLON TypeExpr EQUALS Expr IN SeqExpr` (with return type, no indent)
4. `LET REC IDENT MixedParamList COLON TypeExpr EQUALS INDENT Expr DEDENT IN SeqExpr` (with return type, with indent)

For each rule:
- Insert `LetRecContinuation` after the body Expr (or after DEDENT if indented) and before `IN`
- Update ALL positional `$N` references -- the continuation shifts IN and SeqExpr by +1
- The continuation result (a bindings list, possibly empty) gets prepended to the current binding
- Construct `LetRec((currentBinding :: $continuation), $seqExpr, span)`

CRITICAL: The `LetRecContinuation` nonterminal already exists (lines ~890-919) and returns `(string * string * TypeExpr option * Expr * Span) list`. It handles both the empty case (returns `[]`) and the `AND ...` chain case. Reuse it as-is.

Use the `desugarMixedParams` result to extract param/body, then match on Lambda vs LambdaAnnot to get paramType:
```fsharp
let lambda = desugarMixedParams $4 $6 span
match lambda with
| Lambda(p, b, _) -> LetRec(($3, p, None, b, ruleSpan parseState 3 6) :: $7, $9, span)
| LambdaAnnot(p, ty, b, _) -> LetRec(($3, p, Some ty, b, ruleSpan parseState 3 6) :: $7, $9, span)
| _ -> failwith "impossible"
```

Adjust `$N` references carefully for each variant (the numbers differ based on whether COLON TypeExpr and INDENT/DEDENT are present).

Also read the module-level LetRecDecl rules as reference -- they already use this exact pattern with LetRecContinuation.
