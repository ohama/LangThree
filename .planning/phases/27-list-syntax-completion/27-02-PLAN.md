    **SYN-04: List literal patterns**

    1. In the Parser.fsy header section (`%{ ... %}`), add a helper function:
       ```fsharp
       let rec desugarListPat (pats: Ast.Pattern list) (span: Ast.Span) : Ast.Pattern =
           match pats with
           | [] -> Ast.EmptyListPat(span)
           | p :: rest -> Ast.ConsPat(p, desugarListPat rest span, span)
       ```

    2. Add a new nonterminal `SemiPatList` (place it near SemiExprList or near the Pattern rules):
       ```fsharp
       SemiPatList:
           | Pattern                         { [$1] }
           | Pattern SEMICOLON               { [$1] }
           | Pattern SEMICOLON SemiPatList   { $1 :: $3 }
       ```

    3. In the `Pattern` nonterminal, add two new rules for list literal patterns. The `LBRACKET RBRACKET` rule for `EmptyListPat` already exists — add after it:
       ```fsharp
       | LBRACKET SemiPatList RBRACKET
           { desugarListPat $2 (ruleSpan parseState 1 3) }
       | LBRACKET SemiPatList SEMICOLON RBRACKET
           { desugarListPat $2 (ruleSpan parseState 1 4) }
       ```

    After changes, build and check for shift/reduce conflict warnings from fsyacc. If any appear, they will be in the build output — investigate before proceeding.
