- `s.Length` or `"literal".Length` -> `String.length s` or `String.length "literal"`

- `s.EndsWith(arg)` -> `String.endsWith s arg`
- `s.StartsWith(arg)` -> `String.startsWith s arg`
- `s.Trim()` or `" hi ".Trim ()` -> `String.trim s` or `String.trim " hi "`
- `arr.Length` -> `Array.length arr`

IMPORTANT: Do NOT change expected output sections (`// --- Output:`). The output values remain the same, only the syntax changes.

IMPORTANT: For String module functions like `String.contains`, the argument order is `String.contains str needle` (string first, then argument). Same for `String.endsWith str suffix`, `String.startsWith str prefix`.

IMPORTANT: For literal string calls like `"hello".Length`, rewrite as `String.length "hello"`. For `"hello".Contains("ell")`, rewrite as `String.contains "hello" "ell"`.

  <action>
Read each file and rewrite dot-notation calls to module function calls. Replacements:

- `StringBuilder ()` -> `StringBuilder.create ()` (for consistency with functional API)
- `sb.Append(arg)` -> `StringBuilder.add sb arg`
- `sb.ToString()` or `sb.ToString ()` -> `StringBuilder.toString sb`

For `stringbuilder-with-methods.flt` which ALSO uses String dot methods on the result:
- `result.Trim()` -> `String.trim result`
- `trimmed.StartsWith(arg)` -> `String.startsWith trimmed arg`
- `trimmed.EndsWith(arg)` -> `String.endsWith trimmed arg`
- `trimmed.Length` -> `String.length trimmed`

For `stringbuilder-chaining.flt`: The chaining pattern `sb.Append("a").Append("b")` must be rewritten. Since `StringBuilder.add` returns the StringBuilder, use sequential let bindings:
```
let sb = StringBuilder.create ()
let sb = StringBuilder.add sb "hello"
let sb = StringBuilder.add sb " "
let sb = StringBuilder.add sb "world"
let result = StringBuilder.toString sb
```

IMPORTANT: Do NOT change expected output sections. Output values are the same.
IMPORTANT: `StringBuilder.add` (NOT `StringBuilder.append`) was the rename decision from Phase 60.
