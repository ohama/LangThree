  <name>Task 1: Lex and parse `()` literal, `unit` type, `fun () ->` parameter</name>
  <files>src/LangThree/Lexer.fsl, src/LangThree/Parser.fsy</files>
  <action>
Make these five targeted changes to Lexer.fsl and Parser.fsy:

**Lexer.fsl — add `unit` keyword token (after the `"string"` keyword rule, before the general identifier rule):**
```
| "unit"        { TYPE_UNIT }
```
Also add `| Parser.TYPE_UNIT -> "TYPE_UNIT"` in Format.fs formatToken function (check if Format.fs has a formatToken that needs updating, similar to how TYPE_INT etc. are handled there).

**Parser.fsy — token declaration (after the existing `TYPE_INT TYPE_BOOL TYPE_STRING TYPE_LIST` line):**
```
%token TYPE_UNIT
```

**Parser.fsy — AtomicType grammar (add before the existing `TYPE_INT` rule):**
```
| TYPE_UNIT  { TETuple [] }
```

**Parser.fsy — Atom grammar (add as the FIRST rule in the Atom section, before `LPAREN Expr RPAREN`):**
```
| LPAREN RPAREN  { Tuple([], ruleSpan parseState 1 2) }
```

**Parser.fsy — Expr grammar (add `fun () -> body` rule after the existing `FUN LPAREN IDENT COLON TypeExpr RPAREN ARROW Expr` rule):**
```
| FUN LPAREN RPAREN ARROW Expr
    { LambdaAnnot("__unit", TETuple [], $5, ruleSpan parseState 1 5) }
```

**Parser.fsy — indented let continuation placeholder (line ~94, currently `Var("()", symSpan parseState 6)`):**
Replace `Var("()", symSpan parseState 6)` with `Tuple([], symSpan parseState 6)`. This fixes the Pitfall 1 from research: if `()` is now a real expression, the `Var("()")` sentinel would become an undefined variable reference.

After these changes, run `dotnet build` to verify the lexer/parser compile without conflicts.

**Type.fs — fix formatType to render TTuple [] as "unit":**

In `formatType`, add a case for `TTuple []` BEFORE the general `TTuple ts` case:
```fsharp

| TTuple ts -> ts |> List.map formatType |> String.concat " * "
```

In `formatTypeNormalized` (the inner `format` function), add the same fix:
```fsharp
| TTuple [] -> "unit"
| TTuple ts -> ts |> List.map format |> String.concat " * "
```

**Parser.fsy — `let _ = expr` at Decl level (add to Decl grammar section, covering both flat and indented forms):**
```
| LET UNDERSCORE EQUALS Expr
    { LetDecl("_", $4, ruleSpan parseState 1 4) }

    { LetDecl("_", $5, ruleSpan parseState 1 6) }

    { LetPat(WildcardPat(ruleSpan parseState 2 2), $4, $6, ruleSpan parseState 1 6) }
```
