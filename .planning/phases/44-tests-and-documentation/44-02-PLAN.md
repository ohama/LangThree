Title: `# 가변 변수 (Mutable Variables)`

Opening paragraph: Connect from the previous chapter (mutable data structures like Array/Hashtable operate on containers). This chapter introduces `let mut` for mutable local variables — allowing individual bindings to be reassigned with `<-`.

Sections (progressive from basic to advanced):

**1. 기본 사용법 (Basic Usage)**
- `let mut x = 5` declares a mutable variable
- `x <- 10` reassigns it
- Show simple example with `$ cat` / `$ langthree` format:
  ```
  $ cat mut_basic.l3
  let result = let mut x = 1 in let _ = x <- 99 in x

  $ langthree mut_basic.l3
  99
  ```
- Explain: `let _ = x <- 99` is needed because `<-` returns unit
- Note both `mut` and `mutable` keywords work

**2. 모듈 수준 가변 변수 (Module-Level Mutable Variables)**
- Module-level: no `in` needed
  ```
  $ cat mut_module.l3
  let mut counter = 0
  let _ = counter <- 42
  let result = counter

  $ langthree mut_module.l3
  42
  ```
- Explain module-level behaves like sequential statements

**3. 다양한 타입 (Different Types)**
- Show string and bool examples
- Brief section, 1-2 examples

**4. 중첩 가변 변수 (Nested Mutable Variables)**
- Multiple let mut in same scope
- Example with two mutable vars

**5. 함수와 가변 변수 (Functions and Mutable Variables)**
- let mut inside function body (offside rule)
  ```
  $ cat mut_func.l3
  let counter n =
      let mut sum = 0
      let _ = sum <- n + 10
      sum
  let result = counter 5

  $ langthree mut_func.l3
  15
  ```

**6. 클로저 캡처 (Closure Capture)**
- Closure reading updated mutable variable
- Closure writing to mutable variable
- Two examples with explanation

**7. 조건문과 패턴 매칭에서의 사용 (With Conditionals and Pattern Matching)**
- if-then-else branch reassigning mutable
- match arm reassigning mutable
- Brief examples

**8. 에러 케이스 (Error Cases)**
- Immutable variable reassignment -> E0320 error
- Type mismatch on reassignment -> E0301 error
- Show error output format

**9. 불변 vs 가변: 언제 사용할까? (When to Use Mutable Variables)**
- Table comparing scenarios (similar to the table in ch19)
- Default to immutable `let`
- Use `let mut` when: accumulator in loop, closure-based state, imperative algorithms
- Remind: Array/Hashtable for mutable containers, `let mut` for mutable bindings

**10. 요약 (Summary)**
- Syntax reference table:
  | 구문 | 설명 |
  | `let mut x = expr` | 가변 변수 선언 |
  | `x <- expr` | 재할당 (unit 반환) |
  | `let mutable x = expr` | `mut`의 동의어 |
