<objective>
Add `e1; e2` expression sequencing to LangThree by introducing a `SeqExpr` nonterminal in Parser.fsy, desugaring to the existing `LetPat(WildcardPat, e1, e2)` node.

Purpose: Enables multi-step imperative code (print effects, mutable variable chains) without the verbose `let _ = e1 in e2` boilerplate. This is a pure parser change — no AST nodes, no eval/type-checker changes needed.

Output: Updated Parser.fsy with SeqExpr nonterminal + a new tests/flt/expr/seq/ directory with 5 flt tests covering SEQ-01, SEQ-02, SEQ-03.
</objective>

**Step 1 — Update the `start` rule** (line 103-104):
```

    | SeqExpr EOF   { $1 }
```

**Step 2 — Add `SeqExpr` nonterminal** after the `start` rule and before the `Expr` rule:
```fsharp
// Phase 45 (Expression Sequencing): SeqExpr wraps Expr, adds e1; e2 sequencing
// SEQ-01: e1; e2 evaluates e1 (discards result), then evaluates and returns e2
// SEQ-02: Multi-statement via right-associativity: e1; e2; e3 = e1; (e2; e3)
// Desugars to existing LetPat(WildcardPat, e1, e2) = let _ = e1 in e2
SeqExpr:
    | Expr SEMICOLON SeqExpr
        { LetPat(WildcardPat(symSpan parseState 2), $1, $3, ruleSpan parseState 1 3) }
    | Expr SEMICOLON
        { $1 }   // trailing semicolon: silently ignored
    | Expr
        { $1 }
```

**Step 3 — Update `INDENT Expr DEDENT` body rule** in the `Expr` nonterminal (currently line 174):
```
| INDENT SeqExpr DEDENT   { $2 }
```

**Step 4 — Update all `IN Expr` positions** in `Expr` nonterminal to `IN SeqExpr`. There are approximately 10 occurrences. Each `IN Expr` at the end of a let/let-rec/let-mut/let-pat/let-wildcard/if-then-else rule is a statement position. Specifically:

- `LET IDENT EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET IDENT EQUALS INDENT Expr DEDENT IN Expr` → last `Expr` becomes `SeqExpr`
- `LET IDENT ParamList EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET IDENT ParamList EQUALS INDENT Expr DEDENT IN Expr` → last `Expr` becomes `SeqExpr`
- `LET IDENT LPAREN RPAREN EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET IDENT LPAREN RPAREN EQUALS INDENT Expr DEDENT IN Expr` → last `Expr` becomes `SeqExpr`
- `LET TuplePattern EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET UNDERSCORE EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `IF Expr THEN Expr ELSE Expr` → both branch `Expr` become `SeqExpr`
- `LET REC IDENT ParamList EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET REC IDENT ParamList EQUALS INDENT Expr DEDENT IN Expr` → last `Expr` becomes `SeqExpr`
- `LET MUTABLE IDENT EQUALS Expr IN Expr` → last `Expr` becomes `SeqExpr`
- `LET MUTABLE IDENT EQUALS INDENT Expr DEDENT IN Expr` → last `Expr` becomes `SeqExpr`
- `FUN IDENT ARROW Expr` → `Expr` becomes `SeqExpr`
- `FUN IDENT ARROW INDENT Expr DEDENT` → `Expr` becomes `SeqExpr`
- All other `FUN ... ARROW Expr` variants (annotated, unit param, curried, tuple param) → the lambda body `Expr` becomes `SeqExpr`
- `TRY Expr WITH MatchClauses` → the try body `Expr` becomes `SeqExpr`
- `TRY INDENT Expr DEDENT WITH MatchClauses` → the try body `Expr` becomes `SeqExpr`
- Same for the bare `TRY Expr WITH IDENT ARROW Expr` and `TRY INDENT Expr DEDENT WITH IDENT ARROW Expr` variants — both the try body AND the handler body become `SeqExpr`

**Step 5 — Update `MatchClauses`** (arrow body positions):
```
MatchClauses:
    | PIPE OrPattern ARROW SeqExpr                         { [($2, None, $4)] }
    | PIPE OrPattern WHEN Expr ARROW SeqExpr               { [($2, Some $4, $6)] }
    | PIPE OrPattern ARROW SeqExpr MatchClauses            { ($2, None, $4) :: $5 }
    | PIPE OrPattern WHEN Expr ARROW SeqExpr MatchClauses  { ($2, Some $4, $6) :: $7 }
```
Note: the `WHEN Expr` guard stays as `Expr` (it is a test expression, not a statement body).

**Step 6 — Update `Decl` rules** RHS body positions to `SeqExpr` where the expression is the binding body:
- `LET TuplePattern EQUALS Expr` → `SeqExpr`
- `LET TuplePattern EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET UNDERSCORE EQUALS Expr` → `SeqExpr`
- `LET UNDERSCORE EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET IDENT EQUALS Expr` → `SeqExpr`
- `LET IDENT EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET IDENT ParamList EQUALS Expr` → `SeqExpr`
- `LET IDENT ParamList EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET IDENT LPAREN RPAREN EQUALS Expr` → `SeqExpr`
- `LET IDENT LPAREN RPAREN EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET MUTABLE IDENT EQUALS Expr` → `SeqExpr`
- `LET MUTABLE IDENT EQUALS INDENT Expr DEDENT` → `SeqExpr`
- `LET OpName ParamList EQUALS Expr` → `SeqExpr`
- `LET OpName ParamList EQUALS INDENT Expr DEDENT` → `SeqExpr`
- The `SYN-08` rule `LET IDENT EQUALS Expr IN Expr` in `Decl` → last `Expr` becomes `SeqExpr`

**Do NOT change** these — they are separator/delimiter/argument contexts:
- `LBRACKET Expr SEMICOLON SemiExprList RBRACKET` in `Atom` (list literal)
- `SemiExprList` rule (list element positions)
- `RecordFieldBindings` (record field positions)
- `RecordPatFields`
- `ExprList` (tuple elements)
- `AppExpr` / `AppArgs` / `Atom` argument positions
- Pattern positions
- `WHEN Expr` guard in match clauses
- `MATCH Expr WITH` scrutinee
- `IF Expr THEN` condition
- `LET ... EQUALS Expr` binding RHS (the value being bound, not the body after IN)
- Type expressions
