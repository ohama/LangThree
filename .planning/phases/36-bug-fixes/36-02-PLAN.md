<objective>
Fix the parser so `try failwith "boom" with e -> "caught"` (inline, no leading pipe) is accepted.

Purpose: Currently the TRY-WITH grammar requires `MatchClauses` which always starts with `PIPE`. Bare single-clause `with e -> expr` has no `PIPE` so it fails to parse. Adding a `TryWithClauses` nonterminal that also accepts the bare `Pattern ARROW Expr` form fixes this.

Output: Updated Parser.fsy and regression tests in ExceptionTests.fs.
</objective>

// Phase 36 (PAR-01): TryWithClauses allows bare pattern -> expr (no leading pipe)
