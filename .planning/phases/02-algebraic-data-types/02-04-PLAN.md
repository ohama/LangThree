  <behavior>
**Input:** List of patterns and scrutinee type
**Output:** Warning with missing patterns OR confirmation of exhaustiveness

Test cases (input patterns -> expected output):

1. Simple ADT - Incomplete:
   - Type: `Option<'a>`
   - Patterns: `[Some x]`
   - Expected: Non-exhaustive, missing `[None]`

2. Simple ADT - Complete:
   - Type: `Option<'a>`
   - Patterns: `[None; Some x]`
   - Expected: Exhaustive

3. Nested patterns - Incomplete:
   - Type: `Option<Option<'a>>`
   - Patterns: `[None; Some None]`
   - Expected: Non-exhaustive, missing `[Some (Some _)]`

4. Redundant pattern:
   - Type: `Option<'a>`
   - Patterns: `[None; Some x; None]`
   - Expected: Redundant pattern at index 2 (second `None`)

5. Wildcard makes exhaustive:
   - Type: `Option<'a>`
   - Patterns: `[Some x; _]`
   - Expected: Exhaustive

6. Recursive type - Exhaustive:
   - Type: `Tree`
   - Patterns: `[Leaf; Node _]`
   - Expected: Exhaustive (constructor coverage, not value enumeration)

**Algorithm (Maranget usefulness):**

```
useful(constructors, prevPatterns, p):
  if prevPatterns == []:
    return true  // First pattern always useful

  if p is wildcard:
    // Wildcard useful if any constructor not covered
    for each constructor C in constructors:
      if useful(constructors, specialize(prevPatterns, C), C _):
        return true
    return false

  if p is Constructor(C, args):
    // Specialize previous patterns to C, check args recursively
    return useful(constructors, specialize(prevPatterns, C), args)

specialize(patterns, C):
  patterns.filter(pat => pat matches C or pat is wildcard)
          .map(pat => if pat is C(args) then args else wildcard)
```
