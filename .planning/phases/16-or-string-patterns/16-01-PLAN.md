<objective>
Add string constant patterns and or-patterns to LangThree's pattern matching system.

String patterns: `| "hello" -> expr` matches string literals.
Or-patterns: `| 1 | 2 | 3 -> expr` matches any of several patterns.

Based on 16-RESEARCH.md findings. Changes span 8 modules following existing pattern conventions.
</objective>

<task id="2" title="Add OrPat to Pattern DU and implement">
**Ast.fs:** Add `OrPat of Pattern list * Span` to the Pattern DU.

**Parser.fsy:** This is the tricky part. Need to parse `| 1 | 2 | 3 -> expr`.

Strategy from research: Introduce an `OrPattern` nonterminal that allows `Pattern PIPE Pattern`.
The key challenge is LALR(1) conflict with PIPE as both or-pattern separator and match clause separator.

**Approach 1 (preferred):** Parse or-patterns within the Pattern rule:
```
OrPattern:
    | Pattern PIPE Pattern  { OrPat([$1; $3], ruleSpan parseState 1 3) }
    | OrPattern PIPE Pattern  { match $1 with OrPat(pats, span) -> OrPat(pats @ [$3], ruleSpan parseState 1 3) | _ -> OrPat([$1; $3], ruleSpan parseState 1 3) }
```

Then use `OrPattern` in match clause where a pattern is expected.

**Approach 2 (fallback):** If LALR(1) conflicts, require parentheses: `| (1 | 2 | 3) -> expr`

**Eval.fs matchPattern:** Add:
```fsharp
| OrPat(pats, _), value -> pats |> List.tryPick (fun p -> matchPattern p value)
```

**Infer.fs inferPattern:** Add handling for OrPat — all alternatives must have same type, no bindings (or same bindings).

**Format.fs formatPattern:** Add formatting for OrPat.

**TypeCheck.fs:** Handle OrPat in any pattern traversal functions.

Build and test.
</task>

<task id="3" title="Integrate OrPat with decision tree and exhaustiveness">
**MatchCompile.fs:** Expand or-patterns before decision tree compilation. In `compileMatch`, expand each clause with `OrPat` into multiple clauses (one per alternative):

```fsharp
// Before compilation, expand OrPat in clause patterns
let expandOrPats clauses =
    clauses |> List.collect (fun (pat, guard, body) ->
        match pat with
        | OrPat(pats, _) -> pats |> List.map (fun p -> (p, guard, body))
        | _ -> [(pat, guard, body)]
    )
```

**Exhaustive.fs:** The research says `CasePat` already has `OrPat of CasePat list` (check if this is true). Update `astPatToCasePat` to handle `Ast.OrPat`. Update `specializeRow` to expand OrPat alternatives.

**Verify:** Or-patterns should work with exhaustiveness warnings:

Build and test all existing tests pass.
</task>
