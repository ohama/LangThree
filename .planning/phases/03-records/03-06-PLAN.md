5. In Parser.fsy, add SetField grammar rule. This needs careful placement. Add it as an Expr-level production (assignment has low precedence):
   ```
   // In Expr rule, add:
   | Atom DOT IDENT LARROW Expr  { SetField($1, $3, $5, ruleSpan parseState 1 5) }
   ```
   NOTE: This may conflict with the `Atom DOT IDENT` (FieldAccess) rule in Atom. The parser sees `Atom DOT IDENT` and needs to decide: is this FieldAccess (reduce) or the beginning of SetField (shift LARROW)? Since LARROW is a lookahead, the LR parser should handle this as a shift (continue to LARROW). If there is a conflict, use `%prec` declarations or restructure.

6. In Bidir.fs, add synth case for SetField:
   ```fsharp
   | SetField (expr, fieldName, value, span) ->
       let s1, exprTy = synth ctorEnv recEnv ctx env expr
       let resolvedTy = apply s1 exprTy
       match resolvedTy with
       | TData (typeName, typeArgs) ->
           match Map.tryFind typeName recEnv with
           | Some recInfo ->
               match recInfo.Fields |> List.tryFind (fun f -> f.Name = fieldName) with
               | Some fieldInfo ->
                   if not fieldInfo.IsMutable then
                       raise (TypeException { Kind = ImmutableFieldAssignment(typeName, fieldName); Span = span; Term = Some expr; ContextStack = ctx; Trace = [] })
                   let subst = List.zip recInfo.TypeParams typeArgs |> Map.ofList
                   let fieldTy = apply subst fieldInfo.FieldType
                   let s2, valTy = synth ctorEnv recEnv ctx (applyEnv s1 env) value
                   let s3 = unifyWithContext ctx [] span fieldTy valTy
                   // SetField returns unit (represented as empty tuple or a designated unit type)
                   // Since LangThree doesn't have a unit type yet, return the field type for now
                   // Actually, F# mutation returns unit. Use TTuple [] as unit representation.
                   (compose s3 (compose s2 s1), TTuple [])
               | None ->
                   raise (TypeException { Kind = UnboundField(typeName, fieldName); Span = span; Term = Some expr; ContextStack = ctx; Trace = [] })
           | None ->
               raise (TypeException { Kind = NotARecord typeName; Span = span; Term = Some expr; ContextStack = ctx; Trace = [] })
       | _ ->
           raise (TypeException { Kind = FieldAccessOnNonRecord resolvedTy; Span = span; Term = Some expr; ContextStack = ctx; Trace = [] })
   ```
   NOTE on return type: SetField should return unit. If LangThree uses `TupleValue []` as unit, use `TTuple []`. Otherwise, check what `Var("()", ...)` resolves to and use that type. If there's no unit type, return the field's type (less correct but works).

1. Implement mutable fields using ref cells in Eval.fs.

   The approach: when creating a RecordValue, mutable fields store `RefValue(ref actualValue)` while immutable fields store the value directly. However, this requires adding a `RefValue` variant to Value (Ast.fs). Alternative simpler approach: use F# mutable Map entries. But Map is immutable.

   RECOMMENDED APPROACH (simplest): Use a mutable dictionary or ref cells.

   Actually, the cleanest approach matching F# semantics:
   - Add `RefValue of Value ref` to the Value type in Ast.fs (if not already done in Plan 01)
   - Wait -- Plan 01 did NOT add RefValue. Add it now.

   In Ast.fs, add to Value:
   ```fsharp
   | RefValue of Value ref
   ```

   In Eval.fs:
   - When evaluating RecordExpr, check if fields are mutable (requires RecordEnv access in evaluator). Problem: the evaluator doesn't have RecordEnv.

   SIMPLER APPROACH: Always store all record fields as ref cells. Field access transparently dereferences. SetField updates the ref cell. This means RecordValue stores `Map<string, Value ref>` instead of `Map<string, Value>`.

   EVEN SIMPLER (and matching the research recommendation): Store mutable field values in RefValue wrapper at creation time:
   - Immutable field: `Value` directly
   - Mutable field: `RefValue(ref value)`
   - FieldAccess: if the value is `RefValue`, dereference; otherwise return directly
   - SetField: find the RefValue, update the ref

   But the evaluator needs to know which fields are mutable. Since RecordEnv isn't available in the evaluator, encode mutability in the AST or in the RecordExpr expression itself.

   PRACTICAL APPROACH: Since the type checker validates mutability, the evaluator can trust that SetField is only called on mutable fields. Store ALL fields as plain values in the Map. For SetField, use a different runtime representation:

   Change RecordValue to use `Map<string, Value ref>`:
   ```fsharp
   | RecordValue of typeName: string * fields: Map<string, Value ref>
   ```

   This requires updating all RecordValue creation sites:
   - RecordExpr: `Map.ofList (fields |> List.map (fun (n, e) -> (n, ref (eval env e))))`
   - RecordUpdate: copy refs, update specified ones
   - FieldAccess: dereference `!(Map.find fieldName fields)`
   - SetField: `(Map.find fieldName fields) := eval env value`
   - Equality: compare dereferenced values
   - Pattern match: dereference before matching
   - formatValue: dereference before formatting

   This is the CLEANEST approach. All fields use refs. Immutable protection is enforced at type-check time, not runtime.
