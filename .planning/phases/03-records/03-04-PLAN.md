1. Add `RecordValue` formatting in `formatValue`:
   ```fsharp
   | RecordValue (typeName, fields) ->
       let fieldStrs =
           fields
           |> Map.toList
           |> List.map (fun (name, value) -> sprintf "%s = %s" name (formatValue value))
       sprintf "{ %s }" (String.concat "; " fieldStrs)
   ```

2. Add `RecordPat` case in `matchPattern`:
   ```fsharp
   | RecordPat (fieldPats, _), RecordValue (_, fields) ->
       // Partial matching: only match specified fields
       let bindings =
           fieldPats
           |> List.map (fun (fieldName, pat) ->
               match Map.tryFind fieldName fields with
               | Some value -> matchPattern pat value
               | None -> None)  // Field not found = no match
       if List.forall Option.isSome bindings then
           Some (List.collect Option.get bindings)
       else
           None
   ```

3. Add `RecordExpr` evaluation in `eval`:
   ```fsharp
   | RecordExpr (_, fieldExprs, _) ->
       let fieldValues =
           fieldExprs
           |> List.map (fun (name, expr) -> (name, eval env expr))
           |> Map.ofList
       // Determine type name: for now use empty string, will be resolved at type check time
       // The type name can be inferred from field set (handled in type checker)
       RecordValue ("", fieldValues)
   ```
   NOTE: The type name in RecordValue is not critical for evaluation. The type checker has already validated the fields. However, for better runtime representation, consider storing the resolved type name. Since the evaluator doesn't have access to RecordEnv, one approach is to leave it as empty string for now and enhance later if needed. Alternatively, add a `resolvedTypeName` field to RecordExpr during type checking (but this requires AST mutation which is not idiomatic). The simplest approach: resolve type name by looking up field names at eval time. But the evaluator shouldn't need RecordEnv. Keep it simple: use empty string or the first option from RecordExpr.

   Better approach: Use `typeName` from RecordExpr AST if available (the parser may set it to None). For now, use a placeholder. The copy-and-update case preserves the source record's type name.

4. Add `FieldAccess` evaluation:
   ```fsharp
   | FieldAccess (expr, fieldName, _) ->
       match eval env expr with
       | RecordValue (_, fields) ->
           match Map.tryFind fieldName fields with
           | Some value -> value
           | None -> failwithf "Field not found: %s" fieldName
       | v -> failwithf "Field access on non-record value: %s" (formatValue v)
   ```

5. Add `RecordUpdate` evaluation:
   ```fsharp
   | RecordUpdate (source, updates, _) ->
       match eval env source with
       | RecordValue (typeName, fields) ->
           let updatedFields =
               updates
               |> List.fold (fun acc (name, expr) ->
                   Map.add name (eval env expr) acc) fields
           RecordValue (typeName, updatedFields)
       | v -> failwithf "Copy-and-update on non-record value: %s" (formatValue v)
   ```

6. Add structural equality for RecordValue in `Equal` and `NotEqual`:
   ```fsharp
   // In Equal case, add:
   | RecordValue (t1, f1), RecordValue (t2, f2) -> BoolValue (t1 = t2 && f1 = f2)
   // In NotEqual case, add:
   | RecordValue (t1, f1), RecordValue (t2, f2) -> BoolValue (t1 <> t2 || f1 <> f2)
   ```
   Note: F# structural equality on Map works correctly. Two Maps are equal if they have the same keys and values.
