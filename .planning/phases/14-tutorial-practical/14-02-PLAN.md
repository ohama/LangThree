<task id="1" title="Chapter 12: Prelude and Standard Library">
Read existing tutorial chapters to understand established style.

Write `tutorial/12-prelude.md`:

**Content to cover:**
- How Prelude works: `Prelude/*.fun` files loaded automatically at startup
- Prelude loading order: files sorted alphabetically, each type-checked and evaluated
- Option type (from Prelude/Option.fun):
  - `type Option 'a = None | Some of 'a`
  - Pattern matching on Option: `match opt with | Some x -> x | None -> default`
  - Common Option patterns: map, bind, default value
  - Using Option with pipe: `Some 5 |> optionMap double`
- How to extend Prelude: create new `Prelude/MyTypes.fun` file
  - Type declarations available across all user code
  - Constructor values available in evaluation
- Built-in type environment (type-only, not from Prelude files):
  - `map`, `filter`, `fold`, `length`, `reverse`, `append`, `hd`, `tl`
  - `id`, `const`, `compose`
  - These have type signatures but no runtime implementations in Prelude
  - Must be defined inline with `let rec ... in ...` pattern
- Difference between Prelude types (file-based, fully available) vs built-in type signatures (type-only)

**LangThree-specific:**
- Prelude types work in both --expr mode and file mode
- Built-in string/print functions come from initialBuiltinEnv (not Prelude)
- Constructor values from Prelude (None, Some) are available without `open`
</task>

<task id="2" title="Chapter 13: CLI Reference">
Write `tutorial/13-cli-reference.md`:

**Content to cover:**
- Expression mode: `langthree --expr 'expression'`
  - Evaluates single expression, prints result
  - Single line only (no indentation)
  - Examples with different types
- File mode: `langthree filename.lt`
  - Reads file, applies IndentFilter for indentation
  - Type checks, evaluates all declarations
  - Prints last let binding's value
  - Show example .lt file
- AST emission: `langthree --emit-ast --expr '...'` or `--emit-ast filename.lt`
  - Shows parsed AST structure
  - Useful for debugging parse issues
- Type emission: `langthree --emit-type --expr '...'` or `--emit-type filename.lt`
  - Shows inferred types
  - File mode: shows types of all top-level bindings
  - Excludes built-in and prelude bindings
- Token emission: `langthree --emit-tokens --expr '...'`
  - Shows lexer output (with IndentFilter for file mode)
  - Useful for debugging indentation issues
- REPL: `langthree` or `langthree --repl`
  - Interactive mode
  - No indentation support (single-line expressions only)
  - Environment persists across lines
- File naming: `.lt` extension (convention, not enforced)
- Error messages: line/column info, type error diagnostics
- Warnings: exhaustiveness (W0001, W0002), non-exhaustive handlers (W0003)

**Format:** Each mode gets its own section with multiple examples showing input and output.
