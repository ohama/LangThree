  <action>
Extend `parseModule` and `Decls` grammar rules to support module system syntax.

**1. Extend `parseModule` entry point:**

The existing rules are:
```
parseModule:
    | Decls EOF    { Module($1, ruleSpan parseState 1 2) }
    | EOF          { EmptyModule(ruleSpan parseState 1 1) }
```

Add rules for top-level module and namespace declarations. Top-level `module Name` does NOT require indentation (F# convention -- declarations follow at column 0):

```
parseModule:
    // Top-level module declaration (no indentation required for body)
    | MODULE QualifiedIdent Decls EOF
        { NamedModule($2, $3, ruleSpan parseState 1 4) }
    // Top-level namespace declaration (no indentation required for body)
    | NAMESPACE QualifiedIdent Decls EOF
        { NamespacedModule($2, $3, ruleSpan parseState 1 4) }
    // Existing: bare declarations (implicit module from filename)
    | Decls EOF    { Module($1, ruleSpan parseState 1 2) }
    | EOF          { EmptyModule(ruleSpan parseState 1 1) }
```

IMPORTANT: Place MODULE and NAMESPACE rules BEFORE the existing `Decls EOF` rule. Since `Decls` can start with `LET`, `TYPE`, etc., and the new rules start with `MODULE` / `NAMESPACE`, there should be no LALR(1) conflict (different lookahead tokens).

**2. Extend `Decls` with nested module and open directive:**

Add these alternatives to the existing `Decls` rule. Place them AFTER existing RecordDeclaration rules:

```
// Phase 5 (Modules): Nested module declaration
// module Name = <indent>decls<dedent>
| MODULE IDENT EQUALS INDENT Decls DEDENT
    { [ModuleDecl($2, $5, ruleSpan parseState 1 6)] }
| MODULE IDENT EQUALS INDENT Decls DEDENT Decls
    { ModuleDecl($2, $5, ruleSpan parseState 1 6) :: $7 }

// Phase 5 (Modules): Open directive
| OPEN QualifiedIdent
    { [OpenDecl($2, ruleSpan parseState 1 2)] }
| OPEN QualifiedIdent Decls
    { OpenDecl($2, ruleSpan parseState 1 2) :: $3 }
```

**Key design notes:**
- Nested modules use `module Name = INDENT body DEDENT` (the existing IndentFilter already handles INDENT/DEDENT for any indentation change).
- Top-level `module Name` does NOT use `=` (F# convention: top-level module declarations don't use equals sign).
- `open` takes a QualifiedIdent (dotted path like `MyModule` or `Outer.Inner`).
- No special handling needed in IndentFilter -- it already emits INDENT/DEDENT for indentation changes.

**Watch for LALR(1) conflicts:** If fsyacc reports shift-reduce conflicts, the most likely cause is ambiguity between `MODULE IDENT EQUALS` (nested module) in Decls and `MODULE QualifiedIdent` (top-level) in parseModule. These should not conflict because parseModule handles the top-level case and Decls handles the nested case. But if conflicts arise, add `%nonassoc MODULE` to the precedence declarations.
