Change the AST definitions for LetRec and LetRecDecl to include `TypeExpr option` for the first parameter, update the parser to capture (not discard) type annotations, and update all mechanical pattern-match sites so the project builds cleanly.

Purpose: This is the foundational change -- without it, first parameter type annotations on `let rec` bindings are silently dropped by the parser.
Output: A building project where the AST carries type info but it is not yet enforced by the type checker.

This is a single atomic change -- all sites must be updated together because F# won't compile with partial changes.

**Step 1: Ast.fs -- Change both type definitions**

Line 79 -- LetRec expression:
Change from: `LetRec of name: string * param: string * body: Expr * inExpr: Expr * span: Span`
Change to: `LetRec of name: string * param: string * paramType: TypeExpr option * body: Expr * inExpr: Expr * span: Span`

Line 360 -- LetRecDecl declaration:
Change from: `LetRecDecl of bindings: (string * string * Expr * Span) list * Span`
Change to: `LetRecDecl of bindings: (string * string * TypeExpr option * Expr * Span) list * Span`

Line 312 -- `spanOf` for LetRec: update pattern to add `_` for the new field:
`| LetRec (_, _, _, _, _, s) -> s`

Line 386 -- `declSpanOf` for LetRecDecl: only matches outer span, verify no change needed. If it destructures bindings, add `_`.

**Step 2: Parser.fsy -- Capture type annotations instead of discarding**

For ALL LetRec expression rules (lines ~180-210, 4 rules) and ALL LetRecDeclaration/LetRecContinuation rules (lines ~850-920, 8 rules + 2 OpName rules):

Current pattern extracts param name but discards type:
```
| LambdaAnnot(p, _, b, _) -> LetRec($3, p, b, $8, span)
```

Change to capture type:
```
| Lambda(p, b, _) -> LetRec($3, p, None, b, $8, span)
| LambdaAnnot(p, ty, b, _) -> LetRec($3, p, Some ty, b, $8, span)
```

Same pattern for LetRecDecl binding tuples:
```
| Lambda(p, b, _) -> ($3, p, None, b, ruleSpan parseState 3 6)
| LambdaAnnot(p, ty, b, _) -> ($3, p, Some ty, b, ruleSpan parseState 3 6)
```

For OpName rules (operators) which only use Lambda (never LambdaAnnot): add `None`:
```
($3, p, None, b, ruleSpan parseState 3 6)
```

LetRecContinuation rules: same pattern, return tuple with `None`/`Some ty`.

IMPORTANT: The research lists exact line numbers for each of the 14 LetRec sites and 14 LetRecDecl sites. Reference `65-RESEARCH.md` for the complete site inventory.

**Step 3: TypeCheck.fs -- Update mechanical destructuring sites (NOT the logic change)**

Update these pattern-match-only sites (no logic change, just add `_` for the new field):
- Line 350 `collectMatches`: `LetRec(n, p, _, rhs, body, s)` 
- Line 483 `collectTryWiths`: `LetRec(n, p, _, rhs, body, s)`
- Line 569 `collectModuleRefs`: `LetRec(n, p, _, rhs, body, s)`

  `LetRec(n, p, pty, rhs, body, s) -> LetRec(n, p, pty, rewriteModuleAccess modules rhs, rewriteModuleAccess modules body, s)`

For LetRecDecl in TypeCheck.fs (line 852 area): update binding destructuring to `(name, param, _paramTyOpt, body, sp)`. The LOGIC change (actually using paramTyOpt) is deferred to Plan 02 Task 1.
