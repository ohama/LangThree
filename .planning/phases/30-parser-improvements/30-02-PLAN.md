**Change 1 — SYN-01/SYN-06: Replace single-param expression LetRec with multi-param version**

Find the two existing expression-level let rec rules (around line 129-131):
```fsharp
| LET REC IDENT IDENT EQUALS Expr IN Expr  { LetRec($3, $4, $6, $8, ruleSpan parseState 1 8) }
| LET REC IDENT IDENT EQUALS INDENT Expr DEDENT IN Expr  { LetRec($3, $4, $7, $10, ruleSpan parseState 1 10) }
```

Replace BOTH rules (remove the old, add the new):
```fsharp
| LET REC IDENT ParamList EQUALS Expr IN Expr
    { let span = ruleSpan parseState 1 8
      let lambda = List.foldBack (fun param body -> Lambda(param, body, span)) $4 $6
      match lambda with
      | Lambda(p, b, _) -> LetRec($3, p, b, $8, span)
      | _ -> failwith "impossible: ParamList must produce at least one param" }
| LET REC IDENT ParamList EQUALS INDENT Expr DEDENT IN Expr
    { let span = ruleSpan parseState 1 10
      let lambda = List.foldBack (fun param body -> Lambda(param, body, span)) $4 $7
      match lambda with
      | Lambda(p, b, _) -> LetRec($3, p, b, $10, span)
      | _ -> failwith "impossible: ParamList must produce at least one param" }
```

Note: `ParamList` already handles 1+ params (line 547-549). Replacing the old IDENT IDENT rules with ParamList rules eliminates the LALR conflict described in research pitfall 4.

<task type="auto">
  <name>Task 2: Fix Eval.fs LetRec to use mutable-ref self-referential closure</name>
  <files>src/LangThree/Eval.fs</files>
  <action>
Find the current `LetRec` case in the `eval` function (around line 611-614):
```fsharp
| LetRec (name, param, funcBody, inExpr, _) ->
    let funcVal = FunctionValue (param, funcBody, env)
    let recFuncEnv = Map.add name funcVal env
    eval recEnv moduleEnv recFuncEnv tailPos inExpr
```

Replace it with the mutable-ref pattern (same as `LetRecDecl` around line 860-878):
```fsharp
| LetRec (name, param, funcBody, inExpr, _) ->
    // Create self-referential closure using mutable ref.
    // The naive approach (FunctionValue with env lacking self) breaks when this LetRec
    // is inside a lambda body, because applyFunc's augmentation is not called from
    // the trampoline in that context. Use BuiltinValue + mutable ref (same as LetRecDecl).
    let envRef = ref env
    let wrapper = BuiltinValue (fun argVal ->
        let callEnv = Map.add param argVal !envRef
        eval recEnv moduleEnv callEnv false funcBody)
    let recEnv' = Map.add name wrapper env
    envRef := recEnv'
    eval recEnv moduleEnv recEnv' tailPos inExpr
```

Key notes:
- `recEnv` and `moduleEnv` are already in scope as outer parameters of `eval`.
- `BuiltinValue` is already defined in Ast.fs/Value DU — verify with `grep -n "BuiltinValue" src/LangThree/Eval.fs`.
- The `false` for `tailPos` in the inner call is intentional: recursive calls inside the body are not in tail position from the outer trampoline's perspective.
- `envRef := recEnv'` ensures the closure sees itself in its own environment.

After the change, build:
```

```
