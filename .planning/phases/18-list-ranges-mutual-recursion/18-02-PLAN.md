<objective>
Add module-level `let rec` and mutual recursion with `and` keyword.

Currently `let rec` only works at expression level (with `in`). This adds:
- `let rec f x = body` at module level (single self-recursive function)
- `let rec f x = body and g y = body` (mutual recursion)

Uses AND_KW token (already exists for mutually recursive types).
</objective>

<tasks>

<task id="1" title="Add LetRecDecl to AST and parser">
**Ast.fs:** Add to Decl DU:
```fsharp
| LetRecDecl of bindings: (string * string * Expr * Span) list * Span
// Each binding: (name, param, body, span)
```

**Parser.fsy:** Add grammar rules for module-level let rec:
```
// Single let rec at module level
| LET REC IDENT IDENT EQUALS Expr NEWLINE
    { LetRecDecl([($3, $4, $6, ruleSpan ...)], ruleSpan ...) }

// Mutual recursion with and continuation
| LET REC IDENT IDENT EQUALS Expr AND_KW LetRecContinuation
    { LetRecDecl(($3, $4, $6, ruleSpan ...) :: $8, ruleSpan ...) }

LetRecContinuation:
| IDENT IDENT EQUALS Expr NEWLINE
    { [($1, $2, $4, ruleSpan ...)] }
| IDENT IDENT EQUALS Expr AND_KW LetRecContinuation
    { ($1, $2, $4, ruleSpan ...) :: $6 }
```

Also handle indented forms (INDENT/DEDENT variants).

**Format.fs:** Add formatting for LetRecDecl.
</task>

<task id="2" title="Implement mutual rec type checking">
**TypeCheck.fs typeCheckDecls:** Handle LetRecDecl:

1. For each binding in the group, create a fresh type variable
2. Add all functions to typeEnv with monomorphic types (TArrow(paramType, freshRetVar))
3. Check all bodies with the extended env
4. Generalize types after all bodies are checked
5. Add final generalized types to typeEnv

This mirrors F#'s approach to mutual recursion type inference.

Update any pattern matches on Decl in typeCheckDecls.
</task>

<task id="3" title="Implement mutual rec evaluation">
**Eval.fs evalModuleDecls:** Handle LetRecDecl:

1. Create placeholder closures for all functions
2. Build an environment with all function names bound
3. Recreate closures with the complete environment (so each function can call the others)
4. Add all functions to the module environment

```fsharp
| LetRecDecl(bindings, _) ->
    // Phase 1: Create initial closures with incomplete env
    let names = bindings |> List.map (fun (name, _, _, _) -> name)
    let mutable recEnvLocal = env
    for (name, param, body, _) in bindings do
        let funcVal = FunctionValue(param, body, recEnvLocal)
        recEnvLocal <- Map.add name funcVal recEnvLocal
    // Phase 2: Recreate closures with complete env (all names visible)
    let mutable finalEnv = env
    for (name, param, body, _) in bindings do
        let funcVal = FunctionValue(param, body, recEnvLocal)
        finalEnv <- Map.add name funcVal finalEnv
    (finalEnv, moduleEnv)
```

**Prelude.fs:** Handle LetRecDecl if the prelude evaluator encounters it.
</task>
