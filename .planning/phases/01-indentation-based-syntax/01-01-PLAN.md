  <name>Add SyntaxContext to IndentFilter for match expression tracking</name>
  <files>
    src/LangThree/IndentFilter.fs
  </files>
  <action>
    Extend IndentFilter module to track syntactic context:

    1. Add SyntaxContext discriminated union:
       - TopLevel (default)
       - InMatch of baseColumn: int (track where 'match' keyword starts)

    2. Add Context field to FilterState:
       - Context: SyntaxContext list (stack of contexts for nesting)

    3. Update initialState to include Context = [TopLevel]

    4. Create processNewlineWithContext function that:
       - Takes state, column, and optional next token lookahead
       - When in InMatch context and next token is PIPE:
         * Validate pipe aligns with match base column
         * Throw IndentationError if misaligned with clear message
         * Don't change indent stack for pipe lines
       - Otherwise delegates to existing processNewline logic

    5. Add context detection logic:
       - When MATCH token seen, push InMatch context with current column
       - When exiting match (via DEDENT below match level), pop context

    Implementation note: Keep existing processNewline function unchanged. Add processNewlineWithContext as wrapper that checks context first, then delegates to processNewline for non-match cases. This preserves backward compatibility with let-expressions.
  </action>

    Compilation succeeds with no errors
  </verify>
  <done>

  </done>
