  <name>Add Module grammar rule for top-level declarations</name>
  <files>
    src/LangThree/Parser.fsy

  <action>
    Extend parser grammar to support module-level structure:

    1. Add Decls non-terminal for declaration sequences:
       ```fsharp
       Decls:
           | Decl                      { [$1] }
           | Decl Decls                { $1 :: $2 }
       ```

    2. Add Decl non-terminal for single declarations:
       ```fsharp
       Decl:
           | LET IDENT EQUALS Expr NEWLINE  { LetDecl($2, $4, ruleSpan parseState 1 5) }
           | LET IDENT EQUALS INDENT Expr DEDENT NEWLINE
               { LetDecl($2, $5, ruleSpan parseState 1 7) }
       ```

    3. Add Module as new start symbol (or extend existing start):
       ```fsharp
       Module:
           | Decls EOF                 { Module($1, ruleSpan parseState 1 2) }
           | EOF                       { EmptyModule(ruleSpan parseState 1 1) }
       ```

    4. Update AST in Ast.fs if needed to include Module and LetDecl types:
       ```fsharp
       type Decl =
           | LetDecl of name: string * body: Expr * Span

       type Module =
           | Module of decls: Decl list * Span
           | EmptyModule of Span
       ```

    Implementation note: This is about FILE structure (multiple declarations at top level), NOT the module SYSTEM (namespaces, imports, qualified names - that's Phase 5). Top-level declarations must be at column 0. Each declaration can have indented body.

    NEWLINE tokens after declarations prevent shift/reduce conflicts by making declaration boundaries explicit.
  </action>

    Compilation succeeds with no shift/reduce conflicts
  </verify>
