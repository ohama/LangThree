    Part 1 - IndentFilter.fs:
    1. Add InFunctionApp variant to SyntaxContext:
       - InFunctionApp of baseColumn: int

    2. Add context detection for function applications:
       - When processing tokens, detect pattern: Atom NEWLINE (col > current) Atom
       - This indicates multi-line function application starting
       - Push InFunctionApp context with function's base column
       - Emit INDENT before first argument

    3. Add context exit logic:
       - When indentation returns to or below function base column
       - Pop InFunctionApp context
       - Emit DEDENT after last argument

    Part 2 - Parser.fsy:
    1. Extend AppExpr grammar rules (around line 124):
       ```fsharp
       AppExpr:
           | AppExpr Atom               { App($1, $2, ruleSpan parseState 1 2) }
           | AppExpr INDENT AppArgs DEDENT
               { (* Fold AppArgs into nested App nodes *)
                 List.fold (fun acc arg -> App(acc, arg, ruleSpan parseState 1 4))
                           $1 $3 }
           | Atom                       { $1 }

       AppArgs:
           | Atom                       { [$1] }
           | Atom AppArgs               { $1 :: $2 }
       ```

    2. Add AppArgs non-terminal for multi-line argument lists.

    Implementation note: F# formatting guidelines show arguments indented one level from function name:
    ```fsharp
    someFunction
        arg1
        arg2
    ```

    The INDENT/DEDENT tokens make this structure explicit to the parser, preventing ambiguity between multi-line application and separate expressions.
