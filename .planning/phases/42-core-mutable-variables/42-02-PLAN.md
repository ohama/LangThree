**Strategy for threading mutableVars:** The research identified ~67 call sites for synth/check. Changing all signatures is too invasive. Instead, use a **module-level mutable set** (like the existing `composeCounter` pattern in Eval.fs):

```fsharp
/// Mutable variables currently in scope (Phase 42)
/// Tracked at module level to avoid threading through all synth/check call sites
let mutable mutableVars : Set<string> = Set.empty
```

Place this near the top of Bidir.fs (after the opens, before the functions). This is safe because type checking is single-threaded and mutableVars scope is managed by save/restore in each LetMut case.

**Bidir.fs — Add LetMut synth case (near the Let case in synth):**
```fsharp
| LetMut (name, value, body, span) ->
    let s1, valueTy = synth ctorEnv recEnv (InLetRhs (name, span) :: ctx) env value
    let env' = applyEnv s1 env
    // NO generalization -- mutable variables must be monomorphic
    let scheme = Scheme([], apply s1 valueTy)
    let bodyEnv = Map.add name scheme env'
    let savedMutableVars = mutableVars
    mutableVars <- Set.add name mutableVars
    let s2, bodyTy = synth ctorEnv recEnv (InLetBody (name, span) :: ctx) bodyEnv body
    mutableVars <- savedMutableVars  // restore (name goes out of scope)
    (compose s2 s1, bodyTy)
```

**Bidir.fs — Add Assign synth case (near LetMut):**
```fsharp
| Assign (name, value, span) ->
    if not (Set.contains name mutableVars) then
        raise (TypeException {
            Kind = ImmutableVariableAssignment name
            Span = span
            Term = Some expr
            ContextStack = ctx
            Trace = []
        })
    match Map.tryFind name env with
    | Some scheme ->
        let varTy = instantiate scheme
        let s1, valTy = synth ctorEnv recEnv ctx env value
        let s2 = unifyWithContext ctx [] span (apply s1 varTy) valTy
        (compose s2 s1, TTuple [])  // returns unit
    | None ->
        raise (TypeException {
            Kind = UnboundVar name
            Span = span
            Term = Some expr
            ContextStack = ctx
            Trace = []
        })
```

**Bidir.fs — Add check-mode cases.** In the `check` function, the new nodes need handling. The simplest approach: add fallthrough to synth via subsumption (the existing pattern for most nodes). Check if there's a catch-all `| _ ->` in `check` that delegates to synth. If so, LetMut and Assign will be handled automatically. If not, add explicit cases that call synth and unify with the expected type:
```fsharp
| LetMut _ | Assign _ ->
    let s1, inferredTy = synth ctorEnv recEnv ctx env expr
    let s2 = unifyWithContext ctx [] (spanOf expr) (apply s1 inferredTy) expected
    compose s2 s1
```

**IMPORTANT:** When adding `mutableVars` as a module-level mutable, make sure to reset it to `Set.empty` at the start of each top-level type checking entry point (e.g., `synthTop` or wherever the entry function is). Search for the function that TypeCheck.fs calls into Bidir — that's where reset should happen. This prevents stale mutableVars from a previous file's type checking.
