  <name>Task 1: Create MatchCompile.fs with decision tree types and compilation algorithm</name>
  <files>src/LangThree/MatchCompile.fs</files>
  <action>
Create `src/LangThree/MatchCompile.fs` with `module MatchCompile` that opens `Ast`.

Follow the Jules Jacobs algorithm EXACTLY as described in the research file. The key insight: each clause starts with ONE entry in its Patterns map (`{rootVar -> fullPattern}`), and sub-patterns are only expanded during clause splitting (case a). Do NOT pre-flatten patterns.

**Types to define:**

1. `TestVar = int` -- fresh variable identifier for sub-values during compilation.

2. Mutable counter for fresh variable generation:
   ```fsharp
   let mutable private nextTestVar = 0
   let freshTestVar () =
       let v = nextTestVar
       nextTestVar <- nextTestVar + 1
       v
   let resetTestVarCounter () = nextTestVar <- 0
   ```

3. `MatchRow` record:
   ```fsharp
   type MatchRow = {
       Patterns: Map<TestVar, Pattern>   // variable -> pattern constraint
       Guard: Expr option                // optional when guard
       Body: Expr                        // right-hand side
       Bindings: Map<string, TestVar>    // pattern var name -> test variable
       OriginalIndex: int                // for redundancy tracking
   }
   ```

4. `DecisionTree` discriminated union:
   ```fsharp
   type DecisionTree =
       | Leaf of bindings: Map<string, TestVar> * guard: Expr option
               * body: Expr * fallback: DecisionTree option
       | Switch of testVar: TestVar * constructorName: string
                 * argVars: TestVar list * ifMatch: DecisionTree * ifNoMatch: DecisionTree
       | Fail
   ```

**Functions to implement (in order):**

1. `patternToConstructor (pat: Pattern) : (string * int) option`
   Maps a pattern to its constructor name and arity. Returns None for VarPat/WildcardPat (not constructor tests).
   - `ConstructorPat(name, None, _)` -> `Some(name, 0)`
   - `ConstructorPat(name, Some _, _)` -> `Some(name, 1)`
   - `TuplePat(pats, _)` -> `Some("#tuple_" + string(List.length pats), List.length pats)`
   - `ConsPat _` -> `Some("::", 2)`
   - `EmptyListPat _` -> `Some("[]", 0)`
   - `ConstPat(IntConst n, _)` -> `Some("#int_" + string n, 0)`
   - `ConstPat(BoolConst b, _)` -> `Some("#bool_" + (string b).ToLower(), 0)`
   - `RecordPat(fields, _)` -> `Some("#record_" + string(List.length fields), List.length fields)`
   - `VarPat _ | WildcardPat _` -> `None`

2. `extractSubPatterns (pat: Pattern) : Pattern list`
   Extracts the sub-patterns from a constructor-like pattern:
   - `ConstructorPat(_, None, _)` -> `[]`
   - `ConstructorPat(_, Some arg, _)` -> `[arg]`
   - `TuplePat(pats, _)` -> `pats`
   - `ConsPat(h, t, _)` -> `[h; t]`
   - `EmptyListPat _` -> `[]`
   - `ConstPat _` -> `[]`
   - `RecordPat(fields, _)` -> fields sorted alphabetically by name, then take the pattern from each `(name, pat)` pair
   - `VarPat _ | WildcardPat _` -> `[]` (should not be called on these)

3. `pushVarBindings (row: MatchRow) : MatchRow`
   Step 1 of Jacobs' algorithm. Iterates all entries in `row.Patterns`. For each entry where the pattern is VarPat or WildcardPat, removes it from Patterns and (for VarPat) adds the binding to Bindings. This leaves only constructor/constant tests in Patterns.
   ```fsharp
   let pushVarBindings (row: MatchRow) : MatchRow =
       let mutable patterns = row.Patterns
       let mutable bindings = row.Bindings
       for KeyValue(tv, pat) in row.Patterns do
           match pat with
           | VarPat(name, _) ->
               patterns <- Map.remove tv patterns
               bindings <- Map.add name tv bindings
           | WildcardPat _ ->
               patterns <- Map.remove tv patterns
           | _ -> ()
       { row with Patterns = patterns; Bindings = bindings }
   ```

4. `selectTestVariable (first: MatchRow) (allClauses: MatchRow list) : TestVar`
   Step 2: Heuristic. From the first clause's remaining pattern keys, select the one appearing in the most clauses overall. Tie-break: pick the smallest TestVar (closer to root).
   ```fsharp
   let selectTestVariable (first: MatchRow) (allClauses: MatchRow list) : TestVar =
       first.Patterns
       |> Map.toSeq |> Seq.map fst
       |> Seq.maxBy (fun tv ->
           let count = allClauses |> List.sumBy (fun c ->
               if Map.containsKey tv c.Patterns then 1 else 0)
           (count, -tv))  // maximize count, then minimize tv for tie-breaking
   ```

5. `splitClauses (testVar: TestVar) (ctorName: string) (freshVars: TestVar list) (clauses: MatchRow list) : MatchRow list * MatchRow list`
   Steps 3-4: For each clause, check its pattern at testVar:
   - **Case (a):** Pattern at testVar is a constructor matching ctorName. Remove testVar from Patterns, add sub-patterns mapped to freshVars. Add to YES only.
   - **Case (b):** Pattern at testVar is a DIFFERENT constructor. Add to NO only (unchanged).
   - **Case (c):** No pattern at testVar (already pushed as var/wildcard). Add to BOTH (unchanged).

   ```fsharp
   for clause in clauses do
       match Map.tryFind testVar clause.Patterns with
       | Some pat ->
           match patternToConstructor pat with
           | Some(name, _) when name = ctorName ->
               // Case (a): same constructor, expand sub-patterns
               let subPats = extractSubPatterns pat
               let newPatterns = clause.Patterns |> Map.remove testVar
               let expanded =
                   List.zip freshVars subPats
                   |> List.fold (fun acc (fv, sp) -> Map.add fv sp acc) newPatterns
               yesClauses.Add({ clause with Patterns = expanded })
           | Some _ ->
               // Case (b): different constructor, NO branch only
               noClauses.Add(clause)
           | None ->
               // VarPat/WildcardPat still here (shouldn't happen after pushVarBindings,
               // but handle defensively): treat as case (c)
               yesClauses.Add(clause)
               noClauses.Add(clause)
       | None ->
           // Case (c): no test for this variable, add to BOTH
           yesClauses.Add(clause)
           noClauses.Add(clause)
   ```

6. `compile (clauses: MatchRow list) : DecisionTree`
   The main recursive algorithm:
   ```fsharp
   let rec compile (clauses: MatchRow list) : DecisionTree =
       match clauses with
       | [] -> Fail
       | _ ->
           // Step 1: Push variable bindings
           let clauses' = clauses |> List.map pushVarBindings
           let first = clauses'.Head

           // Base case: first clause has no remaining constructor tests
           if Map.isEmpty first.Patterns then
               let fallback =
                   if clauses'.Tail.IsEmpty then None
                   else Some(compile clauses'.Tail)
               Leaf(first.Bindings, first.Guard, first.Body, fallback)
           else
               // Step 2: Select test variable (heuristic)
               let testVar = selectTestVariable first clauses'
               let pat = Map.find testVar first.Patterns
               let ctorName, arity =
                   match patternToConstructor pat with
                   | Some(n, a) -> (n, a)
                   | None -> failwith "Expected constructor pattern after pushVarBindings"
               let freshVars = List.init arity (fun _ -> freshTestVar())

               // Steps 3-4: Split and generate Switch
               let yesClauses, noClauses = splitClauses testVar ctorName freshVars clauses'
               Switch(testVar, ctorName, freshVars, compile yesClauses, compile noClauses)
   ```

7. `matchesConstructor (value: Value) (ctor: string) : bool`
   Tests if a runtime Value matches a constructor name:
   - `DataValue(name, _)` matches ctor when `name = ctor`
   - `ListValue []` matches `"[]"`
   - `ListValue(_ :: _)` matches `"::"`
   - `IntValue n` matches `"#int_N"` when `string n = ctor.Substring(5)`
   - `BoolValue b` matches `"#bool_B"` when `(string b).ToLower() = ctor.Substring(6)`
   - `TupleValue _` matches `"#tuple_N"` (always true for tuple, arity checked structurally)
   - `RecordValue _` matches `"#record_N"` (always true for record)
   - Otherwise false

8. `destructureValue (ctor: string) (value: Value) : Value list`
   Extracts sub-values after a successful constructor match:
   - `DataValue(_, None)` -> `[]`
   - `DataValue(_, Some arg)` -> `[arg]`
   - `ListValue(h :: t)` for `"::"` -> `[h; ListValue t]`
   - `ListValue []` for `"[]"` -> `[]`
   - `TupleValue vals` -> `vals`
   - `RecordValue(_, fields)` -> fields sorted alphabetically by name, dereference each ref: `fields |> Map.toList |> List.sortBy fst |> List.map (fun (_, r) -> !r)`
   - Otherwise `[]`

9. `evalDecisionTree (evalFn: Env -> Expr -> Value) (env: Env) (varEnv: Map<TestVar, Value>) (tree: DecisionTree) : Value`
   Walks the decision tree at runtime:
   - `Fail` -> `failwith "Match failure: no pattern matched"`
   - `Leaf(bindings, guard, body, fallback)`:
     - Build bodyEnv by folding bindings into env: for each `(name, tv)`, add `name -> Map.find tv varEnv` to env
     - If no guard: `evalFn bodyEnv body`
     - If guard: evaluate guard in bodyEnv. If `BoolValue true`, evaluate body. Otherwise, follow fallback (or fail if None).
   - `Switch(testVar, ctorName, argVars, ifMatch, ifNoMatch)`:
     - Get `testValue = Map.find testVar varEnv`
     - If `matchesConstructor testValue ctorName`:
       - Extract sub-values via `destructureValue ctorName testValue`
       - Extend varEnv with `List.zip argVars subValues`
       - Recurse into ifMatch
     - Else: recurse into ifNoMatch

10. `compileMatch (clauses: MatchClause list) : DecisionTree * TestVar`
    Entry point. Converts MatchClause list to a DecisionTree and returns the root TestVar.
    - Reset the test var counter (or use a local counter -- simplest: reset at start of each compileMatch call)
    - Create rootVar via freshTestVar()
    - Convert each clause `(pattern, guard, body)` at index i to a MatchRow:
      ```fsharp
      { Patterns = Map.ofList [(rootVar, pattern)]
        Guard = guard
        Body = body
        Bindings = Map.empty
        OriginalIndex = i }
      ```
    - Call `compile` on the resulting list
    - Return `(tree, rootVar)`

**Important design decisions:**
- Do NOT import or depend on Eval.fs. Use `evalFn: Env -> Expr -> Value` parameter to avoid circular module dependency.
- The module only opens Ast (for Pattern, Expr, MatchClause, Value, Env types).
- Each clause starts with ONE entry in Patterns: `{rootVar -> fullPattern}`. Sub-patterns are expanded only during clause splitting (case a). This matches Jacobs' algorithm exactly.
- Reset `nextTestVar` at the start of `compileMatch` so each compilation starts from 0.
