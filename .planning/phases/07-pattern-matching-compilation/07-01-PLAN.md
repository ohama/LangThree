<objective>
Create the MatchCompile.fs module implementing Jules Jacobs' binary decision tree compilation algorithm for pattern matching.

Purpose: This is the core algorithm module that transforms match clauses into efficient binary decision trees. It eliminates redundant constructor tests by compiling patterns into a tree where each constructor is tested at most once per path.

Output: A new MatchCompile.fs file with DecisionTree/AccessPath types, pattern normalization, clause splitting (cases a/b/c), heuristic test selection, compilation function, and decision tree evaluator.
</objective>

  <name>Task 1: Create MatchCompile.fs with decision tree types and compilation algorithm</name>
  <files>src/LangThree/MatchCompile.fs</files>
  <action>
Create `src/LangThree/MatchCompile.fs` with `module MatchCompile` that opens `Ast` and `Type`.

**Types to define:**

1. `AccessPath` discriminated union:
   - `Root` — the scrutinee itself
   - `Field of parent: AccessPath * index: int` — navigate into sub-value at index

2. `DecisionTree` discriminated union:
   - `Leaf of bindings: (string * AccessPath) list * bodyIndex: int` — match succeeded
   - `Fail` — no clause matched
   - `Switch of testVar: AccessPath * constructor: string * arity: int * onMatch: DecisionTree * onElse: DecisionTree` — binary test
   - `GuardedLeaf of bindings: (string * AccessPath) list * guardIndex: int * bodyIndex: int * fallthrough: DecisionTree` — guarded match with fallthrough

   Note: GuardedLeaf stores guardIndex (int) not the Expr directly, to keep the tree data-only. The guard expression is looked up by index from a guards array at eval time.

3. `MatchRow` record:
   - `Patterns: Map<AccessPath, Pattern>` — remaining pattern constraints
   - `GuardIndex: int option` — index into guards array (if when-guard present)
   - `ClauseIndex: int` — original clause index for body lookup

**AccessPath comparison:** AccessPath is used as Map key. F# structural comparison on discriminated unions works correctly for this (Root < Field, Field compared by parent then index). No custom IComparable needed.

**Functions to implement:**

1. `normalizePattern (path: AccessPath) (pat: Pattern) : (AccessPath * Pattern) list`
   - VarPat: return `[(path, pat)]` — binding, not a constraint to test
   - WildcardPat: return `[]` — no constraint
   - ConstructorPat(name, None, _): return `[(path, pat)]`
   - ConstructorPat(name, Some argPat, _): return `(path, pat)` plus recursive normalization of argPat at `Field(path, 0)`
   - TuplePat(pats, _): return `(path, pat)` plus recursive normalization of each pat[i] at `Field(path, i)`
   - ConsPat(head, tail, _): return `(path, pat)` plus head at `Field(path, 0)`, tail at `Field(path, 1)`
   - EmptyListPat: return `[(path, pat)]`
   - ConstPat: return `[(path, pat)]`
   - RecordPat(fields, _): return `(path, pat)` plus recursive normalization of each field pattern at `Field(path, i)` where i is the field's index in the sorted field list

2. `clauseFromMatchClause (index: int) (guardIndex: int option) (pat: Pattern) : MatchRow`
   - Normalizes the pattern from Root, builds the Patterns map, sets ClauseIndex and GuardIndex.

3. `extractBindings (row: MatchRow) : (string * AccessPath) list`
   - From remaining patterns in the row, collect VarPat entries as (name, path) pairs.
   - Must traverse ALL pattern entries in the row including sub-patterns that were not yet tested.

4. `allResolved (row: MatchRow) : bool`
   - Returns true if the row has no remaining constructor/constant constraints (only VarPat, WildcardPat, or empty).

5. `patternToConstructor (pat: Pattern) : (string * int) option`
   - Maps pattern to (constructorName, arity) for the splitting algorithm:
     - ConstructorPat(name, None, _) -> Some(name, 0)
     - ConstructorPat(name, Some _, _) -> Some(name, 1)
     - TuplePat(pats, _) -> Some("#tuple_" + string(List.length pats), List.length pats)
     - ConsPat _ -> Some("::", 2)
     - EmptyListPat _ -> Some("[]", 0)
     - ConstPat(IntConst n, _) -> Some("#int_" + string n, 0)
     - ConstPat(BoolConst b, _) -> Some("#bool_" + string(b).ToLower(), 0)
     - RecordPat(fields, _) -> Some("#record_" + string(List.length fields), List.length fields)
     - VarPat _ | WildcardPat _ -> None (wildcard, not a constructor test)

6. `expandMatchBranch (testVar: AccessPath) (ctor: string) (arity: int) (row: MatchRow) : MatchRow`
   - For Case (a): The row's pattern at testVar matches the constructor. Remove the pattern at testVar. For sub-patterns of the matched constructor, add them to the map at Field(testVar, i). The sub-patterns come from the original pattern:
     - ConstructorPat(_, Some argPat, _): single sub-pattern at Field(testVar, 0)
     - TuplePat(pats, _): each pat at Field(testVar, i)
     - ConsPat(h, t, _): h at Field(testVar, 0), t at Field(testVar, 1)
     - RecordPat(fields, _): each field pattern at Field(testVar, i) (sorted order)
   - For Case (c) (wildcard/var at testVar): Remove the pattern at testVar. Add no sub-pattern constraints (wildcards expand to nothing). But if there was a VarPat, we need to keep the binding: add it to a separate bindings collector or keep the VarPat entry for later extraction.

   IMPORTANT: When handling VarPat in case (c), the variable must bind to the ORIGINAL path (testVar), not be discarded. Keep VarPat entries in the row for binding extraction later, but mark them as non-constraining for further compilation.

7. `splitClauses (testVar: AccessPath) (ctor: string) (arity: int) (clauses: MatchRow list) : MatchRow list * MatchRow list`
   - For each clause, check the pattern at testVar:
     - Case (a): patternToConstructor returns Some(name, _) and name = ctor -> matchBranch only (via expandMatchBranch)
     - Case (b): patternToConstructor returns Some(name, _) and name <> ctor -> elseBranch only (row unchanged)
     - Case (c): patternToConstructor returns None (VarPat/WildcardPat) OR no pattern at testVar -> BOTH branches (expand for match, keep as-is for else)
   - Return (matchBranch, elseBranch)

8. `selectTest (clauses: MatchRow list) : AccessPath * string * int`
   - Collect all (path, constructorName, arity) candidates from all constructor patterns across all clauses.
   - For each unique path, count how many clauses have ANY constructor pattern at that path.
   - Select the (path, ctor, arity) with the highest count (maximizes constructor-tested clauses, minimizes wildcard duplication).
   - Tie-breaking: prefer paths closer to Root (shorter path).

9. `compile (clauses: MatchRow list) : DecisionTree`
   - Base cases:
     - Empty clauses -> Fail
     - First clause is allResolved:
       - No guard -> Leaf(extractBindings first, first.ClauseIndex)
       - Has guard -> GuardedLeaf(extractBindings first, first.GuardIndex.Value, first.ClauseIndex, compile (List.tail clauses))
   - Recursive case:
     - Call selectTest to pick (testVar, ctor, arity)
     - Call splitClauses to get (matchClauses, elseClauses)
     - Return Switch(testVar, ctor, arity, compile matchClauses, compile elseClauses)

10. `accessValue (root: Value) (path: AccessPath) : Value`
    - Root -> root
    - Field(parent, idx) -> match accessValue root parent with:
      - DataValue(_, Some arg) when idx = 0 -> arg
      - TupleValue vals -> vals.[idx]
      - ListValue(h :: _) when idx = 0 -> h
      - ListValue(_ :: t) when idx = 1 -> ListValue t
      - RecordValue(_, fields) -> access by sorted field name index: let sortedFields = fields |> Map.toList |> List.sortBy fst in !(snd sortedFields.[idx])
      - _ -> failwithf "Invalid access path: index %d" idx

11. `matchesConstructor (value: Value) (ctor: string) : bool`
    - DataValue(name, _), c -> name = c
    - ListValue [], "[]" -> true
    - ListValue(_ :: _), "::" -> true
    - IntValue n, c when c.StartsWith("#int_") -> string n = c.Substring(5)
    - BoolValue b, c when c.StartsWith("#bool_") -> (string b).ToLower() = c.Substring(6)
    - TupleValue _, c when c.StartsWith("#tuple_") -> true
    - RecordValue _, c when c.StartsWith("#record_") -> true
    - _ -> false

12. `evalDecisionTree (evalFn: Env -> Expr -> Value) (env: Env) (scrutinee: Value) (bodies: Expr array) (guards: Expr array) (tree: DecisionTree) : Value`
    - Takes evalFn as a parameter to avoid circular dependency with Eval.fs.
    - Leaf(bindings, bodyIdx): fold bindings into env using accessValue, then evalFn bindEnv bodies.[bodyIdx]
    - Fail: failwith "Match failure: no pattern matched"
    - Switch(testVar, ctor, arity, onMatch, onElse): let testValue = accessValue scrutinee testVar; if matchesConstructor testValue ctor then recurse onMatch else recurse onElse
    - GuardedLeaf(bindings, guardIdx, bodyIdx, fallthrough): fold bindings into env, eval guard; if BoolValue true then eval body, else recurse fallthrough

13. `compileMatch (clauses: MatchClause list) : DecisionTree * Expr array * Expr array`
    - Entry point: converts MatchClause list to (DecisionTree, bodies array, guards array).
    - For each clause (pattern, guard, body):
      - Assign clauseIndex = i
      - If guard is Some g, assign guardIndex and add g to guards list
      - Call clauseFromMatchClause to create MatchRow
    - Call compile on the MatchRow list
    - Return (tree, Array.ofList bodies, Array.ofList guards)

**Important design decisions:**
- Do NOT import or depend on Eval.fs. Use evalFn parameter for evaluation to avoid circular module dependency.
- The module only opens Ast (for Pattern, Expr, MatchClause, Value, Env types) and Type (for RecordEnv if needed).
- AccessPath uses F# structural equality/comparison which works correctly as Map key.
- Record field access in AccessPath uses index based on alphabetically sorted field names (consistent with RecordPat normalization).
