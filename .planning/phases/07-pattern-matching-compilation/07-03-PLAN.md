**Test categories (minimum 15 tests):**

1. **ADT Constructor Patterns (PMATCH-01)** -- 3 tests
   - Simple Option match: `type Option = None | Some of int` with `Some 42`, expect 42
   - Multi-constructor: `type Color = Red | Green | Blue` with `Green`, expect 2
   - Constructor with nested data: `type Tree = Leaf | Node of Tree * int * Tree`, match on root value

2. **Nested Patterns (PMATCH-01, PMATCH-02)** -- 2 tests
   - Deep nested constructor match: `Node(Node(_, x, _), y, _)` -> `x + y`
   - Nested wrapper types: `type Pair = MkPair of int * int; type Wrapper = Wrap of Pair`

3. **List Patterns (PMATCH-02)** -- 2 tests
   - Empty vs cons: `[] -> 0 | h :: _ -> h`
   - Nested cons: `[] -> 0 | _ :: [] -> 1 | a :: b :: _ -> a + b`

4. **Tuple Patterns (PMATCH-02)** -- 2 tests
   - Simple tuple: `(a, b, c) -> a + b + c`
   - Tuple with wildcards: `(x, _) -> x`

5. **Constant Patterns (PMATCH-02)** -- 2 tests
   - Integer constants with wildcard default: `1 -> 10 | 2 -> 20 | _ -> 0`
   - Boolean constants: `true -> 1 | false -> 0`

6. **Record Patterns (PMATCH-02)** -- 1 test
   - Simple record match: `type Point = { x: int; y: int }` with `{ x = a; y = b } -> a + b`

7. **When Guards (guards, fallthrough)** -- 2 tests
   - Guard with fallthrough: `Some n when n > 10 -> 1 | Some n -> n | None -> 0` with `Some 5`, expect 5
   - Multiple guard fallthrough: `x when x > 10 -> 100 | x when x > 5 -> 50 | x -> x` with 3, expect 3

8. **Wildcard/Variable Catch-all (PMATCH-02, clause splitting case c)** -- 1 test
   - Wildcard catches unmatched: `type AB = A | B | C` with `C`, `A -> 1 | B -> 2 | _ -> 3`, expect 3

9. **Heuristic/Correctness (PMATCH-03)** -- 1 test
   - Tuple of ADTs: `match (A, B) with (A, A) -> 1 | (A, B) -> 2 | (B, A) -> 3 | (B, B) -> 4`, expect 2

10. **No Redundant Tests (PMATCH-01)** -- 1 test (structural)
    - Use `MatchCompile.compileMatch` directly on a set of clauses, then verify the tree structure by walking it: confirm each path from root to leaf never tests the same TestVar against the same constructor twice. This is a unit test, not end-to-end.
