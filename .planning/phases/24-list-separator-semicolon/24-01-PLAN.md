<task type="auto">
  <name>Task 1: Update Parser.fsy — add SemiExprList rule, change list literal grammar</name>
  <files>src/LangThree/Parser.fsy</files>
  <action>
    In Parser.fsy, make these changes:

    1. Add a new grammar rule `SemiExprList` for semicolon-separated expression lists (for use inside list literals only). Place it near the existing `ExprList` rule (around line 279):

    ```
    // Semicolon-separated expression list for list literals [e1; e2; e3]
    SemiExprList:
        | Expr                            { [$1] }
        | Expr SEMICOLON SemiExprList     { $1 :: $3 }
    ```

    2. Change the list literal rule in `Atom` (around line 261) from:
    ```
    | LBRACKET Expr COMMA ExprList RBRACKET  { List($2 :: $4, ruleSpan parseState 1 5) }
    ```
    to:
    ```
    | LBRACKET Expr SEMICOLON SemiExprList RBRACKET  { List($2 :: $4, ruleSpan parseState 1 5) }
    ```

    3. DO NOT change:
    - `LPAREN Expr COMMA ExprList RPAREN` (tuple rule — keeps COMMA)
    - `ExprList` rule itself (still used for tuples)
    - Record syntax rules (already use SEMICOLON correctly)
    - `LBRACKET RBRACKET` (empty list — unchanged)
    - `LBRACKET Expr RBRACKET` (singleton list — unchanged)
    - Range syntax `LBRACKET Expr DOTDOT ...` rules (unchanged)

    The SEMICOLON token is already declared in Parser.fsy (line 50) and lexed in Lexer.fsl (line 135). No lexer changes needed.
  </action>
