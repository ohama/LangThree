  <name>Task 1: Apply MOD-05 recEnv scoping fix in TypeCheck.fs</name>
  <files>src/LangThree/TypeCheck.fs</files>
  <action>
    The current `ModuleDecl` arm in `typeCheckDecls` (around line 670) passes `rEnv` (the
    accumulated parent record env) into the inner `typeCheckDecls` call. This is correct.
    However, the sibling `ModuleDecl`s that come after this one in the same parent scope
    receive the accumulated `rEnv` from prior processing, which does NOT include this
    module's internal record types (they go into `exports.RecEnv` only). The issue is:
    within a module, `RecordExpr` resolution looks at all records currently in scope —
    if the parent `rEnv` was modified by a prior sibling module's `open`, both record
    types could be visible.

    The fix: ensure that when type-checking inside a `ModuleDecl`, the `rEnv` passed to
    inner declarations is exactly the parent's `rEnv` (not the accumulated outer env that
    may have been enriched by `open` of prior modules). This is already the case for the
    `moduleRecEnv` filtering logic — the exports only contain records NEW to this module.
    The parent env is not modified by `ModuleDecl` (returns same `rEnv` to caller).

    Confirm the current behavior: read the `ModuleDecl` arm carefully. The return value
    is `(env, cEnv, rEnv, Map.add name exports mods, warns @ innerWarns)` — `rEnv` is
    returned unchanged to the parent scope. This means sibling modules do NOT see each
    other's internal record types. This is already correct.

    If the current behavior is already correct, verify it empirically by running a test
    case (see Task 2). If `DuplicateFieldName` still occurs for two sibling modules
    defining records with the same field name, investigate further.

    If a fix IS needed, it will be in the `RecordExpr` synthesis path (Bidir.fs lines
    395-423): when resolving a record literal by field names, the `recEnv` lookup should
    prefer the innermost module scope. The cleanest fix without row-polymorphism:
    document the limitation ("when creating a record literal with ambiguous field names,
    qualify the type: `{ kind = x } : Lexer.Token`"). Add a type annotation fallback.

    Only modify TypeCheck.fs if the empirical test in Task 2 reveals an actual bug.
    If no bug, skip the source change and document the finding in the summary.
  </action>

    Add `evalFileModule` helper at the top of the test file:

    ```fsharp
    // Helper to evaluate a .fun file by path (for file import tests)
    let evalFileModule (filePath: string) : Ast.Value =
        let source = System.IO.File.ReadAllText filePath
        let lexbuf = FSharp.Text.Lexing.LexBuffer<_>.FromString source
        Lexer.setInitialPos lexbuf filePath  // IMPORTANT: use real path for resolveImportPath
        let filteredTokens =
            let rec collect () =
                let tok = Lexer.tokenize lexbuf
                if tok = Parser.EOF then [Parser.EOF]
                else tok :: collect ()
            let rawTokens = collect ()
            LangThree.IndentFilter.filter LangThree.IndentFilter.defaultConfig rawTokens |> Seq.toList
        let mutable index = 0
        let tokenizer _ =
            if index < filteredTokens.Length then
                let tok = filteredTokens.[index]
                index <- index + 1
                tok
            else Parser.EOF
        let lexbuf2 = FSharp.Text.Lexing.LexBuffer<_>.FromString source
        Lexer.setInitialPos lexbuf2 filePath
        let m = Parser.parseModule tokenizer lexbuf2
        match TypeCheck.typeCheckModule m with
        | Error diag -> failtest (sprintf "Type checking failed: %s" (Diagnostic.formatDiagnostic diag))
        | Ok (_warnings, recEnv, _modules, _typeEnv) ->
            let decls =
                match m with
                | Ast.Module(d, _) | Ast.NamedModule(_, d, _) | Ast.NamespacedModule(_, d, _) -> d
                | Ast.EmptyModule _ -> []
            let finalEnv, _ = Eval.evalModuleDecls recEnv Map.empty Eval.emptyEnv decls
            match decls |> List.rev |> List.tryPick (function Ast.LetDecl(_, body, _) -> Some body | _ -> None) with
            | Some body -> Eval.eval recEnv Map.empty finalEnv false body
            | None -> Ast.TupleValue []
    ```
