  <files>src/LangThree/Lexer.fsl, src/LangThree/Parser.fsy</files>
  <action>
In Parser.fsy:
1. Add token declarations: `%token EXCEPTION RAISE TRY WHEN`
2. Update MatchClauses rule to use 3-tuple with optional when guard:
   ```
   MatchClauses:
       | PIPE Pattern ARROW Expr                         { [($2, None, $4)] }
       | PIPE Pattern WHEN Expr ARROW Expr               { [($2, Some $4, $6)] }
       | PIPE Pattern ARROW Expr MatchClauses            { ($2, None, $4) :: $5 }
       | PIPE Pattern WHEN Expr ARROW Expr MatchClauses  { ($2, Some $4, $6) :: $7 }
   ```
3. Add exception declaration to Decls (following the singleton/continuation pattern like OpenDecl):
   ```
   | EXCEPTION IDENT
       { [ExceptionDecl($2, None, ruleSpan parseState 1 2)] }
   | EXCEPTION IDENT Decls
       { ExceptionDecl($2, None, ruleSpan parseState 1 2) :: $3 }
   | EXCEPTION IDENT OF TypeExpr
       { [ExceptionDecl($2, Some $4, ruleSpan parseState 1 4)] }
   | EXCEPTION IDENT OF TypeExpr Decls
       { ExceptionDecl($2, Some $4, ruleSpan parseState 1 4) :: $5 }
   ```
4. Add raise expression to Factor (between MINUS Factor and AppExpr):
   `| RAISE Atom  { Raise($2, ruleSpan parseState 1 2) }`
   Use Atom (not AppExpr) to avoid ambiguity -- `raise MyError "msg"` should be `raise (MyError "msg")`.
5. Add try-with expression to Expr:
   `| TRY Expr WITH MatchClauses  { TryWith($2, $4, ruleSpan parseState 1 4) }`

   `| TRY INDENT Expr DEDENT WITH MatchClauses  { TryWith($3, $6, ruleSpan parseState 1 6) }`
