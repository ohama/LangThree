<task type="auto">
  <name>Task 1: Exception runtime evaluation and when guards</name>
  <files>src/LangThree/Eval.fs</files>
  <action>
Exception constructors at runtime use `DataValue(name, valueOpt)` -- the same representation as ADT constructors. No special ExnValue type exists. This means existing `matchPattern` and `evalMatchClauses` work for exceptions without modification (beyond adding when guard support).

1. Define the .NET exception type near the top of Eval.fs (after opens, before eval):
   ```fsharp
   exception LangThreeException of Value
   ```

2. Replace the Raise stub in `eval` with:
   ```fsharp
   | Raise(arg, _) ->
       let exnVal = eval recEnv moduleEnv env arg
       raise (LangThreeException exnVal)
   ```

3. Replace the TryWith stub in `eval` with:
   ```fsharp
   | TryWith(body, handlers, _) ->
       try
           eval recEnv moduleEnv env body
       with
       | LangThreeException exnVal ->
           evalMatchClauses recEnv moduleEnv env exnVal handlers
   ```

4. Update `evalMatchClauses` to handle when guards. Currently it destructures `(pattern, _guard, resultExpr)` but ignores the guard (from Plan 01). Change to:
   ```fsharp
   and evalMatchClauses recEnv moduleEnv env scrutinee clauses =
       match clauses with
       | [] -> failwith "Match failure: no pattern matched"
       | (pattern, guard, resultExpr) :: rest ->
           match matchPattern pattern scrutinee with
           | Some bindings ->
               let extendedEnv = List.fold (fun e (n, v) -> Map.add n v e) env bindings
               match guard with
               | None ->
                   eval recEnv moduleEnv extendedEnv resultExpr
               | Some guardExpr ->
                   match eval recEnv moduleEnv extendedEnv guardExpr with
                   | BoolValue true -> eval recEnv moduleEnv extendedEnv resultExpr
                   | _ -> evalMatchClauses recEnv moduleEnv env scrutinee rest
           | None ->
               evalMatchClauses recEnv moduleEnv env scrutinee rest
   ```

5. In `evalModuleDecls` (the module-level declaration evaluator), add handling for `ExceptionDecl`:
   - For nullary (dataTypeOpt = None): `Map.add name (DataValue(name, None)) env`
   - For with-data (dataTypeOpt = Some _): `Map.add name (FunctionValue("__x", Constructor(name, Some(Var("__x", unknownSpan)), unknownSpan), Map.empty)) env`
   - Also register in moduleEnv.CtorEnv for qualified access.
  </action>
  <verify>Run `dotnet build src/LangThree/LangThree.fsproj && dotnet test tests/LangThree.Tests/LangThree.Tests.fsproj 2>&1 | tail -20`. All tests must pass.</verify>
  <done>raise throws LangThreeException with DataValue. try-with catches and pattern matches. when guards evaluate and skip clause if not BoolValue true. ExceptionDecl registers constructors in eval environment.</done>

  <name>Task 2: IndentFilter InTry context for multi-line try-with</name>
  <files>src/LangThree/IndentFilter.fs</files>
  <action>
Add `InTry` syntax context to IndentFilter, modeled identically to `InMatch`:

1. Add `| InTry of baseColumn: int` to SyntaxContext union.

2. Add `JustSawTry: bool` field to FilterState (default false in initialState).

3. In `processNewlineWithContext`:
   - After the JustSawMatch check, add similar logic for JustSawTry:
     ```fsharp
     let stateWithTryContext =
         if stateWithMatchContext.JustSawTry then
             { stateWithMatchContext with Context = InTry stateWithMatchContext.IndentStack.Head :: stateWithMatchContext.Context; JustSawTry = false }
         else
             stateWithMatchContext
     ```
   - Use stateWithTryContext in place of stateWithMatchContext for the rest of the function.
   - In the DEDENT context popping, add: `| InTry baseCol :: rest when newState.IndentStack.Head <= baseCol -> popContexts rest`
   - In the pipe alignment check, also match `InTry baseCol :: _` (try-with handlers also use pipes at base column):
     ```fsharp
     | InTry baseCol :: _ when nextToken = Some Parser.PIPE ->
         if col <> baseCol then
             raise (IndentationError(...))
         (stateAfterIndent, [])
     ```

4. In `updateContextOnDedent`:
   - Add: `| InTry baseCol :: rest -> if state.IndentStack.Head <= baseCol then { state with Context = rest } else state`

5. In the `filter` function main loop:
   - Add a case for `Parser.TRY`: `| Parser.TRY -> state <- { state with JustSawTry = true; PrevToken = Some token }; yield token`
   - This mirrors the existing `Parser.MATCH` case.
  </action>
