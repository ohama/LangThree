**Bidir.fs** — Add the `ForInExpr` synth case immediately after the `ForExpr` case (after line 194). The loop variable must be bound as an immutable Scheme (NOT added to mutableVars) so that `x <- v` inside the body triggers E0320:

```fsharp
// === ForInExpr (Phase 51 - for-in collection loop) ===
| ForInExpr (var, collExpr, body, span) ->
    let s1, collTy = synth ctorEnv recEnv ctx env collExpr
    // Unify collection type with TList(elemTy) first; if that fails, try TArray(elemTy)
    let elemTv = freshTVar ()
    let s2 =
        try unifyWithContext ctx [] span (apply s1 collTy) (TList elemTv)
        with _ ->
            let elemTv2 = freshTVar ()
            let s2b = unifyWithContext ctx [] span (apply s1 collTy) (TArray elemTv2)
            s2b
    // Re-derive elemTy after unification
    let s12 = compose s2 s1
    let collTyResolved = apply s12 collTy
    let elemTy =
        match collTyResolved with
        | TList t -> t
        | TArray t -> t
        | _ -> freshTVar ()
    let env2 = applyEnv s12 env
    // Bind loop variable as immutable — NOT in mutableVars (FORIN-03)
    let loopEnv = Map.add var (Scheme([], elemTy)) env2
    let s3, _bodyTy = synth ctorEnv recEnv ctx loopEnv body
    (compose s3 s12, TTuple [])  // for-in always returns unit
```

**IMPORTANT NOTE on unification approach**: The above two-step try/catch approach may not match the codebase style. Inspect how `TList` and `TArray` are unified elsewhere in Bidir.fs. A cleaner approach used in this codebase may be: introduce a fresh type variable `elemTv`, unify `collTy` with `TList elemTv` OR `TArray elemTv`, then extract the element type from the resolved collection type. Use whatever pattern avoids a try/catch if a better codebase pattern exists (e.g., unifying against a union or checking the resolved type first). The critical constraint: `var` must be bound as `Scheme([], elemTy)` and NOT added to mutableVars.

```fsharp
// Phase 51: for-in collection loop
| ForInExpr (var, collExpr, body, _) ->
    let collVal = eval recEnv moduleEnv env false collExpr
    let elements =
        match collVal with
        | ListValue xs -> xs
        | ArrayValue arr -> arr |> Array.toList
        | _ -> failwith "for-in: collection must be a list or array"

        let loopEnv = Map.add var elemVal env
        eval recEnv moduleEnv loopEnv false body |> ignore
    TupleValue []
```
