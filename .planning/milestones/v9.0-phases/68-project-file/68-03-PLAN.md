Test cases to cover:

1. **build all targets** -- Create funproj.toml with two `[[executable]]` entries, each pointing to a valid .fun file. Run `$LT build`. Expect "OK: target1" and "OK: target2" in stderr output (redirect 2>&1).

2. **build named target** -- Same setup. Run `$LT build target1`. Expect "OK: target1" but NOT "OK: target2".

3. **build with type error** -- Create funproj.toml with one executable whose .fun file has a type error (e.g., `let x : int = "hello"`). Run `$LT build`. Expect "Error in" in output and non-zero exit.

4. **build without funproj.toml** -- Run `$LT build` from a temp directory without funproj.toml. Expect "funproj.toml not found" error.

5. **build unknown target name** -- Run `$LT build nonexistent`. Expect "no executable target named" error.

For the temp directory pattern, use:
```
DIR=$(mktemp -d) && cat > $DIR/funproj.toml << 'TOML'
[project]
name = "test"
[[executable]]
name = "main"
main = "main.fun"
TOML
printf "let x : int = 42\n" > $DIR/main.fun && cd $DIR && $LT build 2>&1
```

Use `// --- Output:` with `CONTAINS:` matching for stderr output (the OK/Error lines go to stderr, but we redirect 2>&1).

IMPORTANT: The flt `CONTAINS:` directive checks if the line appears anywhere in the output. Use this for all assertions since build output includes variable content like warning counts.
