<objective>
Add sprintf (format‚Üístring), printfn (printf+newline), and % modulo operator.

sprintf reuses existing printf infrastructure (parsePrintfSpecifiers, applyPrintfArgs)
but returns StringValue instead of writing to stdout.

printfn is identical to printf but appends \n before returning unit.

% is a new lexer token for integer modulo.
</objective>

<tasks>

<task id="1" title="Add % modulo operator to lexer and evaluator">
**Lexer.fsl:** Add `%` as a new token. Add rule `| '%' { PERCENT }` BEFORE the catch-all
operator rule.

**Parser.fsy:** Declare `%token PERCENT`. Add grammar rule at multiplicative level
(same as `*` and `/`):
```
| Term PERCENT Factor { ... modulo AST ... }
```

Since there's no `Modulo` AST node, we can either:
- Add `Modulo of Expr * Expr * Span` to Expr DU
- Or desugar to a built-in function call

**Simplest approach:** Add `Modulo` to Expr DU in Ast.fs, add eval case in Eval.fs,
add type check case (int -> int -> int).

**Eval.fs:**
```fsharp
| Modulo (left, right, _) ->
    match eval recEnv moduleEnv env false left, eval recEnv moduleEnv env false right with
    | IntValue l, IntValue r -> IntValue (l % r)
    | _ -> failwith "Type error: % requires integer operands"
```

Update spanOf, collectMatches, rewriteModuleAccess for Modulo.

**TypeCheck/Bidir:** Add synth case ‚Äî both operands int, result int (same as Divide).

**Format.fs:** Add Modulo to formatAst.

Build and test: `10 % 3` ‚Üí `1`
</task>

<task id="2" title="Add printfn built-in function">
**Eval.fs initialBuiltinEnv:** Add `printfn` ‚Äî identical to `printf` but appends `\n` to output.

Copy the printf implementation pattern but modify the final action:
- printf: `stdout.Write(result); stdout.Flush(); TupleValue []`
- printfn: `stdout.Write(result); stdout.Write("\n"); stdout.Flush(); TupleValue []`

Or simpler: create a helper `makePrintfBuiltin` that takes a `bool appendNewline` parameter,
then use it for both printf and printfn.

**TypeCheck.fs initialTypeEnv:** Add `printfn` with same type as `printf`:
```fsharp
"printfn", Scheme([0], TArrow(TString, TVar 0))
```

Test: `printfn "hello %d" 42` ‚Üí prints "hello 42\n" and returns ()
</task>

<task id="3" title="Add sprintf built-in function">
**Eval.fs:** Add `sprintf` ‚Äî same format string parsing as printf, but instead of
writing to stdout, builds and returns a StringValue.

Approach: Create a parallel set of helpers (or parameterize existing ones):
- `applySprintfArgs`: same as `applyPrintfArgs` but collects the final string
- When all specifiers consumed: return `StringValue result` instead of `stdout.Write`

**TypeCheck.fs initialTypeEnv:** Add `sprintf`:
```fsharp
"sprintf", Scheme([0], TArrow(TString, TVar 0))
```

The return type is polymorphic because `sprintf "%d" 42` returns string,
but `sprintf "%d %s"` returns `int -> string` (curried).

Actually: the TYPE should be `string -> string` at base, but since it's curried
with variable arity (like printf), use `Scheme([0], TArrow(TString, TVar 0))`.

Test: `sprintf "%d + %d = %d" 1 2 3` ‚Üí `"1 + 2 = 3"`
</task>
