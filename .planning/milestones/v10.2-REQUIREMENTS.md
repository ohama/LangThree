# Requirements: v10.2 Module Error Quality

**Defined:** 2026-04-01
**Core Value:** 모듈 관련 에러가 사용자에게 정확한 위치와 명확한 메시지를 제공

## v10.2 Requirements

### Error Messages

- [ ] **MERR-01**: Undefined module qualified access (`M.x` where M undefined) produces clear "Unresolved module: M" instead of garbled "non-record type 'w"
- [ ] **MERR-02**: Wrong nested module name (`A.C.x` where C doesn't exist) produces "module A has no submodule C"
- [ ] **MERR-03**: File import missing (`open "missing.fun"`) shows source location of the `open` statement
- [ ] **MERR-04**: Circular file import shows source location of the `open` statement
- [ ] **MERR-05**: `open NonexistentModule` says "Unresolved module" not "Forward reference" when module never defined
- [ ] **MERR-06**: Record type from module without open shows actual type name, not `?`
- [ ] **MERR-07**: Error spans include filename (no `:0:0` without path)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Runtime module errors → structured diagnostics | Low impact, requires Eval.fs refactor |
| Module name collision warnings | Rare scenario, complex to implement |
| Module shadowing warnings | Intentional behavior in F# convention |
