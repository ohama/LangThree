# Requirements: LangThree v7.1

**Defined:** 2026-03-29
**Core Value:** OOP 스타일 dot dispatch를 제거하고 순수 함수형 module function API로 통일

## v7.1 Requirements

### Builtin 함수 추가

- [ ] **BLT-01**: `string_endswith s suffix` — 문자열 끝 검사 builtin
- [ ] **BLT-02**: `string_startswith s prefix` — 문자열 시작 검사 builtin
- [ ] **BLT-03**: `string_trim s` — 문자열 공백 제거 builtin
- [ ] **BLT-04**: `hashtable_trygetvalue ht key` — (bool * value) 튜플 반환 builtin
- [ ] **BLT-05**: `hashtable_count ht` — 해시테이블 요소 수 반환 builtin

### Prelude 모듈 확장

- [ ] **MOD-01**: String 모듈에 endsWith, startsWith, trim, length, contains 함수 추가
- [ ] **MOD-02**: Hashtable 모듈에 tryGetValue, count 함수 추가
- [ ] **MOD-03**: StringBuilder 모듈에서 append→add 이름 변경 (List.append 충돌 해결)

### 구조 변경

- [ ] **STR-01**: Hashtable for-in이 (key, value) 튜플을 생성하도록 변경 (KeyValuePair 레코드 제거)
- [ ] **STR-02**: Eval.fs에서 value-type FieldAccess dispatch 코드 제거
- [ ] **STR-03**: Bidir.fs에서 value-type FieldAccess 타입 체킹 코드 제거

### 테스트 전환

- [ ] **TST-01**: dot notation flt 테스트를 module function 방식으로 전환 + 전체 스위트 PASS

## v7.2 Requirements (deferred)

### FunLexYacc 소스 수정

- **FLY-01**: FunLexYacc 소스코드를 module function API로 전환 (별도 프로젝트)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Dot notation 유지 (하위 호환) | v7.1의 목표가 제거이므로 의도적 breaking change |
| FunLexYacc 소스 수정 | 별도 마일스톤(v7.2)에서 진행 |
| Record field access 제거 | record.field 은 함수형 패턴이므로 유지 |
| Module qualified access 제거 | Module.func 는 함수형 패턴이므로 유지 |
