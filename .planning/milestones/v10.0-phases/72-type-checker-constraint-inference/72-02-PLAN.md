**Infer.fs** -- Modify `instantiate` to emit constraints:

Current implementation ignores the constraint list in Scheme:
```fsharp
let instantiate (Scheme (vars, _, ty)): Type =
```

New implementation -- when instantiating a scheme with constraints, apply the same fresh-var substitution to constraint TypeArgs and push them onto Bidir.pendingConstraints:

```fsharp
let instantiate (Scheme (vars, constraints, ty)): Type =
    match vars with
    | [] ->
        // Monomorphic -- if there are constraints, they should have been resolved already
        // but emit them just in case (for explicit constrained annotations)
        if not (List.isEmpty constraints) then
            Bidir.pendingConstraints <- constraints @ Bidir.pendingConstraints
        ty
    | _ ->
        let freshVars = List.map (fun _ -> freshVar()) vars
        let subst = List.zip vars freshVars |> Map.ofList
        // Apply substitution to constraints and push onto pending
        if not (List.isEmpty constraints) then
            let freshConstraints =
                constraints |> List.map (fun c ->
                    { c with TypeArg = Type.apply subst c.TypeArg })
            Bidir.pendingConstraints <- freshConstraints @ Bidir.pendingConstraints
        Type.apply subst ty
```

**Infer.fs** -- Modify `generalize` to capture pending constraints:

Current implementation creates Scheme with empty constraints:
```fsharp
let generalize (env: TypeEnv) (ty: Type): Scheme =
    let envFree = freeVarsEnv env
    let tyFree = freeVars ty
    let vars = Set.difference tyFree envFree |> Set.toList
    Scheme (vars, [], ty)
```

New implementation -- drain pendingConstraints, partition into those that mention generalized vars (keep in Scheme) vs those that are fully resolved (attempt resolution now):

```fsharp
let generalize (env: TypeEnv) (ty: Type): Scheme =
    let envFree = freeVarsEnv env
    let tyFree = freeVars ty
    let vars = Set.difference tyFree envFree |> Set.toList
    let varsSet = Set.ofList vars
    // Drain pending constraints
    let pending = Bidir.pendingConstraints
    Bidir.pendingConstraints <- []
    // Partition: constraints mentioning generalized vars stay in Scheme;
    // constraints on concrete types (no generalized vars) must be resolved now
    let deferred, concrete =
        pending |> List.partition (fun c ->
            let constraintFree = Type.freeVars c.TypeArg
            not (Set.isEmpty (Set.intersect constraintFree varsSet)))
    // Resolve concrete constraints against InstanceEnv
    concrete |> List.iter (fun c ->
        let instEnv = TypeCheck.currentInstEnv
        let instances = Map.tryFind c.ClassName instEnv |> Option.defaultValue []
        let resolved = instances |> List.exists (fun ii -> ii.InstanceType = c.TypeArg)
        if not resolved then
            raise (Diagnostic.TypeException {
                Kind = Diagnostic.NoInstance(c.ClassName, c.TypeArg)
                Span = Ast.unknownSpan
                Term = None
                ContextStack = []
                Trace = []
            })
    )
    // Remove duplicate deferred constraints
    let uniqueDeferred = deferred |> List.distinctBy (fun c -> (c.ClassName, c.TypeArg))
    Scheme (vars, uniqueDeferred, ty)
```

**Bidir.fs Var case** -- Currently (line 51-61) the Var case just calls `instantiate scheme`. After Plan 02 Task 1, `instantiate` already emits constraints. But we need to apply any substitution from constraint resolution to the instantiated type.

The current code is:
```fsharp
| Var (name, span) ->
    match Map.tryFind name env with
    | Some scheme -> (empty, instantiate scheme)
```

This should work as-is because:
1. `instantiate` pushes constraints onto `pendingConstraints`
2. Those constraints are resolved/deferred at the next `generalize` call
3. The substitution from constraint resolution is applied at `generalize` time

**IMPORTANT edge case:** When `show_twice 42` is called, the application unfolds as:
1. `synth` of `show_twice` -> `instantiate` the constrained scheme -> emits constraint `Show ?1` for fresh var ?1 -> returns `?1 -> string`
2. `synth` of `42` -> `TInt`
3. Application unification: `?1` unifies with `int` -> substitution `{?1 -> int}`
4. This substitution MUST be applied to the pending constraint `Show ?1` to get `Show int`

The problem: pendingConstraints stores `Show (TVar ?1)` but when `?1` is unified to `int`, the substitution is not applied to pendingConstraints.

**Fix in `generalize`:** Before partitioning, apply the current accumulated substitution to all pending constraints. But `generalize` doesn't receive a substitution...

**Alternative approach:** Apply substitutions to constraints at resolution time instead. In `generalize`, when resolving concrete constraints, the constraint's TypeArg should already be resolved because `generalize` receives `ty` which has substitutions applied. The constraint TypeArgs contain TVar IDs that may have been unified. We need to apply the environment's substitution to constraint TypeArgs.

**Better fix:** In `generalize`, apply the substitution (derived from comparing env's free vars with ty's free vars) to constraint TypeArgs. Actually, the correct approach: when we call `generalize env ty`, the `ty` has substitutions applied already (caller does `apply s ty`), but the pendingConstraints may have stale TVar refs.

**Correct fix in Infer.fs `generalize`:** Accept an optional substitution parameter, or better: have the caller apply the substitution to pendingConstraints before calling generalize. The cleanest approach:

Add a helper function in Bidir.fs:

```fsharp
/// Apply a substitution to all pending constraints
let applySubstToConstraints (s: Subst) =
    pendingConstraints <- pendingConstraints |> List.map (fun c ->
        { c with TypeArg = Type.apply s c.TypeArg })
```

Then in the LetDecl arm of TypeCheck.fs, after computing the substitution `s` and `ty'`:

```fsharp
| LetDecl(name, body, _) ->
    let refsInBody = collectModuleRefs mods body
    let rewrittenBody = rewriteModuleAccess mods body
    let (envForSynth, ctorEnvForSynth, recEnvForSynth) =
        if Set.isEmpty refsInBody then (env, cEnv, rEnv)
        else mergeModuleExportsForTypeCheck mods refsInBody env cEnv rEnv
    let s, ty = Bidir.synth ctorEnvForSynth recEnvForSynth [] envForSynth rewrittenBody
    let ty' = apply s ty
    // Apply substitution to pending constraints so TVar refs are resolved
    Bidir.applySubstToConstraints s
    let scheme = generalize (applyEnv s env) ty'
    let env' = Map.add name scheme env
    let matchWarnings = checkMatchWarnings cEnv body
    (env', cEnv, rEnv, clsEnv, iEnv, mods, warns @ matchWarnings)
```

Do the same for LetMutDecl, LetPatDecl, and LetRecDecl arms -- after computing the final substitution, call `Bidir.applySubstToConstraints` before calling `generalize`.
