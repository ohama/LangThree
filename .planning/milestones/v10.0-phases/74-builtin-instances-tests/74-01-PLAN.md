**Eq typeclass and instances:**

  <name>Task 2: Fix Prelude.fs to elaborate typeclass declarations before eval</name>
  <files>src/LangThree/Prelude.fs</files>
  <action>
In `Prelude.fs`, the `loadPrelude` function at line ~297-299 currently does:

```fsharp
let decls = getDecls m
let evalEnv = Map.fold (fun acc k v -> Map.add k v acc) result.Env Eval.initialBuiltinEnv
let (finalEnv, fileModuleEnv) = Eval.evalModuleDecls mergedRecEnv result.ModuleValueEnv evalEnv decls
```

This passes raw `decls` to `evalModuleDecls`. But `InstanceDecl` nodes are NOT handled by Eval — they need to be elaborated into `LetDecl` nodes first (just like Program.fs does at lines 191 and 410).

**Fix:** Add `elaborateTypeclasses` call before `evalModuleDecls`:

```fsharp
let decls = getDecls m
let elaboratedDecls = Elaborate.elaborateTypeclasses decls
let evalEnv = Map.fold (fun acc k v -> Map.add k v acc) result.Env Eval.initialBuiltinEnv
let (finalEnv, fileModuleEnv) = Eval.evalModuleDecls mergedRecEnv result.ModuleValueEnv evalEnv elaboratedDecls
```

Also add `open Elaborate` at the top of the file (or use the fully qualified `Elaborate.elaborateTypeclasses`).

**Also fix `loadAndEvalFileImpl`** (line ~163-164) which has the same problem — file imports with typeclass instances won't elaborate either:

```fsharp
let decls = getDecls m
let (fileEnv, fileModEnv) = Eval.evalModuleDecls recEnv modEnv env decls
```

Change to:
```fsharp
let decls = getDecls m
let elaboratedDecls = Elaborate.elaborateTypeclasses decls
let (fileEnv, fileModEnv) = Eval.evalModuleDecls recEnv modEnv env elaboratedDecls
```
