# Roadmap: LangThree v7.1 Remove Dot Notation

OOP 스타일 dot dispatch를 제거하고 순수 함수형 module function API로 통일한다. 먼저 누락된 builtin 함수와 Prelude 모듈 함수를 추가하여 module function 대안을 확보하고, Hashtable for-in을 tuple 방식으로 전환한 뒤, 기존 테스트를 module function 방식으로 전환하고, 마지막으로 Eval.fs와 Bidir.fs에서 value-type FieldAccess dispatch 코드를 제거한다.

## Phases

### Phase 60: Builtins & Prelude Modules

**Goal:** 사용자가 module function으로 모든 string/hashtable/stringbuilder 연산을 수행할 수 있다 (dot notation 제거 전 대안 확보)

**Dependencies:** None (builds on existing builtins and Prelude infrastructure)

**Requirements:** BLT-01, BLT-02, BLT-03, BLT-04, BLT-05, MOD-01, MOD-02, MOD-03

**Success Criteria:**
1. `String.endsWith "hello.txt" ".txt"` returns `true`, `String.startsWith "hello" "he"` returns `true`, `String.trim " hi "` returns `"hi"`
2. `String.length "abc"` returns `3` and `String.contains "hello" "ell"` returns `true` (existing builtins exposed via module)
3. `Hashtable.tryGetValue ht key` returns `(true, value)` or `(false, ...)` tuple, `Hashtable.count ht` returns entry count
4. `StringBuilder.add sb "text"` appends text (renamed from append to avoid List.append conflict)
5. All new builtins have type schemes in TypeCheck.fs and flt tests pass

**Plans:** 2 plans

Plans:
- [x] 60-01-PLAN.md -- Add 5 builtin functions to Eval.fs and type schemes to TypeCheck.fs
- [x] 60-02-PLAN.md -- Update Prelude .fun modules and create flt integration tests

---

### Phase 61: Hashtable Tuple & Test Conversion

**Goal:** Hashtable for-in이 tuple을 생성하고, 모든 flt 테스트가 module function 방식으로 동작한다

**Dependencies:** Phase 60 (module function alternatives must exist before tests can be converted)

**Requirements:** STR-01, TST-01

**Success Criteria:**
1. `for (k, v) in ht do ...` destructures key-value tuples directly (KeyValuePair record no longer used)
2. 기존 dot notation flt 테스트가 module function 호출 방식으로 전환되어 전체 스위트 PASS
3. `for kv in ht do kv.Key` 등 KeyValuePair 기반 코드가 더 이상 존재하지 않음

**Plans:** 2 plans

Plans:
- [ ] 61-01-PLAN.md -- AST + Parser + Bidir + Eval changes for tuple-based ForInExpr
- [ ] 61-02-PLAN.md -- Fix stale flt paths and convert dot-notation tests to module functions

---

### Phase 62: Remove Dot Dispatch

**Goal:** Eval.fs와 Bidir.fs에서 value-type FieldAccess dispatch 코드가 완전히 제거되어 순수 함수형 API만 남는다

**Dependencies:** Phase 61 (all tests must pass with module functions before removing dispatch code)

**Requirements:** STR-02, STR-03

**Success Criteria:**
1. Eval.fs에서 StringValue, ArrayValue, StringBuilderValue, HashSetValue, QueueValue, MutableListValue, HashtableValue, TData("KeyValuePair",...) FieldAccess 분기가 제거됨
2. Bidir.fs에서 동일한 value-type FieldAccess 타입 체킹 분기가 제거됨
3. Record field access (`record.field`)와 module qualified access (`Module.func`)는 정상 동작
4. 전체 flt 테스트 스위트 PASS (dot notation 코드 없이)
