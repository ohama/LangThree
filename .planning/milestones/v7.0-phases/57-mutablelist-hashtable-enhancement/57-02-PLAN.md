**Eval.fs change** — Add HashtableValue FieldAccess dispatch:

In the `FieldAccess` arm, after the `| MutableListValue ml ->` block from Plan 01 (and BEFORE `| RecordValue (_, fields) ->`), add:

```fsharp
// Phase 57: HashtableValue method dispatch
| HashtableValue ht ->
    match fieldName with
    | "TryGetValue" ->
        BuiltinValue (fun keyArg ->
            match ht.TryGetValue(keyArg) with
            | true, v  -> TupleValue [BoolValue true;  v]
            | false, _ -> TupleValue [BoolValue false; TupleValue []])
    | "Count" -> IntValue ht.Count
    | "Keys"  -> ListValue (ht.Keys |> Seq.toList)
    | _ -> failwithf "Hashtable has no property or method '%s'" fieldName
```

CRITICAL: `.TryGetValue` returns a 2-tuple in BOTH cases — `TupleValue [BoolValue true; v]` when found, `TupleValue [BoolValue false; TupleValue []]` when missing. Do NOT return a 1-tuple on the missing case.

CRITICAL: `.Keys` returns `ListValue (ht.Keys |> Seq.toList)` — wrapping in ListValue, not a raw seq.

**Bidir.fs change** — Add THashtable FieldAccess type rules:

In the `FieldAccess` arm, after `| TData("MutableList", []) ->` block from Plan 01 (and BEFORE `| TData (typeName, typeArgs) ->`), add:

```fsharp
// Phase 57: Hashtable field access types (THashtable is the type for Hashtable.create())
| THashtable (keyTy, valTy) ->
    match fieldName with
    | "TryGetValue" ->
        (s1, TArrow(keyTy, TTuple [TBool; valTy]))
    | "Count" -> (s1, TInt)
    | "Keys"  -> (s1, TList keyTy)
    | _ ->
        raise (TypeException { Kind = FieldAccessOnNonRecord resolvedTy; Span = span; Term = Some expr; ContextStack = ctx; Trace = [] })
```

CRITICAL: The match arm is `| THashtable (keyTy, valTy) ->` NOT `| TData("Hashtable", []) ->`. HashtableValue has type `THashtable(k,v)` in the type system — a dedicated parameterized type constructor, not TData. Using TData will NEVER match and ht.TryGetValue will get a type error.
