Update `lexAndFilter` to capture positions from lexbuf during lexing and return `PositionedToken list`:

```fsharp
let lexAndFilter (input: string) (filename: string) : PositionedToken list =
    let lexbuf = LexBuffer<char>.FromString input
    Lexer.setInitialPos lexbuf filename
    let rec collect () =
        let startPos = lexbuf.StartPos
        let tok = Lexer.tokenize lexbuf
        let endPos = lexbuf.EndPos
        if tok = Parser.EOF then
            [{ Token = Parser.EOF; StartPos = startPos; EndPos = endPos }]
        else
            { Token = tok; StartPos = startPos; EndPos = endPos } :: collect ()
    let rawTokens = collect ()
    filterPositioned defaultConfig rawTokens
```

IMPORTANT: Capture `lexbuf.StartPos` BEFORE calling `Lexer.tokenize` and `lexbuf.EndPos` AFTER. This is how fslex works -- tokenize advances the lexbuf positions.

Update `parseModuleFromString` to set lexbuf positions from each PositionedToken:

```fsharp
let parseModuleFromString (input: string) (filename: string) : Module =
    let filteredTokens = lexAndFilter input filename
    let lexbuf = LexBuffer<char>.FromString input
    Lexer.setInitialPos lexbuf filename
    let mutable index = 0
    let tokenizer (lb: LexBuffer<_>) =
        if index < filteredTokens.Length then
            let pt = filteredTokens.[index]
            index <- index + 1
            lb.StartPos <- pt.StartPos
            lb.EndPos <- pt.EndPos
            pt.Token
        else
            Parser.EOF
    Parser.parseModule tokenizer lexbuf
```

The critical fix is `lb.StartPos <- pt.StartPos` and `lb.EndPos <- pt.EndPos` -- this is what was missing and caused all Spans to be zeroed.

Also check if there are any other callers that depend on `lexAndFilter` returning `Parser.token list`. If `lex` or `formatTokens` uses `lexAndFilter`, those need updating. If they use a separate `lex` function, no changes needed there.

Note: The `open LangThree.IndentFilter` at the top of Program.fs already imports the module, so `PositionedToken` and `filterPositioned` will be available.
