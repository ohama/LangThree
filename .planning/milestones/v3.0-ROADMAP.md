## Milestone Summary

**Key Decisions:**
- BuiltinValue DU for native F# functions — ArrayValue and HashtableValue follow same pattern
- ArrayValue uses Value array (no outer ref) — in-place element mutation via arr.[i] <- v
- Array equality uses ReferenceEquals — two distinct arrays are never equal by value
- HashtableValue uses Dictionary<Value, Value> as backing store; reference equality
- OOB errors in builtins use LangThreeException (catchable by language try-with)
- callValueRef forward reference pattern: builtins that invoke user closures use a mutable ref wired after eval is defined

**Issues Resolved:**
- TypeCheck.fs SubModules bug: ModuleDecl must filter outer mods from innerMods before assigning SubModules
- Multi-arg lambda `fun a b ->` is a parse error; curried form `fun a -> fun b -> ...` required

**Issues Deferred:**
- None

**Technical Debt Incurred:**
