Chapter structure and content:

**Title:** `# 명령형 에르고노믹스 (Imperative Ergonomics)`

**Intro paragraph:** v5.0에서 LangThree는 명령형 스타일 코드를 더 자연스럽게 작성할 수 있는 네 가지 문법 기능을 추가했습니다. 이 장에서는 표현식 시퀀싱, 루프, 인덱싱 문법, else 없는 if 표현식을 살펴봅니다.

**Section 1: 표현식 시퀀싱 (Expression Sequencing)**

- Concept: `e1; e2` evaluates both, returns value of e2. Replaces verbose `let _ = e1 in e2`.
- Example 1 (basic two-expression sequence):
  ```
  $ cat seq_basic.l3
  let _ = println "hello"; println "world"

  $ langthree seq_basic.l3
  hello
  world
  ()
  ```
- Example 2 (chained sequencing with mutable variable):
  ```
  $ cat seq_chain.l3
  let result = let mut x = 0 in x <- 1; x <- x + 1; x <- x + 1; x

  $ langthree seq_chain.l3
  3
  ```
- Example 3 (sequencing inside indented block):
  ```
  $ cat seq_block.l3
  let f () =
      println "a"; println "b"; println "c"
  let result = f ()

  $ langthree seq_block.l3
  a
  b
  c
  ()
  ```
- Explanation: `;`은 오른쪽 결합(right-associative)이므로 `e1; e2; e3`는 `e1; (e2; e3)`로 파싱됩니다. 최종 반환값은 마지막 표현식의 값입니다.

**Section 2: 루프 (Loops)**

Sub-section 2.1: while 루프
- Concept: `while cond do body` — condition checked before each iteration, returns unit.
- Example 1 (basic while):
  ```
  $ cat while_basic.l3
  let mut i = 0
  let _ = while i < 3 do i <- i + 1
  let _ = println (to_string i)

  $ langthree while_basic.l3
  3
  ()
  ```
- Example 2 (while with multi-statement body using `;`):
  ```
  $ cat while_body.l3
  let mut count = 0
  let mut sum = 0
  let _ =
      while count < 4 do
          sum <- sum + count; count <- count + 1
  let _ = println (to_string sum)

  $ langthree while_body.l3
  6
  ()
  ```
- Explain: 루프 본체에서 여러 문장을 실행하려면 `;`으로 연결합니다. 들여쓰기 블록 안에서 `;`을 사용하면 됩니다.

Sub-section 2.2: for 루프
- Concept: `for i = start to end do body` (ascending) and `for i = start downto end do body` (descending). Loop variable `i` is immutable.
- Example 1 (ascending for loop):
  ```
  $ cat for_asc.l3
  let mut total = 0
  let _ =
      for i = 0 to 3 do
          total <- total + i
  let _ = println (to_string total)

  $ langthree for_asc.l3
  6
  ()
  ```
- Example 2 (descending for loop):
  ```
  $ cat for_desc.l3
  let mut total = 0
  let _ =
      for i = 3 downto 0 do
          total <- total + i
  let _ = println (to_string total)

  $ langthree for_desc.l3
  6
  ()
  ```
- Explain: `to`는 오름차순(start ≤ end), `downto`는 내림차순(start ≥ end). start > end인 경우(`to`) 또는 start < end인 경우(`downto`) 루프 본체는 한 번도 실행되지 않고 unit을 반환합니다.

Sub-section 2.3: 루프 변수의 불변성
- Explain: for 루프 변수(`i`)는 불변입니다. 루프 본체 안에서 `i <- 42`를 시도하면 E0320 에러가 발생합니다:
  ```
  $ cat for_err.l3
  let _ =
      for i = 0 to 9 do
          i <- 42

  $ langthree for_err.l3
  error[E0320]: Cannot assign to immutable variable 'i'. Use 'let mut' to declare mutable variables.
  ```
- Explain: 루프 카운터를 직접 수정할 필요가 있다면 별도의 `let mut` 변수를 사용하세요.

**Section 3: 인덱싱 문법 (Indexing Syntax)**

Sub-section 3.1: 배열 인덱싱
- Concept: `arr.[i]` reads, `arr.[i] <- v` writes. Replaces `Array.get`/`Array.set` for cleaner syntax.
- Example 1 (read):
  ```
  $ cat arr_index_read.l3
  let arr = array_create 3 0
  let _ = array_set arr 0 10
  let _ = array_set arr 1 20
  let _ = array_set arr 2 30
  let _ = println (to_string arr.[0])
  let _ = println (to_string arr.[1])
  let _ = println (to_string arr.[2])

  $ langthree arr_index_read.l3
  10
  20
  30
  ()
  ```
- Example 2 (write):
  ```
  $ cat arr_index_write.l3
  let arr = array_create 3 0
  let _ = arr.[0] <- 42
  let _ = arr.[1] <- 99
  let _ = println (to_string arr.[0])
  let _ = println (to_string arr.[1])

  $ langthree arr_index_write.l3
  42
  99
  ()
  ```

Sub-section 3.2: 해시테이블 인덱싱
- Concept: `ht.[key]` reads, `ht.[key] <- v` writes.
- Example 1 (read):
  ```
  $ cat ht_index_read.l3
  let ht = hashtable_create ()
  let _ = hashtable_set ht "x" 100
  let _ = hashtable_set ht "y" 200
  let _ = println (to_string ht.["x"])
  let _ = println (to_string ht.["y"])

  $ langthree ht_index_read.l3
  100
  200
  ()
  ```
- Example 2 (write):
  ```
  $ cat ht_index_write.l3
  let ht = hashtable_create ()
  let _ = ht.["name"] <- "Alice"
  let _ = ht.["score"] <- 95
  let _ = println ht.["name"]
  let _ = println (to_string ht.["score"])

  $ langthree ht_index_write.l3
  Alice
  95
  ()
  ```

Sub-section 3.3: 체이닝
- Concept: `matrix.[r].[c]` — IndexGet is left-associative, enabling chained indexing on nested arrays.
- Example (2D array):
  ```
  $ cat matrix.l3
  let row0 = array_create 2 0
  let row1 = array_create 2 0
  let _ = row0.[0] <- 1
  let _ = row0.[1] <- 2
  let _ = row1.[0] <- 3
  let _ = row1.[1] <- 4
  let matrix = array_create 2 row0
  let _ = matrix.[1] <- row1
  let _ = println (to_string matrix.[0].[0])
  let _ = println (to_string matrix.[1].[1])

  $ langthree matrix.l3
  1
  4
  ()
  ```

**Section 4: else 없는 if 표현식**

- Concept: `if cond then expr` (without else) is accepted when `then` branch returns unit. Desugars to `if cond then expr else ()`.
- Example 1 (unit branch — valid):
  ```
  $ cat if_then.l3
  let x = 5
  let _ = if x > 0 then println "positive"

  $ langthree if_then.l3
  positive
  ()
  ```
- Example 2 (non-unit branch — type error):
  ```
  $ cat if_then_err.l3
  let _ = if true then 42

  $ langthree if_then_err.l3
  error[E0301]: Type mismatch: expected int but got unit
  ```
- Explain: `if cond then expr`는 암묵적으로 `else ()`가 붙습니다. else 브랜치가 unit인데 then 브랜치가 int이면 타입 불일치 에러가 납니다. 값을 반환하는 조건문이라면 반드시 `if cond then expr1 else expr2` 형태로 작성하세요.
- Example 3 (practical usage — with mutable side effect):
  ```
  $ cat if_then_mut.l3
  let mut x = 0
  let _ = if true then x <- 42
  let result = x

  $ langthree if_then_mut.l3
  42
  ```

**Section 5: 루프와 시퀀싱 조합 — 실용 예제**

A practical example combining all features — count even numbers in an array using a for loop, indexing, sequencing, and if-then:

```
$ cat imperative_example.l3
let arr = array_create 5 0
let _ = arr.[0] <- 2
let _ = arr.[1] <- 3
let _ = arr.[2] <- 4
let _ = arr.[3] <- 7
let _ = arr.[4] <- 8
let mut even_count = 0
let _ =
    for i = 0 to 4 do
        if arr.[i] mod 2 = 0 then even_count <- even_count + 1
let result = even_count

$ langthree imperative_example.l3
3
```

**Section 6: 구문 요약 (Syntax Summary)**

Table:

| 구문 | 설명 |
|------|------|
| `e1; e2` | 순서대로 평가, e2의 값 반환 |
| `e1; e2; e3` | 오른쪽 결합 체이닝 |
| `while cond do body` | 조건이 거짓이 될 때까지 반복 |
| `for i = s to e do body` | i를 s부터 e까지 1씩 증가 |
| `for i = s downto e do body` | i를 s부터 e까지 1씩 감소 |
| `arr.[i]` | 배열 원소 읽기 (IndexGet) |
| `arr.[i] <- v` | 배열 원소 쓰기 (IndexSet) |
| `ht.[key]` | 해시테이블 값 읽기 |
| `ht.[key] <- v` | 해시테이블 값 쓰기 |
| `if cond then expr` | else 없는 if (then 브랜치는 unit이어야 함) |
