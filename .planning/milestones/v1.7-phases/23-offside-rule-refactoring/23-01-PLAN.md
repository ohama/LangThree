<objective>
Refactor IndentFilter to use F#-style CtxtLetDecl context stack with offside column
tracking, replacing the fragile LetSeqDepth counter approach.

Based on F# compiler LexFilter.fs research:
- LET token → push CtxtLetDecl(blockLet, offsideCol)
- blockLet = true when in expression block (let x = INDENT), false in module
- EQUALS → push nested SeqBlock for RHS
- When token reaches offside column of CtxtLetDecl → insert IN (= F#'s ODECLEND)
- Explicit IN → consume and pop CtxtLetDecl
</objective>

<task id="2" title="Implement CtxtLetDecl push/pop logic in filter loop">
The core logic change. In the main `filter` function:

**When LET token is encountered:**
```fsharp
| Parser.LET ->
    // Determine if this is a "block let" (implicit in allowed)
    // Block let = inside an expression indent block (not module, not top-level)
    let blockLet = match state.Context with
        | InLetDecl _ :: _ -> true    // nested let inside another let body
        | InModule :: _ -> false       // module body - declarations, no implicit in
        | TopLevel :: _ -> false       // top level - declarations
        | InMatch _ :: _ -> true       // match body is expression context
        | InTry _ :: _ -> true         // try body is expression context
        | _ ->
            // Check indent stack: if depth > 1, we're in an expression block
            state.IndentStack.Length > 1
    if blockLet then
        // Push CtxtLetDecl with offside at the LET token's column
        // The column comes from the current indent level
        let offsideCol = state.IndentStack.Head
        state <- { state with Context = InLetDecl(true, offsideCol) :: state.Context }
    state <- { state with PrevToken = Some token }
    yield token
```

**When MODULE token is encountered:**
```fsharp
| Parser.MODULE ->
    state <- { state with Context = InModule :: state.Context; PrevToken = Some token }
    yield token
```

Wait, the MODULE context should be pushed when entering the module body (after INDENT),
not when seeing MODULE. Let me think...

Actually, the simplest approach: track `InModule` via the existing INDENT mechanism.
When `MODULE IDENT EQUALS INDENT` happens, push InModule. When DEDENT, pop.

But we don't process INDENT in the filter — it's emitted by processNewlineWithContext.

**Alternative simpler approach:** Instead of tracking InModule as a context, just check
if the LET appears at the module indent level. If `IndentStack = [4, 0]` and LET is at
col 4, AND the previous context is from a module entry, it's module-level.

**Simplest correct approach:** Use the `PrevToken` chain. When we see:
- `MODULE IDENT EQUALS NEWLINE → INDENT` → the next LET is module-level
- `LET IDENT EQUALS NEWLINE → INDENT` → the next LET is block-level

Track this with a flag or by checking what caused the current INDENT.

**ACTUALLY — the cleanest way matching F#:**
Only push CtxtLetDecl when LET appears inside an existing CtxtLetDecl or after
certain INDENT patterns. At TopLevel and InModule, LET is a declaration (no implicit IN).

The rule: `blockLet = true` when the let appears in a position where it's an expression,
not a declaration. Expression positions:
- After EQUALS + INDENT (the body of another let)
- After ARROW + INDENT (lambda body)
- After IN (explicit)
- Inside match/try clause body

The flag `blockLet` is essentially: "is this LET inside an expression context?"
At top-level and inside module bodies, it's false.

**Implementation:** When processing INDENT tokens (emitted by processNewlineWithContext),
check what caused the indent:
- PrevToken = EQUALS and context is NOT InModule → expression block
- PrevToken = ARROW → expression block (lambda body)

When processing LET in the main loop:
- If we're in an expression block (tracked by checking IndentStack depth and
  whether the current INDENT was from an expression context), push CtxtLetDecl.

For this plan, let me use the approach of tracking expression blocks via context:

Add `InExprBlock of offsideCol: int` to SyntaxContext. Push when INDENT comes from
`let x = INDENT` or `fun x -> INDENT`. Don't push when INDENT comes from `module M = INDENT`.

Then `blockLet = match context with | InExprBlock _ :: _ | InLetDecl _ :: _ -> true | _ -> false`
</task>

<task id="3" title="Implement offside-based IN insertion">
The key change: when processing NEWLINE tokens, check if any CtxtLetDecl in the context
stack has its offside column reached.

**In processNewlineWithContext (or in the filter loop after NEWLINE processing):**

After computing `emitted` tokens:

```fsharp
// Check for offside rule: if next token's column <= offsideCol of a CtxtLetDecl(blockLet=true),
// that let binding's scope has ended → insert IN and pop the context.
let rec checkOffside ctx =
    match ctx with
    | InLetDecl(true, offsideCol) :: rest when col <= offsideCol ->
        // Pop this context, insert IN
        state <- { state with Context = rest }
        yield Parser.IN
        // Continue checking — there may be nested CtxtLetDecls
        yield! checkOffside rest
    | _ -> ()

// Only check offside if we're not in match/try pipe context
match nextToken with
| Some Parser.PIPE -> () // don't check offside for match pipes
| _ -> checkOffside state.Context
```

This replaces all the `LetSeqDepth` logic. The offside rule naturally handles:
- Single let + body: CtxtLetDecl(col=4) → next token at col=4 → IN inserted
- Let chain: each LET pushes its own CtxtLetDecl → each gets IN when its offside is reached
- Nested: inner let has higher offsideCol → pops first, then outer pops
- Module: InModule context means blockLet=false → no IN inserted

**Handle explicit IN token:**
```fsharp
| Parser.IN ->
    // Pop the matching CtxtLetDecl from context
    match state.Context with
    | InLetDecl(_, _) :: rest ->
        state <- { state with Context = rest; PrevToken = Some token }
    | _ ->
        state <- { state with PrevToken = Some token }
    yield token
```
</task>

<task id="4" title="Handle INDENT/DEDENT with expression block tracking">
Track expression blocks vs module blocks when INDENT is emitted:

**After processNewlineWithContext emits INDENT:**
```fsharp
// When INDENT is emitted, push the appropriate block context
if List.contains Parser.INDENT emitted then
    match state.PrevToken with
    | Some Parser.EQUALS when not (isInModuleContext state.Context) ->
        // Expression block: let x = INDENT body
        state <- { state with Context = InExprBlock(col) :: state.Context }
    | Some Parser.ARROW ->
        // Lambda body: fun x -> INDENT body
        state <- { state with Context = InExprBlock(col) :: state.Context }
    | _ -> () // Module INDENT or other — don't push expression block
```

**When DEDENT is encountered:**
```fsharp
| Parser.DEDENT ->
    // Pop expression/module block contexts as needed
    let rec popBlocks ctx =
        match ctx with
        | InExprBlock _ :: rest -> rest  // pop expression block
        | InModule :: rest -> rest       // pop module block
        | InLetDecl(true, _) :: rest ->
            // DEDENT past a let binding — insert IN first
            // This handles: let x = INDENT ... DEDENT where DEDENT closes the let scope
            rest
        | _ -> ctx
    state <- { state with Context = popBlocks state.Context }
    state <- updateContextOnDedent state
    yield token
```

The offside rule on DEDENT: if we DEDENT below an InLetDecl's offside column,
that binding is complete → insert IN and pop.
</task>
