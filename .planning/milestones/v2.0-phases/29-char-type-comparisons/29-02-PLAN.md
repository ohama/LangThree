<task type="auto">
  <name>Task 1: Widen comparison type inference in Bidir.fs</name>
  <files>src/LangThree/Bidir.fs</files>
  <action>
Find the comparison operator case in `synth`. It currently looks like:

```fsharp
| LessThan (e1, e2, _) | GreaterThan (e1, e2, _)
| LessEqual (e1, e2, _) | GreaterEqual (e1, e2, _) ->
    let s = inferBinaryOp ctorEnv recEnv ctx env e1 e2 TInt TInt
    (s, TBool)
```

Replace with the synth-then-unify-then-check pattern (mirrors the `Add` operator in Bidir.fs):

```fsharp
| LessThan (e1, e2, span) | GreaterThan (e1, e2, span)
| LessEqual (e1, e2, span) | GreaterEqual (e1, e2, span) ->
    let s1, t1 = synth ctorEnv recEnv ctx env e1
    let s2, t2 = synth ctorEnv recEnv ctx (applyEnv s1 env) e2
    let appliedT1 = apply s2 t1
    let s3 = unifyWithContext ctx [] span appliedT1 t2
    let resultTy = apply s3 appliedT1
    match resultTy with
    | TInt | TString | TChar -> (compose s3 (compose s2 s1), TBool)
    | TVar _ ->
        // Ambiguous — default to int for backward compatibility
        let s4 = unifyWithContext ctx [] span resultTy TInt
        (compose s4 (compose s3 (compose s2 s1)), TBool)
    | _ ->
        raise (TypeException {
            Kind = UnifyMismatch (TInt, resultTy)
            Span = span
            Term = Some expr
            ContextStack = ctx
            Trace = []
        })
```

Note: The `span` variable name is needed here (not `_`) because it is used in `unifyWithContext` calls. Check the existing pattern-match variable names in the comparison case to confirm they match the new binding names.

<task type="auto">
  <name>Task 2: Widen comparison evaluation in Eval.fs</name>
  <files>src/LangThree/Eval.fs</files>
  <action>
Find the four comparison operator eval cases in `eval`. They currently only handle `IntValue`. Extend each to also handle `StringValue` (using `System.String.CompareOrdinal`) and `CharValue`:

```fsharp
| LessThan (left, right, _) ->
    match eval recEnv moduleEnv env false left, eval recEnv moduleEnv env false right with
    | IntValue l, IntValue r -> BoolValue (l < r)
    | StringValue l, StringValue r -> BoolValue (System.String.CompareOrdinal(l, r) < 0)
    | CharValue l, CharValue r -> BoolValue (l < r)
    | _ -> failwith "Type error: < requires operands of same ordered type (int, string, or char)"

| GreaterThan (left, right, _) ->
    match eval recEnv moduleEnv env false left, eval recEnv moduleEnv env false right with
    | IntValue l, IntValue r -> BoolValue (l > r)
    | StringValue l, StringValue r -> BoolValue (System.String.CompareOrdinal(l, r) > 0)
    | CharValue l, CharValue r -> BoolValue (l > r)
    | _ -> failwith "Type error: > requires operands of same ordered type (int, string, or char)"

| LessEqual (left, right, _) ->
    match eval recEnv moduleEnv env false left, eval recEnv moduleEnv env false right with
    | IntValue l, IntValue r -> BoolValue (l <= r)
    | StringValue l, StringValue r -> BoolValue (System.String.CompareOrdinal(l, r) <= 0)
    | CharValue l, CharValue r -> BoolValue (l <= r)
    | _ -> failwith "Type error: <= requires operands of same ordered type (int, string, or char)"

| GreaterEqual (left, right, _) ->
    match eval recEnv moduleEnv env false left, eval recEnv moduleEnv env false right with
    | IntValue l, IntValue r -> BoolValue (l >= r)
    | StringValue l, StringValue r -> BoolValue (System.String.CompareOrdinal(l, r) >= 0)
    | CharValue l, CharValue r -> BoolValue (l >= r)
    | _ -> failwith "Type error: >= requires operands of same ordered type (int, string, or char)"
```

Use `System.String.CompareOrdinal` (not `System.String.Compare`) for ordinal byte-by-byte semantics. This ensures `"abc" < "def"` returns true predictably regardless of locale.
