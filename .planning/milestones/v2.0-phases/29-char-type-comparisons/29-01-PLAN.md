**Ast.fs** — Add three variants in their respective DUs:

1. In the `TypeExpr` DU, add `| TEChar` (alongside TEInt, TEBool, TEString).

2. In the `Expr` DU, add `| Char of char * Span` (alongside `| String of string * Span`).

3. In the `Constant` DU (used for pattern matching), add `| CharConst of char` (alongside `StringConst`, `IntConst`, `BoolConst`).

4. In the `Value` DU (which has `[<CustomEquality; CustomComparison>]`), add `| CharValue of char`. Then update the three explicit implementations:
   - In `valueEqual`: add `| CharValue a, CharValue b -> a = b`
   - In `valueCompare`: add `| CharValue a, CharValue b -> compare a b`
   - In `GetHashCode()` match: add `| CharValue c -> hash c`

**Type.fs** — Add `| TChar` to the `Type` DU. Then update all match exhaustion points:
   - `formatType`/`formatTypeNormalized`: add `| TChar -> "char"`
   - `freeVars`: extend the base-case line to include TChar: `| TInt | TBool | TString | TChar | TExn -> Set.empty`
   - `apply`: add `| TChar -> TChar`

**Lexer.fsl** — Add char literal rules in the `tokenize` rule BEFORE the `type_var` rule. Use the inline approach (no sub-lexer needed — FsLex longest-match handles disambiguation because `'A'` is 3 chars vs type_var `'a` is 2 chars):

```
| '\'' '\\' 'n' '\''    { CHAR '\n' }
| '\'' '\\' 't' '\''    { CHAR '\t' }
| '\'' '\\' '\\' '\''   { CHAR '\\' }
| '\'' '\\' '\'' '\''   { CHAR '\'' }
| '\'' [^ '\'' '\\'] '\'' { CHAR ((lexeme lexbuf).[1]) }
```

Also add a keyword rule for `"char"` → `TYPE_CHAR` in the keywords section (alongside "string" → TYPE_STRING, "int" → TYPE_INT, etc.).

**Parser.fsy** — Add token declarations:
```
%token <char> CHAR
%token TYPE_CHAR
```

Add grammar rules:
- In `AtomicTypeExpr` rule: `| TYPE_CHAR { TEChar }`
- In `AtomicExpr` rule: `| CHAR { Char ($1, symSpan parseState 1) }`
- In the pattern rule where ConstPat is built: `| CHAR { ConstPat (CharConst $1, symSpan parseState 1) }`

**Elaborate.fs** — Add `| TEChar -> (TChar, vars)` in `elaborateWithVars`. Also add `| Ast.TEChar -> TChar` in `substTypeExprWithMap` and `| Ast.TEChar -> Set.empty` in `collectTypeExprVars`.

**Eval.fs** — Four additions:

   ```fsharp

   ```

2. In `formatValue`, find `StringValue s -> sprintf "\"%s\"" s` and add:
   ```fsharp
   | CharValue c -> sprintf "'%c'" c
   ```

3. In `valuesEqual` (or wherever string/int value equality is checked), add:
   ```fsharp
   | CharValue a, CharValue b -> a = b
   ```

4. In `matchPattern`, find the `ConstPat(StringConst ...)` case and add:
   ```fsharp
   | ConstPat (CharConst c1, _), CharValue c2 -> if c1 = c2 then Some [] else None
   ```

5. In `initialBuiltinEnv`, add the two conversion builtins:
   ```fsharp
   "char_to_int", BuiltinValue (fun v ->
       match v with
       | CharValue c -> IntValue (int c)
       | _ -> failwith "char_to_int: expected char argument")

   "int_to_char", BuiltinValue (fun v ->
       match v with
       | IntValue n ->
           if n < 0 || n > 127 then
               failwithf "int_to_char: value %d out of ASCII range (0-127)" n
           else
               CharValue (char n)
       | _ -> failwith "int_to_char: expected int argument")
   ```

**TypeCheck.fs** — In `initialTypeEnv`, add type schemes for the two builtins:
```fsharp
"char_to_int", Scheme([], TArrow(TChar, TInt))
"int_to_char", Scheme([], TArrow(TInt, TChar))
```


**Format.fs** — In `formatPattern`, find the `Ast.StringConst s` case and add:
```fsharp

```

Note: Exhaustive.fs needs no change — it already maps all `ConstPat _` to `WildcardPat`.
