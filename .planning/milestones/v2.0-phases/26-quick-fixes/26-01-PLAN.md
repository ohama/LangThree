    In `src/LangThree/Eval.fs`, find `initialBuiltinEnv` (around line 145). Locate the closing `]` of the map literal. Add this entry before the closing bracket:

    ```fsharp
    // failwith : string -> 'a  (raises LangThreeException so try-with can catch it)
    "failwith", BuiltinValue (fun v ->
        match v with
        | StringValue msg -> raise (LangThreeException (StringValue msg))
        | _ -> failwith "failwith: expected string argument")
    ```

    CRITICAL: Must use `LangThreeException (StringValue msg)` — NOT `System.Exception`. The language's `try-with` evaluator in `Eval.fs` only catches `LangThreeException`. Using `System.Exception` would make `try-with` unable to catch it in user code.

    In `src/LangThree/TypeCheck.fs`, find `initialTypeEnv` (around line 14). Locate the closing `]` of the map literal. Add this entry before the closing bracket:

    ```fsharp
    // failwith : string -> 'a  (polymorphic return — unifies with any expected type, like raise)
    "failwith", Scheme([0], TArrow(TString, TVar 0))
    ```

    CRITICAL: Return type must be `TVar 0` (inside `Scheme([0], ...)`), NOT `TTuple []` (unit). If it were unit, `if cond then failwith "msg" else someValue` would fail type-checking because branches would have different types. The polymorphic return unifies with whatever the context expects.

    After editing, verify the map entries have correct F# list separator syntax (`;` between entries or use the pattern already established in each file).
