**Ast.fs** -- Extend the `Decl` discriminated union (currently has LetDecl, TypeDecl, RecordTypeDecl) with three new variants:

```fsharp
type Decl =
    | LetDecl of name: string * body: Expr * Span
    | TypeDecl of TypeDecl
    | RecordTypeDecl of RecordDecl
    // Phase 5 (Modules): Module system declarations
    | ModuleDecl of name: string * decls: Decl list * Span
    | OpenDecl of path: string list * Span
    | NamespaceDecl of path: string list * decls: Decl list * Span
```

Update `declSpanOf` to handle the new variants:
- `ModuleDecl(_, _, s) -> s`
- `OpenDecl(_, s) -> s`
- `NamespaceDecl(_, _, s) -> s`

Also extend the `Module` type to support a top-level module name:
```fsharp
type Module =
    | Module of decls: Decl list * Span
    | NamedModule of name: string list * decls: Decl list * Span
    | NamespacedModule of name: string list * decls: Decl list * Span
    | EmptyModule of Span
```

Update `moduleSpanOf` correspondingly:
- `NamedModule(_, _, s) -> s`
- `NamespacedModule(_, _, s) -> s`

**Diagnostic.fs** -- Add four new `TypeErrorKind` variants after `GadtReturnTypeMismatch`:

```fsharp
// Phase 5 (Modules): Module error kinds
| CircularModuleDependency of cycle: string list   // E0501
| UnresolvedModule of name: string                 // E0502
| DuplicateModuleName of name: string              // E0503
| ForwardModuleReference of name: string           // E0504
```

Add corresponding cases in `typeErrorToDiagnostic`:
- `CircularModuleDependency cycle` -> E0501, message showing cycle path (e.g., "Circular module dependency: A -> B -> A"), hint "Remove the circular dependency between modules"
- `UnresolvedModule name` -> E0502, message "Unresolved module: {name}", hint "Make sure the module is defined before use"
- `DuplicateModuleName name` -> E0503, message "Duplicate module name: {name}", hint "Each module must have a unique name"
- `ForwardModuleReference name` -> E0504, message "Forward reference to module: {name}", hint "Modules must be defined before they can be opened (top-to-bottom order)"
  </action>
