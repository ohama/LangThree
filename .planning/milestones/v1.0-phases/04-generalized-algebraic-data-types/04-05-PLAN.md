**GADT-02: Type Refinement in Pattern Matching**
- Basic refinement: match on `Int n` knows `n : int` when result type is `int`
  ```
  type 'a expr =
      | Int : int -> int expr
      | Bool : bool -> bool expr

  let eval_int (e : int expr) : int =
      match e with
      | Int n -> n
  ```
  (Should type-check -- `Int n` refines 'a to int, so result int is correct. Bool branch impossible for int expr.)

- Multi-branch refinement with type annotation:
  ```
  type 'a expr =
      | Int : int -> int expr
      | Bool : bool -> bool expr
      | If : bool expr * 'a expr * 'a expr -> 'a expr

  let eval (e : 'a expr) : 'a =
      match e with
      | Int n -> n
      | Bool b -> b
      | If (c, t, f) -> if true then t else f
  ```
  Wait -- this won't work because `eval` would need to recursively call itself on `c`, `t`, `f` with different type instantiations (polymorphic recursion). Let's start simpler:

  ```
  type 'a expr =
      | Int : int -> int expr
      | Bool : bool -> bool expr

  let to_int (e : int expr) : int =
      match e with
      | Int n -> n
  ```

  And separately test that wrong result type fails:
  ```
  let bad (e : int expr) : bool =
      match e with
      | Int n -> n
  ```
  This should fail because Int branch returns int but expected bool.

**GADT-03: Existential Types**
- Existential type basic usage:
  ```
  type showable =
      | Show : 'a * ('a -> string) -> showable

  let show_it (s : showable) : string =
      match s with
      | Show (x, f) -> f x
  ```
  Should type-check: 'a is existential, f : 'a -> string, x : 'a, f x : string.

- Existential escape error:
  ```
  type showable =
      | Show : 'a * ('a -> string) -> showable

  let leak (s : showable) : 'b =
      match s with
      | Show (x, f) -> x
  ```
  Should produce E0402: x has existential type 'a which would escape.

  Actually, the escape detection depends on whether the result type mentions existential vars. If the function returns 'b (a fresh var), and x : 'a_fresh (existential), then unifying 'b with 'a_fresh would make 'b = 'a_fresh, and the result type DOES mention the existential. This should trigger the escape error.

**GADT-04: Annotation Required Error**
- GADT match without annotation:
  ```
  type 'a expr =
      | Int : int -> int expr

  let bad e =
      match e with
      | Int n -> n
  ```
  Should produce E0401: "GADT pattern match requires type annotation"
