**Important format notes:**
- In --expr mode, the interpreter evaluates and prints the *result value*. print/println/printf return `TupleValue []` displayed as `()`.
- Side-effect output (e.g., `hello`) appears BEFORE the result `()`.
- For print (no newline): output is `hello()` — the side effect text immediately followed by the result on the same line.
- For println (with newline): output is `hello` on its own line, then `()` on the next line.
- Run the actual binary to confirm exact output format for each case before writing the .flt file. This avoids Output mismatch test failures.
- String literal quoting in --expr shell args: use single-quote wrapping for the outer shell, double-quote escaping inside for F# strings. E.g.: `--expr 'print "hello"'`

**Pattern for file-mode tests with input:**
Look at an existing file-mode test (e.g., `tests/flt/file/let-sequence.flt`) to confirm the `%input` and `--- Input:` format. File-mode tests use `--file %input` where `%input` is a temp file created from the `--- Input:` section.

**tests/flt/file/print-sequence.flt:**

```
// Test: print side effects happen in order during file evaluation
// --- Command: /Users/ohama/vibe-coding/LangThree/src/LangThree/bin/Release/net10.0/LangThree --file %input
// --- Input:
let _ = print "a"

let _ = print "b"

let result = 42
// --- Output:
ab42
```

Explanation:
- `let _ = print "a"` evaluates `print "a"`, side-effects `a` to stdout, binds unit to `_`
- `let _ = print "b"` evaluates `print "b"`, side-effects `b` to stdout, binds unit to `_`
- `let result = 42` evaluates 42; in file mode, Program.fs prints the last binding's value
- Output: `a` then `b` (from print calls in order) then `42` (from the final binding display)
- Since print does not add newlines, all appear on one line: `ab42`
