# Milestone v10.1: Type Class Error Reporting & Module Integration

**Status:** IN PROGRESS
**Phases:** 75-78
**Estimated Plans:** 8

## Phases

### Phase 75: E0701 Source Location Fix (R1, R8)
**Goal:** All NoInstance errors show the source location where the constraint originated
- Add `Span` field to `Constraint` type
- Thread span through `pendingConstraints` from instantiation sites
- Apply substitution to constraint TypeArg before error message (show concrete type)

### Phase 76: Module Export for TypeClass/Instance (R2, R3, R4)
**Goal:** TypeClassDecl and InstanceDecl inside modules are properly exported
- Add `ClassEnv` and `InstanceEnv` to `ModuleExports`
- Propagate inner ClassEnv/InstanceEnv from module processing (stop discarding)
- Instance elaboration produces bindings visible outside module
- Typeclass redeclaration produces E0707 error

### Phase 77: Constraint Annotation Validation & E0704 (R5, R6, R7)
**Goal:** Type annotations with constraints are validated; method mismatch uses E0704
- Validate class names in `TEConstrained` during elaboration or type checking
- Catch method body type mismatch and re-raise as E0704
- Fix E0702 span to use the user's declaration span

### Phase 78: Integration Tests (All)
**Goal:** Comprehensive flt tests for all bug fixes
- E0701 span tests (top-level, module, nested)
- Module typeclass/instance export tests
- Typeclass redeclaration error test
- Constraint annotation validation test
- E0704 method mismatch test
- Regression tests for existing functionality
