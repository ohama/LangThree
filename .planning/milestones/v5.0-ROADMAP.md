# Milestone v5.0: Imperative Ergonomics

**Status:** SHIPPED 2026-03-28
**Phases:** 45-49
**Total Plans:** 5

## Overview

v5.0 adds four imperative syntax constructs that complement v4.0's mutable variables: expression sequencing (`;`), while and for loops, array/hashtable indexing syntax (`arr.[i]`), and if-then without else. Sequencing comes first because loops depend on it in their bodies. Indexing syntax is independent and follows. If-then is the simplest addition and rounds out the feature set before tests and documentation close the milestone.

## Milestone Summary

**Key Decisions:**
- SeqExpr nonterminal wraps Expr for statement positions — OCaml parser.mly pattern (avoids LALR conflict with list/record semicolons)
- DOTLBRACKET single token for `.[` — avoids LALR conflict with `.IDENT` field access
- Parser-only desugar for `e1; e2` (→ LetPat(WildcardPat)) and `if-then` (→ If with unit else) — zero AST/Eval/Bidir changes
- IndexGet/IndexSet as new AST nodes (not desugar) — need type-level TArray/THashtable dispatch
- For-loop variable immutability via mutableVars exclusion — reuses E0320 from Phase 42

**Issues Resolved:**
- Multi-statement loop bodies require explicit `;` (newline-implicit sequencing not implemented)

**Issues Deferred:**
- Newline-based implicit sequencing inside blocks (currently only `;` works)

**Technical Debt Incurred:**
- None significant
