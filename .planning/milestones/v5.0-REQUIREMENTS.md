# Requirements: LangThree v5.0

**Defined:** 2026-03-28
**Core Value:** v4.0 가변 변수와 자연스럽게 어우러지는 명령형 구문 요소 추가

## v5.0 Requirements

### Array/Hashtable Indexing Syntax

- [x] **IDX-01**: `arr.[i]` reads array element (desugars to `array_get arr i`)
- [x] **IDX-02**: `arr.[i] <- v` writes array element (desugars to `array_set arr i v`)
- [x] **IDX-03**: `ht.[key]` reads hashtable value (desugars to `hashtable_get ht key`)
- [x] **IDX-04**: `ht.[key] <- v` writes hashtable value (desugars to `hashtable_set ht key v`)
- [x] **IDX-05**: Chained indexing works: `matrix.[r].[c]`

### Expression Sequencing

- [x] **SEQ-01**: `e1; e2` evaluates e1 (discards result), then evaluates and returns e2
- [x] **SEQ-02**: Multi-statement sequencing `e1; e2; e3` works (left-associative)
- [x] **SEQ-03**: Sequencing works with indentation-based blocks (offside rule)

### Loop Constructs

- [x] **LOOP-01**: `while cond do body` — repeats body while cond is true, returns unit
- [x] **LOOP-02**: `for i = start to end do body` — ascending loop, i bound in body
- [x] **LOOP-03**: `for i = start downto end do body` — descending loop
- [x] **LOOP-04**: Loop variable `i` is immutable within body (cannot reassign)

### If-Then Without Else

- [x] **IFTHEN-01**: `if cond then expr` accepted when expr returns unit
- [x] **IFTHEN-02**: `if cond then expr` produces type error when expr is non-unit

### Test Coverage

- [x] **TST-28**: Array/hashtable indexing syntax flt tests
- [x] **TST-29**: Expression sequencing flt tests
- [x] **TST-30**: Loop construct flt tests (while, for-to, for-downto)
- [x] **TST-31**: If-then without else flt tests
- [x] **TST-32**: Tutorial chapter update for v5.0 features

## Out of Scope

| Feature | Reason |
|---------|--------|
| `arr[i]` C-style indexing | F# uses `arr.[i]` — consistent with record dot notation |
| `for x in collection do` | Collection iteration via for-in deferred — use `iter`/`map` HOFs |
| List indexing `list.[i]` | Lists are linked lists — O(n) access discouraged by design |
| `do` blocks / `begin`/`end` | Sequencing via `;` sufficient — no block delimiters needed |
| Mutable loop variable | `for` loop variable is always immutable — F# convention |
