# Milestone v10.1: Type Class Error Reporting & Module Integration

## Problem Statement

v10.0 shipped type classes but module system integration has 10 bugs:
- E0701 (NoInstance) errors show `<unknown location>` — no source span
- Instances inside modules not exported (runtime "Undefined variable")
- Typeclasses inside modules not exported (E0703 when used outside)
- Typeclass redeclaration silently ignored
- Unknown class in `=>` annotation silently accepted
- E0704 (MethodTypeMismatch) never fires — falls through to E0301
- E0702 span shows `:0:0` for Prelude-defined classes

## Requirements

### R1: E0701 source location (Bug 1)
`Constraint` type must carry source span. All NoInstance errors must show the let-binding or expression where the constraint was generated.

### R2: Instance export from modules (Bug 2, 4, 5)
`InstanceDecl` inside `ModuleDecl` must produce runtime-accessible bindings. Instance methods elaborated inside a module must be reachable from outside (either via qualified access or `open`).

### R3: Typeclass export from modules (Bug 3)
`TypeClassDecl` inside `ModuleDecl` must register the class in `ClassEnv` beyond the module boundary. Instances written outside a module for a class defined inside must work.

### R4: Typeclass redeclaration error (Bug 6)
Redefining a typeclass with the same name must produce a diagnostic (error or warning).

### R5: Validate class name in constraint annotation (Bug 7)
`TEConstrained` must validate class names against `ClassEnv`. Unknown class name must produce E0703.

### R6: E0704 for method type mismatch (Bug 10)
Instance method body type mismatch should produce E0704 instead of generic E0301.

### R7: E0702 span for Prelude classes (Bug 8)
Duplicate instance error for Prelude-defined classes must show the user's declaration span, not the Prelude's internal span.

### R8: E0701 concrete type in message (Bug 9)
When a constraint fails to resolve at a concrete call site, the error should show the concrete type, not an internal type variable.

## Success Criteria

- All 10 bugs from the test report are fixed
- All existing typeclass flt tests still pass
- New flt tests cover each bug fix
- No regressions in module system tests
