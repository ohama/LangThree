# Requirements: v11.0 Error Reporting Enhancement

**Defined:** 2026-04-01
**Core Value:** 에러 메시지가 문제의 원인과 해결 방법을 명확하게 알려줌

## v11.0 Requirements

### Source Snippets

- [ ] **SNIP-01**: 타입 에러 메시지에 소스 코드 라인과 `^^^` 밑줄이 표시됨
- [ ] **SNIP-02**: 멀티라인 스팬은 첫/마지막 줄만 표시
- [ ] **SNIP-03**: `<unknown>`, `<expr>` 등 파일이 없는 경우 스니펫 생략

### Did You Mean

- [ ] **DYM-01**: `UnboundVar` 에러에 Levenshtein distance 기반 유사 변수 제안
- [ ] **DYM-02**: `UnboundConstructor` 에러에 유사 생성자 제안
- [ ] **DYM-03**: `UnresolvedModule` 에러에 유사 모듈명 제안
- [ ] **DYM-04**: `UnboundField` 에러에 해당 레코드의 필드명 제안
- [ ] **DYM-05**: `UnknownTypeClass` 에러에 유사 클래스명 제안

### Parser Errors

- [ ] **PERR-01**: parse error에 "expected X but got Y" 형태의 메시지 포함
- [ ] **PERR-02**: parse error에 소스 위치 (파일명, 줄, 컬럼) 포함

### Typeclass Error Detail

- [ ] **TCERR-01**: E0701 NoInstance 에러에 사용 가능한 인스턴스 목록 표시

### Multi-Error Reporting

- [ ] **MULTI-01**: 선언 단위로 에러를 수집하여 한번에 보고 (첫 번째 에러에서 중단하지 않음)
- [ ] **MULTI-02**: 후속 선언의 cascading 에러 최소화

### Runtime Error Location

- [ ] **RTERR-01**: 런타임 에러 (0 나누기, 인덱스 초과 등)에 소스 위치 포함

## Out of Scope

| Feature | Reason |
|---------|--------|
| ANSI 컬러 출력 | 사용자 요청에 따라 제외 |
| JSON 구조화 출력 | 사용자 요청에 따라 제외 |
| 파서 에러 복구 (error production) | LALR 한계로 실용성 낮음, parse_error_rich만 구현 |
| 경고 레벨 제어 (-Wall 등) | 현재 경고 3종으로 불필요 |
