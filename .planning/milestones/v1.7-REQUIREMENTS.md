# Requirements: LangThree v1.6

**Defined:** 2026-03-20
**Core Value:** F#-style offside rule로 `in` 키워드 완전 생략 지원

## v1.6 Requirements

### Offside Context (CTX)

- [x] **CTX-01**: CtxtLetDecl 컨텍스트 — let 토큰 시 컨텍스트 스택에 푸시, blockLet 여부 추적
- [x] **CTX-02**: CtxtSeqBlock 컨텍스트 — = 뒤 RHS 블록, offside column 기반 범위 추적
- [x] **CTX-03**: 모듈/expression 구분 — SeqBlock 안 let은 blockLet, 괄호 안 let은 non-block

### Token Transformation (TOK)

- [x] **TOK-01**: ODECLEND 삽입 — offside column에 도달 시 암시적 `in` 역할의 토큰 삽입
- [x] **TOK-02**: 명시적 IN 호환 — 기존 explicit `in`도 동작 유지
- [x] **TOK-03**: OBLOCKSEP 삽입 — 같은 레벨의 선언 구분 (모듈 내부)

### Coverage (COV)

- [x] **COV-01**: 단순 let 체인 — `let x = 1 / let y = 2 / x + y` (in 없이)
- [x] **COV-02**: let rec 체인 — `let rec f = ... / let rec g = ... / body`
- [x] **COV-03**: 중첩 let — `let b = (let inner = 5 / inner * 2) / a + b`
- [x] **COV-04**: let rec + multiline fun -> — `let rec map f = fun xs -> ... / body`
- [x] **COV-05**: 모듈 안 let — module M = 안의 let 선언은 IN 삽입하지 않음
- [x] **COV-06**: 기존 테스트 전부 통과
