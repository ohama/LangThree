# Milestone v10.0: Type Classes

**Status:** ✅ SHIPPED 2026-03-31
**Phases:** 70-74
**Total Plans:** 11

## Overview

v10.0 adds Haskell-style type classes to LangThree's existing Hindley-Milner type system via dictionary passing. Five phases: type infrastructure, parsing/AST, constraint inference, dictionary elaboration, and built-in instances.

## Milestone Summary

**Key Decisions:**
- Dictionary passing via elaborateTypeclasses (InstanceDecl → LetDecl desugaring)
- pendingConstraints as mutable ref in Bidir.fs (same pattern as mutableVars)
- Constraint-aware generalize/instantiate shadow Infer.fs via F# open module
- TypeClassDecl idempotent (prelude + user can both declare same class)
- Eq class separate from hardcoded = operator (migration deferred)
- Pipe-prefix method syntax in typeclass bodies (| show : 'a -> string)

**Issues Resolved:**
- Prelude.fs missing elaborateTypeclasses call before eval
- Parser LALR conflicts with method syntax (resolved with pipe prefix)
- IndentFilter TYPECLASS/INSTANCE treated as module context (JustSawModule)

**Deferred to v10.1:**
- Constrained instances (Show 'a => Show (list 'a))
- Superclass constraints
- Num typeclass (+ - * migration)
- derive automatic instance generation
- = operator migration to Eq
