# Milestone v9.0: Project Build System

**Status:** ✅ SHIPPED 2026-03-31
**Phases:** 67-68
**Total Plans:** 7

## Overview

v9.0 delivers a Cargo-style project build system in two phases: first extending the CLI with `--check`, `--deps`, `--prelude` flags, environment variable support, and file import caching; then adding `funproj.toml` project file parsing with `build` and `test` subcommands for multi-target project management.

## Phases

### Phase 67: CLI Extensions

**Goal**: Users can type-check, inspect dependencies, and configure Prelude paths without executing programs
**Depends on**: Phase 66 (v8.1 complete)
**Requirements**: CLI-01, CLI-02, CLI-03, CLI-04, CLI-05
**Plans**: 4 plans

Plans:
- [x] 67-01-PLAN.md -- Argu flags + Prelude path resolution priority chain
- [x] 67-02-PLAN.md -- --check and --deps mode implementation
- [x] 67-03-PLAN.md -- File import caching
- [x] 67-04-PLAN.md -- flt integration tests for CLI features

**Success Criteria** (all verified ✓):
  1. `langthree --check file.fun` type-checks without executing, reporting to stderr
  2. `langthree --deps file.fun` prints recursive dependency tree with indentation
  3. `langthree --prelude /path file.fun` uses specified Prelude directory
  4. `LANGTHREE_PRELUDE` env var used when `--prelude` not given
  5. File import caching within single process invocation

### Phase 68: Project File

**Goal**: Users can manage multi-target projects with a single `funproj.toml` configuration file
**Depends on**: Phase 67
**Requirements**: PROJ-01, PROJ-02, PROJ-03, PROJ-04, PROJ-05, PROJ-06, PROJ-07
**Plans**: 3 plans

Plans:
- [x] 68-01-PLAN.md -- Tomlyn NuGet + ProjectFile.fs TOML parsing module
- [x] 68-02-PLAN.md -- Build/Test subcommands in Cli.fs + Prelude priority + Program.fs dispatch
- [x] 68-03-PLAN.md -- flt integration tests for build, test, and project prelude

**Success Criteria** (all verified ✓):
  1. `funproj.toml` with `[project]`, `[[executable]]`, `[[test]]` parsed and validated
  2. `langthree build [name]` type-checks executable targets
  3. `langthree test [name]` executes test targets
  4. `[project].prelude` sets Prelude path (lower priority than CLI/env)
  5. flt integration tests cover all features

## Milestone Summary

**Key Decisions:**
- Prelude priority order: --prelude flag > LANGTHREE_PRELUDE env > funproj.toml [project].prelude > auto-discovery
- File import caching stores file's own exports only (not merged with caller) for diamond dependency safety
- Argu subcommands via [<CliPrefix(CliPrefix.None)>] on individual Build/Test cases
- Tomlyn 2.3.0 with [<CLIMutable>] F# records for TOML POCO deserialization
- funproj.toml paths resolved relative to project file directory, not CWD

**Issues Resolved:**
- Program.fs prelude loading order bug (loadPrelude called before flag extraction)
- Repl.fs and ModuleTests.fs call site updates for new loadPrelude signature
- flt backslash escaping issue (\x22 hex escape needed instead of \")

- None
