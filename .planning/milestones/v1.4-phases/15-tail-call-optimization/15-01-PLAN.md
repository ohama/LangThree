<task id="3" title="Implement trampoline in App and PipeRight">
Extract `applyFunc` helper that handles FunctionValue/BuiltinValue dispatch:

```fsharp
let applyFunc recEnv moduleEnv funcVal argVal funcExpr tailPos =
    match funcVal with
    | FunctionValue (param, body, closureEnv) ->
        // Add self-reference for recursive calls
        let augmented = match funcExpr with
            | Var (name, _) -> Map.add name funcVal closureEnv
            | _ -> closureEnv
        let callEnv = Map.add param argVal augmented
        eval recEnv moduleEnv callEnv tailPos body
    | BuiltinValue fn -> fn argVal
    | _ -> failwith "Type error: attempted to call non-function"
```

In `App` case, add trampoline loop:

```fsharp
| App (funcExpr, argExpr, _) ->
    let funcVal = eval recEnv moduleEnv env false funcExpr
    let argValue = eval recEnv moduleEnv env false argExpr
    let mutable result = applyFunc recEnv moduleEnv funcVal argValue funcExpr tailPos
    while (match result with TailCall _ -> true | _ -> false) do
        match result with
        | TailCall (f, a) -> result <- applyFunc recEnv moduleEnv f a funcExpr true
        | _ -> ()
    result
```

In `applyFunc`, when `tailPos=true` for FunctionValue, the inner eval runs with `tailPos=true`.
If the body ends with another App in tail position, it returns TailCall instead of recursing.

Apply same pattern to `PipeRight`:

```fsharp
| PipeRight (left, right, _) ->
    let argVal = eval recEnv moduleEnv env false left
    let funcVal = eval recEnv moduleEnv env false right
    let mutable result = applyFunc recEnv moduleEnv funcVal argVal right tailPos
    while (match result with TailCall _ -> true | _ -> false) do
        match result with
        | TailCall (f, a) -> result <- applyFunc recEnv moduleEnv f a right true
        | _ -> ()
    result
```

**Critical:** When `tailPos=true` in App, and the body returns a TailCall, the
trampoline loop catches it. When `tailPos=false`, the body evaluates with `tailPos=false`
so no TailCall is ever returned — normal recursive behavior.
</task>
