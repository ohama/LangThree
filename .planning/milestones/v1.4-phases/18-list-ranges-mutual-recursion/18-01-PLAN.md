<objective>
Add list range syntax: `[1..5]` and `[1..2..10]` for generating integer lists.

Add DOTDOT token to lexer, Range AST node, parser grammar, evaluation (list generation),
type checking (int -> int list), and formatting.
</objective>

<tasks>

<task id="1" title="Add DOTDOT token and Range AST">
**Lexer.fsl:** Add DOTDOT token rule BEFORE the DOT rule:
```
| ".."    { DOTDOT }
```

**Parser.fsy:** Declare DOTDOT token. Add Range grammar rules inside list expressions:
```
| LBRACKET Expr DOTDOT Expr RBRACKET  { Range($2, None, $4, ruleSpan ...) }
| LBRACKET Expr DOTDOT Expr DOTDOT Expr RBRACKET  { Range($2, Some $4, $6, ruleSpan ...) }
```

**Ast.fs:** Add Range to Expr DU:
```fsharp
| Range of start: Expr * step: Expr option * stop: Expr * Span
```

Update `spanOf` for Range.

**Format.fs:** Add formatAst for Range node.
</task>

<task id="2" title="Implement Range evaluation and type checking">
**Eval.fs:** Add eval case for Range:
```fsharp
| Range(startExpr, stepExpr, stopExpr, _) ->
    let startVal = match eval ... startExpr with IntValue n -> n | _ -> failwith "Range start must be int"
    let stopVal = match eval ... stopExpr with IntValue n -> n | _ -> failwith "Range stop must be int"
    let step = match stepExpr with
        | Some e -> (match eval ... e with IntValue n -> n | _ -> failwith "Range step must be int") - startVal
        | None -> 1
    // Generate list: [start; start+step; start+2*step; ...] while <= stop
    let rec gen n = if n > stopVal then [] else IntValue n :: gen (n + step)
    ListValue (if step > 0 then gen startVal else [])
```

**TypeCheck.fs / Bidir.fs:** Add synth case — Range always produces `int list`:
- start, step, stop must all be `int`
- Result type is `TList TInt`

Update any exhaustive pattern matches on Expr (collectMatches, rewriteModuleAccess, etc.).
</task>
