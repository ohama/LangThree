<task id="1" title="Create tutorial directory and Chapter 1: Getting Started">
Create `tutorial/` directory and write `tutorial/01-getting-started.md`:

**Content to cover:**
- How to run LangThree: `--expr` mode, file mode, `--emit-ast`, `--emit-type`, REPL
- Integer literals and arithmetic: `+`, `-`, `*`, `/`
- Boolean literals and operators: `true`, `false`, `&&`, `||`, `not` (if exists)
- String literals with escape sequences: `"hello\nworld"`
- String concatenation with `+` operator
- Comparison operators: `=`, `<>`, `<`, `>`, `<=`, `>=`
- Comments (if supported)
- Short-circuit evaluation: `false && (1/0 = 0)` doesn't crash

**Format:** Each example shows the command and expected output:
```
$ langthree --expr '1 + 2 * 3'
7
```

For file mode examples, show the file content and output.

**Important syntax notes to mention:**
- Indentation-based (F# style, no semicolons/braces)
- `let` bindings at module level (no `in` needed)
- Last binding's value is printed in file mode
- Warning about Prelude.fun (may appear in --expr mode)
</task>

<task id="2" title="Chapter 2: Functions">
Write `tutorial/02-functions.md`:

**Content to cover:**
- Anonymous functions: `fun x -> x + 1`
- Let bindings (expression): `let x = 5 in x + 1`
- Let bindings (module level): `let x = 5` (file mode)
- Multi-parameter functions: `let add x y = x + y` (desugars to nested lambdas)
- Recursive functions: `let rec fact n = if n <= 1 then 1 else n * fact (n - 1) in fact 5`
- Higher-order functions: passing functions as arguments
- Closures: functions capturing outer scope
- Type annotations on lambdas: `fun (x: int) -> x + 1`
- Currying: `let add x y = x + y` then `let add5 = add 5`

**Note:** `let rec` only works at expression level (with `in`), not at module level. This is a known limitation — mention it.
**Note:** Multi-param `let f x y =` works at module level (desugars to lambdas).
</task>

<task id="4" title="Chapter 4: Pattern Matching">
Write `tutorial/04-pattern-matching.md`:

**Content to cover:**
- Basic match syntax (indentation-based):
  ```
  match x with
  | 0 -> "zero"
  | 1 -> "one"
  | _ -> "other"
  ```
- Pattern types:
  - Constant patterns: `| 0 ->`, `| true ->`
  - Variable patterns: `| x ->`
  - Wildcard: `| _ ->`
  - Tuple patterns: `| (x, y) ->`
  - List patterns: `| [] ->`, `| x :: rest ->`
  - Constructor patterns: `| Some x ->`, `| None ->`
  - Nested patterns: `| Some (x :: rest) ->`
  - Record patterns: `| { x = a; y = b } ->`
- When guards: `| x when x > 0 -> "positive"`
- Exhaustiveness checking: compiler warns about missing cases
- Let-pattern destructuring: `let (x, y) = (1, 2) in ...`
- Single-line match (in --expr mode): `match x with | A -> 1 | B -> 2`

**Note:** String constant patterns are NOT supported (mention this).
**Note:** Decision tree compilation means efficient matching at runtime.
</task>
