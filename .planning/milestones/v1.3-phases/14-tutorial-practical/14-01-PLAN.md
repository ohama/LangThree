<task id="1" title="Chapter 8: Modules and Namespaces">
Read tutorial/01-getting-started.md through tutorial/07-gadt.md first to understand the established style.

Write `tutorial/08-modules.md`:

**Content to cover:**
- Module declaration:
  ```
  module M =
      let x = 42
  ```
- Qualified access: `M.x`
- Open directive: `open M` then use `x` directly
- Nested modules:
  ```
  module Outer =
      module Inner =
          let value = 42
  ```
- Chained qualified access: `Outer.Inner.value`
- Modules with type declarations (ADT, records, exceptions inside modules)
- Namespace declaration: `namespace App`
- Module with functions and pattern matching
- Multiple modules in same file with open

**LangThree-specific:**
- Indentation defines module body scope (no `end` keyword)
- `module M =` uses `=` sign (unlike top-level `module M`)
- Circular dependencies not allowed (top-to-bottom ordering)
- Qualified constructor access: `M.SomeConstructor`
</task>

<task id="2" title="Chapter 9: Exceptions">
Write `tutorial/09-exceptions.md`:

**Content to cover:**
- Exception declaration: `exception NotFound`
- Exception with data: `exception InvalidArg of string`
- Raising exceptions: `raise NotFound`, `raise (InvalidArg "bad")`
- Try-with handling:
  ```
  let result = try
      raise NotFound
  with
  | NotFound -> 42
  ```
- Multiple handlers: `| NotFound -> ... | InvalidArg msg -> ...`
- When guards in exception handlers: `| InvalidArg msg when string_length msg > 0 -> ...`
- Nested try-with: inner exception propagates to outer handler
- Exception re-raising when no handler matches
- W0003 warning for non-exhaustive handlers
- Exceptions inside modules: `module E = exception Err`

**LangThree-specific:**
- `raise` takes an Atom (use parens for constructors with args): `raise (Error msg)`
- Exception type is open (can't exhaustively match — hence W0003 warning)
- `try` block uses same indentation rules as `match`
- When guards: `| Error x when x > 0 -> ...`
</task>

<task id="3" title="Chapter 10: Pipes and Composition">
Write `tutorial/10-pipes-and-composition.md`:

**Content to cover:**
- Pipe operator `|>`: `5 |> double |> inc`
- Pipe with built-in functions: `"hello" |> string_length`
- Pipe with lambda: `10 |> (fun x -> x * 2)`
- Forward composition `>>`: `let f = double >> inc` (apply double then inc)
- Backward composition `<<`: `let g = inc << double` (apply double then inc)
- Chained composition: `add1 >> mul2 >> sub3`
- Pipe with function application: `[1,2,3] |> mymap (fun x -> x * 2)` (inline map since no module-level let rec)
- Comparison to F#: same semantics, same operators
- When to use pipe vs composition

**LangThree-specific:**
- Pipe works with both user functions (FunctionValue) and built-in functions (BuiltinValue)
- Composition creates closure-based functions (not special AST rewriting)
- Precedence: PIPE_RIGHT < COMPOSE < other operators
</task>

<task id="4" title="Chapter 11: Strings and Output">
Write `tutorial/11-strings-and-output.md`:

**Content to cover:**
- String literals and escapes: `"hello\nworld"`, `"tab\there"`
- String concatenation with `+`: `"hello" + " " + "world"`
- Built-in string functions:
  - `string_length "hello"` → 5
  - `string_concat "hello" " world"` → "hello world" (curried)
  - `string_sub "hello" 1 3` → "ell" (start, length)
  - `string_contains "hello world" "world"` → true
  - `to_string 42` → "42", `to_string true` → "true"
  - `string_to_int "123"` → 123
- Print functions:
  - `print "hello"` — no newline, returns unit
  - `println "hello"` — with newline, returns unit
  - `printf "x=%d, s=%s" 42 "hi"` — format string
- Printf format specifiers: `%d` (int), `%s` (string), `%b` (bool), `%%` (literal %)
- Side-effect sequencing with `let _ =`:
  ```
  let _ = print "a"
  let _ = print "b"
  let result = 42
  ```
- Pipe with string functions: `"hello" |> string_length`

**LangThree-specific:**
- `string_sub` uses start+length (not start+end)
- `string_concat` is curried: `string_concat "prefix"` returns a function
- `to_string` accepts int, bool, string at runtime (polymorphic type scheme)
- `printf` is curried: each `%` specifier consumes one argument
</task>
