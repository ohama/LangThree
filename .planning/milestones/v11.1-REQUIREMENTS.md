# Requirements: v11.1 Poison Type Multi-Error Reporting

**Defined:** 2026-04-01
**Core Value:** 한 파일에서 여러 타입 에러를 한번에 보고하여 수정 사이클 단축

## v11.1 Requirements

### Core Infrastructure
- [ ] **PT-01**: `TError` Poison Type 추가 — 에러 발생 표현식의 타입
- [ ] **PT-02**: `TError`와의 유니피케이션은 항상 성공 (캐스케이딩 방지)
- [ ] **PT-03**: 에러 축적기 (`accumulatedErrors` in Bidir.fs)

### Error Collection
- [ ] **PT-04**: 선언 내부의 타입 에러를 축적하고 `TError` 반환하여 계속 진행
- [ ] **PT-05**: 한 파일에서 여러 에러 수집 후 한번에 출력
- [ ] **PT-06**: 기존 단일 에러 동작과 호환 (에러 없으면 동일)

### Cascading Prevention
- [ ] **PT-07**: `TError`에서 파생된 2차 에러 필터링 (false positive 방지)

## Out of Scope
| Feature | Reason |
|---------|--------|
| 표현식 내부 다중 에러 | 한 표현식 안에서는 첫 에러에서 중단 (복잡도 대비 효과 낮음) |
| 파서 에러 복구 | LALR 한계, 별도 마일스톤 |
