**Parser.fsy — Add expression-level rules in the Expr production.** Place near other LET rules. Add these variants:

1. `LET MUTABLE IDENT EQUALS Expr IN Expr` — standard inline let mut:
   ```
   { LetMut($3, $5, $7, ruleSpan parseState 1 7) }
   ```

2. `LET MUTABLE IDENT EQUALS INDENT Expr DEDENT IN Expr` — indented value, inline body:
   ```
   { LetMut($3, $6, $9, ruleSpan parseState 1 9) }
   ```

3. `LET MUTABLE IDENT EQUALS INDENT Expr DEDENT` — standalone (no IN; body is unit tuple):
   ```
   { LetMut($3, $6, Tuple([], symSpan parseState 7), ruleSpan parseState 1 7) }
   ```

4. `IDENT LARROW Expr` — variable reassignment:
   ```
   { Assign($1, $3, ruleSpan parseState 1 3) }
   ```
   Place this rule in the Expr production. The parser's LALR(1) lookahead will see LARROW after IDENT and shift rather than reducing IDENT to Atom. This is the expected behavior (shift wins over reduce by default in fsyacc).

**Parser.fsy — Add module-level (Decl) rules.** Place near LetDecl rules:

5. `LET MUTABLE IDENT EQUALS Expr` — simple module-level let mut:
   ```
   { LetMutDecl($3, $5, ruleSpan parseState 1 5) }
   ```

6. `LET MUTABLE IDENT EQUALS INDENT Expr DEDENT` — indented module-level let mut:
   ```
   { LetMutDecl($3, $6, ruleSpan parseState 1 7) }
   ```

**IMPORTANT:** The `IDENT LARROW Expr` rule must be at the Expr level (not Atom level). Check for any shift-reduce conflicts in the fsyacc build output. A conflict on LARROW after IDENT is expected and resolved correctly by default shift preference.
