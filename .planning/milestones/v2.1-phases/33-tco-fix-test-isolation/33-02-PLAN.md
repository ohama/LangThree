<objective>
Eliminate the MatchCompile module's global mutable counter (nextTestVar / freshTestVar / resetTestVarCounter) by making the counter local to compileMatch and threading it through compile as a parameter.

Purpose: Under parallel test execution, concurrent calls to compileMatch reset the shared counter to 0 mid-compilation. This causes TestVar ID collisions inside decision trees, making Leaf nodes bind wrong values at runtime. The fix moves the counter inside compileMatch (one mutable per invocation, no sharing) and updates compile to accept freshTestVar as an explicit parameter.

Output: MatchCompile.fs with three module-level declarations removed, compile signature updated to accept `freshTestVar: unit -> TestVar`, all recursive compile calls updated to thread the parameter through, and compileMatch updated to define a local closure. ISO-01 and ISO-02 satisfied.
</objective>
