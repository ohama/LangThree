# Milestone v11.0: Error Reporting Enhancement

**Status:** IN PROGRESS
**Phases:** 83-88

## Phases

### Phase 83: Source Code Snippets (SNIP-01, SNIP-02, SNIP-03)
**Goal:** 에러 메시지에 소스 코드 라인과 `^^^` 밑줄 표시
- 파일 내용 캐싱 모듈 (SourceDisplay.fs)
- formatDiagnostic 수정: span → 소스 라인 + 밑줄 렌더링
- 멀티라인 스팬 처리, unknown/expr 파일 스킵

### Phase 84: "Did You Mean?" Suggestions (DYM-01~05)
**Goal:** 미정의 이름 에러에 유사 이름 제안
- Levenshtein edit distance 함수 (Suggest.fs)
- TypeError에 scope 정보 추가
- UnboundVar, UnboundConstructor, UnresolvedModule, UnboundField, UnknownTypeClass에 적용

### Phase 85: Parser Error Improvement (PERR-01, PERR-02)
**Goal:** parse error에 예상 토큰과 소스 위치 포함
- parse_error_rich 콜백 구현 (Parser.fsy)
- ParseErrorContext에서 expected/actual 토큰 추출
- ParseException with location 전파

### Phase 86: Typeclass Error Detail (TCERR-01)
**Goal:** NoInstance 에러에 사용 가능한 인스턴스 목록 표시
- E0701 생성 시 currentInstEnv에서 해당 클래스의 인스턴스 목록 포함
- typeErrorToDiagnostic에서 notes에 인스턴스 목록 추가

### Phase 87: Multi-Error Reporting (MULTI-01, MULTI-02)
**Goal:** 선언 단위로 여러 에러를 수집하여 한번에 보고
- typeCheckDecls에서 선언별 try-catch (전략 B)
- 에러 수집 후 최종 반환
- Program.fs에서 복수 에러 포맷 및 출력

### Phase 88: Runtime Error Location (RTERR-01)
**Goal:** 런타임 에러에 소스 위치 포함
- eval 함수에서 현재 표현식 Span 추적
- RuntimeError exception with Span
- 런타임 에러 포맷에 위치 정보 포함
