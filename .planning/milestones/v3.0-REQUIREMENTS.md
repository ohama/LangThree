## v3.0 Requirements

### Mutable Array

- [x] **ARR-01**: `Array.create n default` — 크기 n의 배열 생성 (기본값으로 초기화)
- [x] **ARR-02**: `Array.get arr i` — 인덱스 i의 요소 반환 (범위 초과 시 예외)
- [x] **ARR-03**: `Array.set arr i v` — 인덱스 i에 값 v를 설정 (in-place mutation)
- [x] **ARR-04**: `Array.length arr` — 배열의 길이 반환
- [x] **ARR-05**: `Array.ofList xs` — 리스트를 배열로 변환
- [x] **ARR-06**: `Array.toList arr` — 배열을 리스트로 변환

### Mutable Hashtable

- [x] **HT-01**: `Hashtable.create ()` — 빈 해시테이블 생성
- [x] **HT-02**: `Hashtable.get ht key` — 키로 값 조회 (없으면 예외)
- [x] **HT-03**: `Hashtable.set ht key value` — 키에 값 설정 (in-place mutation)
- [x] **HT-04**: `Hashtable.containsKey ht key` — 키 존재 여부 확인
- [x] **HT-05**: `Hashtable.keys ht` — 모든 키를 리스트로 반환
- [x] **HT-06**: `Hashtable.remove ht key` — 키-값 쌍 제거

### Array 고차 함수

- [x] **ARR-07**: `Array.iter f arr` — 배열의 각 요소에 함수 적용 (부수효과용)
- [x] **ARR-08**: `Array.map f arr` — 배열의 각 요소에 함수 적용하여 새 배열 반환
- [x] **ARR-09**: `Array.fold f init arr` — 배열을 하나의 값으로 축약
- [x] **ARR-10**: `Array.init n f` — f(0), f(1), ..., f(n-1) 으로 배열 생성

### Test Coverage

- [x] **TST-18**: Array 기본 연산 flt 테스트 (create, get, set, length)
- [x] **TST-19**: Array 변환 flt 테스트 (ofList, toList)
- [x] **TST-20**: Array 고차 함수 flt 테스트 (iter, map, fold, init)
- [x] **TST-21**: Hashtable 기본 연산 flt 테스트 (create, get, set, containsKey)
- [x] **TST-22**: Hashtable 추가 연산 flt 테스트 (keys, remove)
- [x] **TST-23**: 튜토리얼 챕터 작성 (Array + Hashtable)
