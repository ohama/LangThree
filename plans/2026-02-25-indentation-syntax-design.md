# Design: Phase 1 - Indentation-Based Syntax

**Date:** 2026-02-25
**Status:** Approved
**Author:** Claude + User

## Overview

LangThree adopts F#-style indentation-based syntax (offside rule), replacing explicit delimiters with whitespace-significant parsing.

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Implementation strategy | FunLang copy + modify | Incremental, preserves working base |
| Processing stage | Token stream filter | Matches F# (LexFilter.fs), testable |
| Compatibility | Indentation-only | Clean break, F# style |
| Tab handling | Reject with error | Prevents subtle bugs |
| Indent width | Configurable (2/4/8) | User preference |
| Priority | let/match/if first | Core constructs |

## Architecture

```
Source.fun
    │
    ▼
┌──────────┐   Raw Tokens    ┌───────────────┐
│ Lexer.fsl │ ─────────────► │ IndentFilter.fs│
└──────────┘   (NEWLINE 포함) └───────────────┘
                                    │
                         Filtered Tokens
                      (INDENT/DEDENT 포함)
                                    │
                                    ▼
                            ┌────────────┐
                            │ Parser.fsy │
                            └────────────┘
                                    │
                                    ▼
                                  AST
```

**New files:**
- `IndentFilter.fs` - Token stream filter (like F#'s LexFilter.fs)

**Modified files:**
- `Lexer.fsl` - NEWLINE token generation, Tab rejection
- `Parser.fsy` - INDENT/DEDENT token handling

## Token Design

### New Tokens

| Token | Description | Generated by |
|-------|-------------|--------------|
| `NEWLINE(col)` | Newline + next line column position | Lexer |
| `INDENT` | Indentation increase | IndentFilter |
| `DEDENT` | Indentation decrease | IndentFilter |

### Token Transformation Example

```
Input:   LET IDENT EQUALS NEWLINE(4) NUMBER NEWLINE(0) ...
Output:  LET IDENT EQUALS INDENT NUMBER DEDENT ...
```

### Indent Stack Operation

```
Stack: [0]
  → NEWLINE(4) → col > top → push 4 → emit INDENT → Stack: [0, 4]
  → NEWLINE(0) → col < top → pop 4 → emit DEDENT → Stack: [0]
```

## IndentFilter Module

```fsharp
// IndentFilter.fs

module IndentFilter

type IndentConfig = {
    IndentWidth: int  // 2, 4, or 8
}

type FilterState = {
    Stack: int list           // indent level stack, starts with [0]
    PendingTokens: Token list // buffered tokens to emit
}

/// Tab character check - error if found
let checkNoTabs (line: string) (lineNum: int) : Result<unit, string>

/// NEWLINE(col) processing - generate INDENT/DEDENT
let processNewline (state: FilterState) (col: int) : FilterState * Token list

/// Token stream filtering (main function)
let filter (config: IndentConfig) (tokens: Token seq) : Token seq
```

### Core Logic

```fsharp
let processNewline state col =
    let rec unwind acc stack =
        match stack with
        | top :: rest when col < top ->
            unwind (DEDENT :: acc) rest
        | top :: _ when col = top ->
            (List.rev acc, stack)
        | top :: _ when col > top ->
            ([INDENT], col :: stack)
        | _ ->
            failwith $"Invalid indentation at column {col}"

    unwind [] state.Stack
```

### Error Messages

- Tab found → `"Tab character not allowed, use spaces"`
- Invalid indent → `"Invalid indentation: expected {expected}, got {actual}"`

## Lexer Changes

```fsharp
// Before: ignore whitespace and newlines
| whitespace+   { tokenize lexbuf }
| newline       { lexbuf.EndPos <- lexbuf.EndPos.NextLine
                  tokenize lexbuf }

// After: reject tabs, generate NEWLINE token
| ' '+          { tokenize lexbuf }                    // spaces only
| '\t'          { failwith "Tab not allowed, use spaces" }
| newline       { lexbuf.EndPos <- lexbuf.EndPos.NextLine
                  let col = getNextLineIndent lexbuf
                  NEWLINE col }
```

## Parser Changes

```fsharp
// Before (FunLang)
let_expr:
    | LET IDENT EQUALS expr IN expr  { ... }

// After (LangThree) - use INDENT/DEDENT
let_expr:
    | LET IDENT EQUALS INDENT expr DEDENT  { ... }
    | LET IDENT EQUALS expr                 { single-line }

match_expr:
    | MATCH expr WITH INDENT match_cases DEDENT  { ... }

if_expr:
    | IF expr THEN INDENT expr DEDENT ELSE INDENT expr DEDENT  { ... }
    | IF expr THEN expr ELSE expr                               { single-line }
```

**Removed tokens:**
- `IN` (let...in → let + indent)

## Test Cases

```fsharp
// 1. let-binding indentation
let x =
    1 + 2

// 2. match expression
let result =
    match opt with
    | Some x -> x
    | None -> 0

// 3. if-then-else
let abs n =
    if n < 0 then
        -n
    else
        n

// 4. nested indentation
let complex =
    let inner =
        match x with
        | A -> 1
        | B -> 2
    inner + 1

// 5. Tab error
let bad =
	42    // Error: Tab not allowed, use spaces

// 6. Invalid indent
let wrong =
    1
  2       // Error: Invalid indentation
```
