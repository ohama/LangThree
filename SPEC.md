# LangThree Language Specification

ML кі„м—ҙ н•ЁмҲҳнҳ• н”„лЎңк·ёлһҳл°Қ м–ём–ҙ LangThreeмқҳ нҳ•мӢқ м •мқҳ.

## 1. Lexical Specification

### 1.1 Character Classes

```
digit       = ['0'-'9']
letter      = ['a'-'z' 'A'-'Z']
whitespace  = [' ' '\t']
newline     = '\n' | '\r\n'
ident_start = letter | '_'
ident_char  = letter | digit | '_'
type_var    = '\'' letter (letter | digit | '_')*
op_char     = ['!' '$' '%' '&' '*' '+' '-' '.' '/' '<' '=' '>' '?' '@' '^' '|' '~']
```

### 1.5 Literals

| Token | Pattern | Example |
|-------|---------|---------|
| NUMBER | `digit+` | `42`, `0`, `100` |
| STRING | `"..."` (escapes: `\n`, `\t`, `\\`, `\"`) | `"hello\nworld"` |
| IDENT | `ident_start ident_char*` | `x`, `myFunc`, `_temp` |
| TYPE_VAR | `'\'' letter ident_char*` | `'a`, `'key` |

### 1.6 User-Defined Operators (INFIXOP)

2ж–Үеӯ—д»ҘдёҠгҒ® `op_char` еҲ—гҒҜгҖҒе…Ҳй ӯж–Үеӯ—гҒ§е„Әе…Ҳй ҶдҪҚгҒҢжұәгҒҫгӮӢ:

| First Character | Token | Precedence | Associativity | Example |
|----------------|-------|------------|---------------|---------|
| `=` `<` `>` `\|` `&` `$` `!` | INFIXOP0 | comparison level | left | `<\|>` |
| `@` `^` | INFIXOP1 | concatenation | right | `^^`, `++` |
| `+` `-` | INFIXOP2 | additive | left | custom `+.` |
| `*` `/` `%` | INFIXOP3 | multiplicative | left | custom `*.` |
| `**` prefix | INFIXOP4 | exponentiation | right | `**` |

Single-character operators (`+`, `-`, `*`, `/`, `%`, `<`, `>`, `=`) are always lexed as built-in tokens, not INFIXOP.

### 1.7 Special Tokens

| Token | Description |
|-------|-------------|
| NEWLINE(col) | Line break + next line's indentation column |
| INDENT | Generated by IndentFilter (column > stack top) |
| DEDENT | Generated by IndentFilter (column < stack top) |
| EOF | End of input |

### 1.8 Comments

```
// single-line comment (to end of line)
(* block comment вҖ” nestable *)
```

### 1.9 Whitespace Rules

- Tabs are forbidden вҖ” spaces only
- Indentation is significant (F# style offside rule)
- Blank lines and comment-only lines do not generate NEWLINE tokens

## 3. Grammar

### 3.1 Program Structure

```
Program     ::= Module
Module      ::= 'module' QualifiedIdent Decls EOF
              | 'namespace' QualifiedIdent Decls EOF
              | Decls EOF

QualifiedIdent ::= IDENT ('.' IDENT)*
```

### 3.2 Declarations

```
Decls       ::= Decl*

Decl        ::= LetDecl | TypeDecl | RecordDecl | TypeAliasDecl
              | LetRecDecl | ModuleDecl | OpenDecl | ExceptionDecl

LetDecl     ::= 'let' IDENT '=' Expr
              | 'let' IDENT ParamList '=' Expr
              | 'let' '_' '=' Expr
              | 'let' OpName ParamList '=' Expr

LetRecDecl  ::= 'let' 'rec' IDENT ParamList '=' Expr
                ('and' IDENT ParamList '=' Expr)*

TypeDecl    ::= 'type' IDENT TypeParams '=' ['|'] Constructors
                ('and' IDENT TypeParams '=' ['|'] Constructors)*

RecordDecl  ::= 'type' IDENT TypeParams '=' '{' RecordFields '}'

TypeAliasDecl ::= 'type' IDENT TypeParams '=' TypeExpr

ModuleDecl  ::= 'module' IDENT '=' INDENT Decls DEDENT

OpenDecl    ::= 'open' QualifiedIdent

ExceptionDecl ::= 'exception' IDENT ('of' TypeExpr)?

ParamList   ::= IDENT+
OpName      ::= '(' INFIXOP ')'
TypeParams  ::= TYPE_VAR*
```

### 3.3 Constructors

```
Constructors ::= Constructor ('|' Constructor)*

Constructor  ::= IDENT                              // nullary ADT
               | IDENT 'of' TypeExpr                // ADT with data
               | IDENT ':' TypeExpr '->' TypeExpr   // GADT
```

### 3.4 Expressions

```
Expr    ::= 'let' IDENT '=' Expr 'in' Expr
          | 'let' IDENT ParamList '=' Expr 'in' Expr
          | 'let' TuplePattern '=' Expr 'in' Expr
          | 'let' '_' '=' Expr 'in' Expr
          | 'let' 'rec' IDENT IDENT '=' Expr 'in' Expr
          | 'if' Expr 'then' Expr 'else' Expr
          | 'fun' IDENT '->' Expr
          | 'fun' '(' IDENT ':' TypeExpr ')' '->' Expr
          | 'fun' '(' ')' '->' Expr
          | 'fun' TuplePattern '->' Expr
          | 'match' Expr 'with' MatchClauses
          | 'try' Expr 'with' MatchClauses
          | Expr '|>' Expr
          | Expr '>>' Expr
          | Expr '<<' Expr
          | Expr '||' Expr
          | Expr '&&' Expr
          | Expr CompOp Expr
          | Expr '::' Expr
          | Expr INFIXOP Expr
          | Expr '+' Term | Expr '-' Term
          | Atom '.' IDENT '<-' Expr
          | INDENT Expr DEDENT
          | Term

CompOp  ::= '=' | '<' | '>' | '<=' | '>=' | '<>'

Term    ::= Term '*' Factor | Term '/' Factor | Term '%' Factor
          | Factor

Factor  ::= '-' Factor | 'raise' Atom
          | AppExpr

AppExpr ::= AppExpr Atom | Atom

Atom    ::= '(' ')'                    // unit
          | NUMBER | IDENT | TRUE | FALSE | STRING
          | '(' Expr ')'
          | '(' Expr ':' TypeExpr ')'  // type annotation
          | '(' Expr ',' ExprList ')'  // tuple
          | '[' ']'                     // empty list
          | '[' Expr ']'               // singleton list
          | '[' Expr ';' SemiExprList ']'  // list
          | '[' Expr '..' Expr ']'     // range
          | '[' Expr '..' Expr '..' Expr ']'  // range with step
          | '(' INFIXOP ')'           // operator as function
          | Atom '.' IDENT            // field access
          | '{' RecordExprInner '}'   // record
```

### 3.5 Patterns

```
Pattern ::= IDENT                        // variable or nullary constructor
          | '_'                           // wildcard
          | NUMBER | '-' NUMBER           // int constant (positive/negative)
          | TRUE | FALSE                  // bool constant
          | STRING                        // string constant
          | '(' PatternList ')'          // tuple pattern
          | '[' ']'                       // empty list
          | Pattern '::' Pattern         // cons
          | IDENT Pattern                // constructor with argument
          | '{' RecordPatFields '}'      // record pattern

OrPattern   ::= Pattern ('|' Pattern)*
MatchClause ::= '|' OrPattern ('when' Expr)? '->' Expr
MatchClauses ::= MatchClause+

PatternList  ::= Pattern ',' Pattern (',' Pattern)*
```

### 3.6 Type Expressions

```
TypeExpr    ::= TupleType '->' TypeExpr   // function type
              | TupleType

TupleType   ::= AtomicType '*' TupleType  // tuple type
              | AtomicType

AtomicType  ::= 'int' | 'bool' | 'string' | 'unit'
              | TYPE_VAR                   // 'a, 'b, ...
              | IDENT                      // named type
              | AtomicType 'list'          // list type
              | AtomicType IDENT           // parameterized type (e.g., int Expr)
              | '(' TypeExpr ')'
```

### 3.7 Records

```
RecordFields      ::= RecordField (';' RecordField)* ';'?
RecordField       ::= ['mutable'] IDENT ':' TypeExpr

RecordExprInner   ::= RecordFieldBindings
                    | Expr 'with' RecordFieldBindings
RecordFieldBindings ::= IDENT '=' Expr (';' IDENT '=' Expr)* ';'?

RecordPatFields   ::= IDENT '=' Pattern (';' IDENT '=' Pattern)* ';'?
```

### 3.8 Lists

```
ExprList     ::= Expr (',' Expr)*          // for tuples (comma-separated)
SemiExprList ::= Expr (';' Expr)*          // for lists (semicolon-separated)
```

## 4. Indentation Rules (IndentFilter)

### 4.1 INDENT / DEDENT Generation

Python м•Ңкі лҰ¬мҰҳ кё°л°ҳ. Indent stack `[0]`м—җм„ң мӢңмһ‘:

- `col > top` вҶ’ INDENT л°ңмғқ, colмқ„ мҠӨнғқм—җ push
- `col = top` вҶ’ лҸҷмқј л ҲлІЁ (нҶ нҒ° м—ҶмқҢ)
- `col < top` вҶ’ DEDENT л°ңмғқ, topмқ„ pop (л°ҳліө к°ҖлҠҘ вҖ” лӢӨлӢЁкі„ dedent)

### 4.2 Context Stack

IndentFilterлҠ” лӢӨмқҢ л¬ёл§Ҙмқ„ м¶”м Ғ:

| Context | Trigger | Effect |
|---------|---------|--------|
| InMatch(col) | `match` нҶ нҒ° + NEWLINE | `\|` нҢҢмқҙн”„лҘј colм—җ м •л ¬ к°•м ң |
| InTry(col) | `try` нҶ нҒ° + NEWLINE | `\|` нҢҢмқҙн”„лҘј colм—җ м •л ¬ к°•м ң |
| InLetDecl(blockLet, offsideCol) | `let` in expression context | blockLet=trueл©ҙ offside columnм—җм„ң implicit `in` мӮҪмһ… |
| InExprBlock(col) | INDENT after `=`, `->`, `in` | Expression block м§„мһ… |
| InModule | INDENT after `module` | Module body (implicit `in` мӮҪмһ… м•Ҳ н•Ё) |
| InFunctionApp(col) | Multi-line function application | мһҗлҸҷ INDENT/DEDENT |

### 4.3 Offside Rule (Implicit `in`)

Expression context м•Ҳмқҳ `let`мқҖ `InLetDecl(blockLet=true, offsideCol)` м»Ён…ҚмҠӨнҠёлҘј push. лӢӨмқҢ нҶ нҒ°мқҳ columnмқҙ offsideCol мқҙн•ҳмқҙл©ҙ implicit `IN` нҶ нҒ° мӮҪмһ…:

```
let result =
    let x = 10       // InLetDecl(true, 4) push
    let y = 20       // col=4 <= offsideCol=4 вҶ’ IN мӮҪмһ…, InLetDecl pop, мғҲ InLetDecl push
    x + y            // col=4 <= offsideCol=4 вҶ’ IN мӮҪмһ…, InLetDecl pop
```

Module contextмқҳ `let`мқҖ `blockLet=false` вҶ’ implicit `in` мӮҪмһ…н•ҳм§Җ м•ҠмқҢ.

### 4.4 Pipe Alignment

`match`/`try` л’Өмқҳ `|` нҢҢмқҙн”„лҠ” `match`/`try` нӮӨмӣҢл“ңмқҳ columnм—җ м •л ¬н•ҙм•ј н•Ё:

```
let result =
    match x with     // InMatch(4) push
    | 0 -> "zero"    // | must be at column 4
    | _ -> "other"   // | must be at column 4
```

## 5. Type System

### 5.1 Core Types

```
Type = int | bool | string | unit
     | 'a, 'b, ...                    // type variables
     | Type -> Type                    // function type
     | Type * Type * ...               // tuple type
     | Type list                       // list type
     | TypeName<Type, ...>             // parameterized type (ADT/GADT)
     | exn                             // exception type
```

### 5.2 Type Inference

Hindley-Milner нғҖмһ… м¶”лЎ  + bidirectional type checking:

- **Synthesis (synth)**: н‘ңнҳ„мӢқм—җм„ң нғҖмһ…мқ„ м¶”лЎ 
- **Checking (check)**: н‘ңнҳ„мӢқмқҙ кё°лҢҖ нғҖмһ…кіј мқјм№ҳн•ҳлҠ”м§Җ кІҖмҰқ
- **Let-polymorphism**: `let` кІҪкі„м—җм„ң нғҖмһ… мқјл°ҳнҷ” (generalization)

### 5.3 GADT Type Refinement

GADT нҢЁн„ҙ л§Өм№ӯм—җм„ң л¶„кё°лі„ лҸ…лҰҪм Ғ нғҖмһ… м •м ң:

```
type Expr 'a =
    | IntLit : int -> int Expr
    | BoolLit : bool -> bool Expr

let eval e =
    match e with
    | IntLit n -> n      // мқҙ л¶„кё°: 'a = int, n : int
    | BoolLit b -> b     // мқҙ л¶„кё°: 'a = bool, b : bool
```

- мЈјм„қ м—ҶмқҢ: лӢӨнҳ•м Ғ л°ҳнҷҳ (`eval : 'a Expr -> 'a`)
- `: int` мЈјм„қ: лӘЁл“  л¶„кё°к°Җ int л°ҳнҷҳ к°•м ң
- л¶Ҳк°ҖлҠҘн•ң мғқм„ұмһҗ мһҗлҸҷ м ңмҷё (completeness filtering)

### 5.4 Exhaustiveness Checking

нҢЁн„ҙ л§Өм№ӯмқҳ мҷ„м „м„ұ кІҖмӮ¬ (W0001 кІҪкі ):
- лӘЁл“  ADT мғқм„ұмһҗ м»ӨлІ„ нҷ•мқё
- GADTм—җм„ңлҠ” нғҖмһ… м •ліҙлЎң л¶Ҳк°ҖлҠҘн•ң мғқм„ұмһҗ н•„н„°л§Ғ
- Or-patternмқҖ к°Ғ лҢҖм•Ҳмқ„ лі„лҸ„лЎң мІҳлҰ¬
- When guardк°Җ мһҲлҠ” л¶„кё°лҠ” мҷ„м „м„ұм—җм„ң м ңмҷё

### 5.6 Prelude (Standard Library)

**Operators:** `++` (list append), `<|>` (Option fallback), `^^` (string concat)
